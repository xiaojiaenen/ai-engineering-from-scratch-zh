# Emu3：面向图像与视频生成的下一词元预测

> BAAI的Emu3（王等人，2024年9月）本应成为2024年终结扩散模型与自回归模型之争的结果。一个单一的Llama风格的仅解码器Transformer，仅通过下一词元预测目标进行训练，在统一的文本+VQ图像词元+3D VQ视频词元词表上，于图像生成方面击败了SDXL，在感知方面击败了LLaVA-1.6。没有CLIP损失。没有扩散调度。推理时使用了无分类器指导以提升质量，但核心训练目标是基于教师强制的下一词元预测。发表于《自然》期刊。本课将阅读Emu3的论文——为什么更好的分词器加上规模就是你所需要的——并与扩散方法进行对比。

**类型：** 学习
**语言：** Python（标准库，3D视频词元数学 + 自回归采样器骨架）
**先决条件：** 阶段12 · 11（Chameleon）
**时间：** 约120分钟

## 学习目标

- 解释为什么Emu3的单一损失下一词元目标能够奏效，尽管长期以来存在图像质量需要扩散模型的假设。
- 描述3D视频词元化器：时空VQ码本是什么样子，为什么补丁具有时间跨度。
- 比较Emu3与Stable Diffusion XL在（训练计算量、推理成本、质量上限）方面的表现。
- 列出同一个Emu3模型扮演的三种角色：Emu3-Gen（图像生成）、Emu3-Chat（感知）、Emu3-Stage2（视频生成）。

## 问题所在

截至2024年的传统观点是：图像生成需要扩散模型。其论点是：离散的图像词元会损失过多信息以重建细节，并且自回归采样会在数千个词元上累积误差。Stable Diffusion、DALL-E 3、Imagen、Midjourney都使用某种形式的扩散模型。Chameleon（课程12.11）在一定程度上以较小规模证伪了这一点，但在质量上未能匹配SDXL。

Emu3正面挑战了这一论点。其主张是：更好的视觉词元化器 + 足够的规模 + 下一词元损失 = 在同一个也能进行感知的模型中实现击败扩散模型的图像生成。

该论文发表时，这一赌注颇具争议。两年后，开源的统一生成家族（Emu3, Show-o, Janus-Pro, Transfusion）已成为研究的默认路径；生产前沿模型似乎也使用了某种变体。

## 核心概念

### Emu3词元化器

关键成分是视觉词元化器。Emu3训练了一个自定义的IBQ类词元化器（逆瓶颈量化器，SBER-MoVQGAN系列），实现了每词元8倍的空间分辨率缩减。一张512x512的图像在码本大小为32768的情况下，变成64x64 = 4096个词元。

这大于Chameleon在K=8192时每张512x512图像产生1024个词元，但每词元成本更低（码本查找更小，编解码器更简单）。关键指标：重建PSNR为30.5 dB，与Stable Diffusion的连续潜空间（32 dB）具有竞争力。

对于视频：一个3D VQ词元化器将一个时空补丁（4x4x4像素）编码为一个整数。一个8 FPS的4秒片段有32帧；在256x256分辨率下，空间缩减4倍，时间缩减4倍，词元数量为 (256/4) * (256/4) * (32/4) = 64 * 64 * 8 = 32,768 个词元。

词元化器的质量决定了上限。Emu3的部分贡献在于“我们训练了一个非常好的词元化器”。

### 单一损失训练

Emu3使用一个目标：在文本词元、2D图像词元和3D视频词元共享词表上的下一词元预测。训练期间权重会乘以模态特定的系数以平衡贡献，但损失函数是相同的。

在以下混合数据上训练：
- 图像生成：`<text caption> <image> image_tokens </image>`
- 图像感知：`<image> image_tokens </image> <question> text_tokens`
- 视频生成：`<text caption> <video> video_tokens </video>`
- 视频感知：类似。
- 纯文本：标准下一词元预测。

模型从数据分布中学习何时输出图像词元 vs 文本词元。生成能力在模型预测`<image>`标签后的图像词元时产生。

### 无分类器指导与温度

自回归图像生成在推理时使用无分类器指导后效果显著提升。Emu3使用了它：生成两次，一次使用完整描述，一次使用空描述，然后以指导权重（典型值为3.0-7.0）混合logits。这与扩散模型使用的CFG技巧相同，被借鉴到了自回归设置中。

温度很重要：太高会产生伪影，太低会导致模式崩塌。Emu3推荐的温度是：感知任务1.0，图像生成0.8。

### 一种模型，三种角色

Emu3以三个功能上不同的API形式发布，但共享同一套底层权重：

- Emu3-Gen。图像生成。输入文本，输出图像词元。
- Emu3-Chat。VQA和图像描述。输入图像（词元），输出文本。
- Emu3-Stage2。视频生成和视频VQA。输入文本或视频，输出文本或视频。

没有任务特定的头部。只是不同的提示模板。同一个检查点。

### 基准测试

来自Emu3论文（2024年9月）：

- 图像生成：在MJHQ-30K FID（5.4 vs 5.6）上击败SDXL，在GenEval整体分数（0.54 vs 0.55——统计平局）上相当，在Deep-Eval的综合得分上表现相当。
- 图像感知：在VQAv2（75.1 vs 72.4）上击败LLaVA-1.6，在MMMU上大致持平。
- 视频生成：4秒片段的质量在FVD上与Sora时代公开基准测试的模型具有竞争力。

这些数字并非总是领先——Emu3在某些方面得一分，在另一些方面失一分——但“下一词元预测就是你所需要的”这一主张在跨模态任务中是站得住脚的。

### 计算成本

Emu3使用一个70亿参数的模型，在约3000亿个多模态词元上进行了训练。GPU小时数大致与Llama-2-7B的预训练相当（在A100级别硬件上约2000-4000 GPU年）。像Stable Diffusion 3这样的扩散模型训练预算相似，但需要单独的文本编码器和更复杂的流水线。

在推理方面，Emu3每张图像的生成速度比SDXL慢：4096个图像词元以30词元/秒的速度处理，生成一张512x512的图像大约需要2分钟，而SDXL只需2-5秒。推测性解码和KV缓存优化缩小了差距但未能完全消除。自回归图像生成是计算密集型的；这是一个存在的权衡。

### 重要性

Emu3的深层贡献是概念性的。如果下一词元预测能扩展到匹配扩散模型的图像生成能力，那么统一模型路径（一个损失，一个骨干网络，任意模态）就是可行的。未来的模型不再需要单独的文本编码器、单独的扩散调度器、单独的VAE。一个Transformer，每个模态一个词元化器，然后扩大规模。

Show-o、Janus-Pro和InternVL-U都建立在或挑战这一论点之上。截至2025年，中国实验室（BAAI、DeepSeek）在这一方向上的发表比美国实验室更为积极。

## 动手实践

`code/main.py` 构建了两个玩具组件：

- 一个2D vs 3D VQ词元计数计算器：给定（分辨率，补丁大小，片段长度，帧率），计算图像 vs 视频的词元数量。
- 一个具有温度和无分类器指导的自回归图像词元采样器。

CFG的实现符合Emu3的配方——使用指导权重混合条件logits和无条件logits。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| 下一词元预测 | "NTP" | 标准自回归损失：给定词元[0..i]预测词元[i+1]；在词元化后适用于所有模态 |
| IBQ词元化器 | "逆瓶颈量化器" | 一类码本更大（32768+）且重建效果优于Chameleon的VQ-VAE |
| 3D VQ | "时空量化器" | 码本按（时间，行，列）索引；一个词元覆盖一个4x4x4像素立方体 |
| 无分类器指导 | "CFG" | 使用权重gamma混合条件logits和无条件logits；在推理时提升图像质量 |
| 统一词表 | "共享词元" | 文本 + 图像 + 视频都来自同一个整数空间；模型预测接下来是哪个模态的词元 |
| MJHQ-30K | "图像生成基准" | 具有3万条提示的Midjourney质量基准；Emu3在此报告FID |
