# 变色龙与早期融合纯令牌多模态模型

> 我们迄今为止看到的每一种视觉语言模型都将图像和文本分开处理。视觉令牌来自视觉编码器，流经投影器，然后在大语言模型内部与文本相遇。视觉和文本词汇表从未重叠。变色龙（Meta，2024年5月）提出：如果它们重叠会怎样？训练一个VQ-VAE，将图像转换为来自共享词汇表的离散令牌序列。现在每个多模态文档都是一个序列——文本令牌和图像令牌交错排列，使用单一的自回归损失函数。附带好处：模型能够生成混合模态输出——在单次推理调用中交替生成文本和图像令牌。本课程将解析早期融合理论，并从零开始构建一个简化版本。

**类型：** 构建
**语言：** Python（标准库、VQ-VAE分词器 + 交错解码器）
**先修要求：** 阶段12 · 05，阶段8（生成式AI）
**时间：** 约180分钟

## 学习目标

- 解释为什么共享词汇表 + 单一损失函数会改变模型的能力。
- 描述VQ-VAE如何将图像分词为离散序列，以适应Transformer的下一个令牌预测目标。
- 说出变色龙训练稳定性的技巧：QK-Norm、Dropout放置、LayerNorm顺序。
- 比较变色龙与BLIP-2的Q-Former方法，并描述各自的适用场景。

## 问题背景

基于适配器的视觉语言模型（LLaVA、BLIP-2、Qwen-VL）将文本和图像视为两种不同的东西。文本令牌经过`embed(text_token)`；图像经过`visual_encoder(image) → projector → ... pseudo_tokens`。模型有两条输入路径，中途合并。

三个后果：

1. 大语言模型只能消费图像，而不能生成图像。输出仅为文本。
2. 混合模态文档（段落和图像交替出现，如文章）处理起来很别扭——你要么在模型外部解析多模态输入，要么链式生成。
3. 分布不匹配。视觉令牌和文本令牌存在于隐藏空间的不同区域，造成微妙的对齐问题。

变色龙否定了这个前提：图像只是来自共享词汇表的离散令牌序列。在交错文档上训练模型，使用一个损失函数、一个自回归解码器，你就能免费解锁混合模态生成。

## 核心概念

### VQ-VAE作为图像分词器

分词器是一个向量量化变分自编码器。架构如下：

- 编码器：CNN + ViT，将图像映射到空间特征图，例如32x32个维度为256的特征。
- 码本：一个由K个向量（变色龙使用8192个）组成的可学习词汇表，维度也为256。
- 量化：对于每个空间特征，通过L2距离查找最近的码本条目。用整数索引替换连续特征。
- 解码器：CNN，将量化后的特征恢复为像素。

训练：VAE重建损失 + 承诺损失 + 码本损失。码本索引构成了图像的离散字母表。

对于变色龙：一张图像变成32*32 = 1024个令牌，这些令牌来自一个包含8192个条目的词汇表。与文本令牌（来自大语言模型的BPE词汇表，假设为32000个）连接。最终词汇表大小：40192。Transformer只看到一个序列，一个损失函数。

### 共享词汇表

变色龙的词汇表结合了文本令牌、图像令牌和模态分隔符。每个令牌都有一个单一的ID。输入嵌入层将每个ID映射到一个D维隐藏向量。输出投影将隐藏向量映射回词汇表的对数概率。Softmax选择下一个令牌，无论其模态如何。

分隔符很重要：`<image>`和`</image>`标签界定了图像令牌序列。在生成时，如果模型发出`<image>`，下游软件就知道接下来的1024个令牌是VQ索引，需要发送给解码器进行像素渲染。

### 混合模态生成

推理就是在共享词汇表中进行下一个令牌预测。示例提示：“画一只猫并描述它。”变色龙发出：

```
<image> 4821 1029 2891 ... (1024 image tokens) </image>
The cat is orange, sitting on a windowsill...
```

模型自主决定顺序——它可能先生成图像再生成文本，先文本再图像，或者交替生成。相同的解码器，相同的损失函数。

与适配器视觉语言模型（生成仅限文本）相比，变色龙重新开启了关于模型输出模态的讨论。

### 训练稳定性——QK-Norm、Dropout、LayerNorm顺序

早期融合训练在大规模下不稳定。变色龙的论文记录了三个技巧：

- QK-Norm。在注意力机制内部，对查询和键投影应用LayerNorm，在点积之前。防止深层网络中对数概率幅度爆炸。被多个2024年后的大规模模型使用。
- Dropout放置。在每个残差相加之后放置Dropout，而不仅仅在注意力和MLP之后。当来自图像令牌的梯度可能占主导地位时，需要更强的正则化。
- LayerNorm顺序。在残差分支上使用Pre-LN（标准做法），并在最后一个模块的跳跃连接上额外使用一个LN。稳定最后一层的梯度流。

没有这些技巧，340亿参数的变色龙训练在多个检查点发散。有了这些技巧，它才能收敛。训练方案与架构本身一样，都是重要贡献。

### 分词器的重建上限

VQ-VAE是有损的。在8192个码本条目和每张512x512图像1024个令牌的设置下，重建PSNR上限约为26-28 dB。这对于可识别的图像生成足够，但明显差于连续空间扩散（Stable Diffusion 3能达到32+ dB）。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）能提升这个上限。Emu3（课程12.12）仅通过更好的分词器就实现了SDXL级别的生成质量。

### 变色龙 vs BLIP-2 / LLaVA

变色龙（早期融合，共享词汇表）：
- 一个损失函数，一个解码器。
- 生成混合模态输出。
- 分词器是质量上限。
- 昂贵：推理路径上每生成一张图像都需要一个VQ-VAE解码器。

BLIP-2 / LLaVA（晚期融合，独立塔）：
- 视觉输入，仅文本输出。
- 复用预训练的大语言模型。
- 理解任务无分词器瓶颈。
- 便宜：单次前向传播。

根据任务选择。如果需要图像生成，选择变色龙家族。如果只需要理解，适配器视觉语言模型更简单，并能复用更多预训练计算资源。

### Fuyu和AnyGPT

Fuyu（Adept，2023）是一种相关方法：完全跳过单独的视觉编码器，将原始图像块通过大语言模型的输入投影层，就像它们是令牌一样，没有分词器。比变色龙更简单，但失去了共享词汇表的输出生成能力。

AnyGPT（Zhan等人，2024）将变色龙扩展到四种模态：文本、图像、语音、音乐。每种模态都使用相同的VQ-VAE技巧，共享一个Transformer。任意模态到任意模态的生成。将在课程12.16中更详细地介绍。

## 动手实践

`code/main.py`构建了一个简化的端到端早期融合模型：

- 一个微型的VQ-VAE风格量化器，将8x8图像块映射到码本索引（K=16）。
- 一个共享词汇表：(文本id 0..31) + (图像id 32..47) + (分隔符48, 49)。
- 一个简化的自回归解码器（二元组表），在合成的图像描述 + 图像令牌序列上训练。
- 给定提示后，采样循环发出交替的文本 + 图像令牌。

代码故意保持Transformer很小（二元组），以便你可以端到端地追踪信号流。

## 练习

1. 变色龙使用K=8192个码本条目和每张512x512图像1024个令牌。估算相对于24位RGB图像的压缩比。它是有损的吗？损失程度如何？

2. 一张4K图像（3840x2160）在相同VQ-VAE密度下会产生多少图像令牌？一个变色龙风格的模型能在一次推理调用中生成4K图像吗？什么会先出问题——上下文、分词器质量还是KV缓存？

3. 用纯Python实现QK-Norm。给定一个64维的查询和键，展示LayerNorm前后的点积结果。为什么幅度控制在深层很重要？

4. 阅读变色龙论文第2.3节关于训练稳定性的内容。描述论文中在没有QK-Norm的34B参数模型上观察到的具体失败模式。“范数爆炸”的特征是什么？

5. 扩展简化版解码器，使其在给定纯文本提示时发出混合模态响应。根据训练数据分布（60%文本优先 / 40%图像优先），测量模型选择图像优先和文本优先的频率。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|------------|----------|
| 早期融合 | “统一令牌” | 图像从第一步就被转换为与Transformer词汇表共享的离散令牌 |
| VQ-VAE | “图像分词器” | CNN + ViT + 码本，将图像映射为Transformer可以预测的整数索引 |
| 共享词汇表 | “一本字典” | 一个覆盖文本 + 图像 + 模态分隔符的单一令牌ID空间 |
| QK-Norm | “注意力稳定器” | 在查询和键的点积之前应用LayerNorm，防止范数爆炸 |
| 混合模态生成 | “文本+图像输出” | 在单次推理中自主生成交错排列的文本和图像令牌 |
| 码本大小 | “K个条目” | VQ-VAE可以量化到的离散向量数量；在压缩率和保真度之间权衡 |
| 分词器上限 | “重建极限” | 解码VQ令牌能达到的最佳PSNR；限制了模型的图像质量 |
