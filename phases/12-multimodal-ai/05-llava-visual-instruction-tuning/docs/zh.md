# LLaVA 与视觉指令微调

> LLaVA (2023年4月) 是地球上被复制最多的多模态架构。它用2层MLP替代了BLIP-2的Q-Former，用简单的token拼接替代了Flamingo的门控交叉注意力机制，并使用由GPT-4从纯文本标题生成的158k视觉-指令对话进行训练。2023年至2026年间构建视觉语言模型(VLM)的任何实践者都构建了LLaVA的某个变体。LLaVA-1.5添加了AnyRes。LLaVA-NeXT提高了分辨率。LLaVA-OneVision在一个统一方案中整合了图像、多图像和视频。本节课将解读其方案，实现投影器，并解释“更简单的设计为何胜出”。

**类型：** 构建
**语言：** Python（标准库，投影器 + 指令模板构建器）
**先修知识：** Phase 12 · 02 (CLIP), Phase 11 (LLM工程 — 指令微调)
**时间：** 约180分钟

## 学习目标

- 构建一个2层MLP投影器，将ViT图像块嵌入（维度1024）映射到LLM的嵌入维度（维度4096）。
- 学习LLaVA的两阶段方案：(1) 在558k图像-标题对上进行投影器对齐，(2) 在158k由GPT-4生成的对话上进行视觉指令微调。
- 构建一个包含图像token占位符、系统提示以及用户/助手对话轮次的LLaVA格式提示。
- 解释为什么尽管Q-Former在token预算方面有优势，社区仍然从Q-Former转向MLP。

## 问题所在

BLIP-2的Q-Former（课程12.03）将一张图像压缩为32个token。干净、高效、对基准测试友好。但它存在两个问题。

首先，Q-Former是可训练的，但其损失并非最终任务。第一阶段训练ITC+ITM+ITG损失。第二阶段训练语言建模(LM)损失。查询学习到一些中间表示，然后LLM需要解码它。信息在瓶颈处丢失。

其次，Q-Former有1.88亿参数，在LLaVA 2023年的规模下，你必须与目标LLM协同设计它。更换LLM，需要重新训练Q-Former。更换视觉编码器，也需要重新训练。每种组合都是一个独立的研发项目。

LLaVA的答案简单得令人意外：取ViT的576个图像块token，将每个token通过一个2层MLP (`1024 → 4096 → 4096`)，然后将所有576个token直接送入LLM的输入序列。没有瓶颈。没有在奇怪目标上的第一阶段预训练。只是用直接的LM损失训练MLP。

数据从哪里来？LLaVA的第二个洞见：使用GPT-4（纯文本）生成指令数据。将图像的COCO标题和边界框数据提供给GPT-4，要求它生成对话、描述和复杂的推理问题。免费获得158k指令-响应对话对。无需人工标注。

结果：一个在8块A100上运行一天即可完成的VLM，在MMMU基准上击败了Flamingo，并发布了社区可以扩展的开源检查点。到2023年底，它已衍生出50多个分支。

## 核心概念

### 架构

130亿参数的LLaVA-1.5：
- 视觉编码器：CLIP ViT-L/14 @ 336（第一阶段冻结，第二阶段可选解冻）。
- 投影器：带GELU激活的2层MLP，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来是Llama-3.1-8B）。

在一张图像+文本提示上的前向传播：

```
img -> ViT -> 576 patches of dim 1024
patches -> MLP -> 576 tokens of dim 4096
prompt: system + "<image>" placeholder + user question
replace <image> token with the 576 projected tokens
feed the full sequence to the LLM
decode response
```

图像占据LLM上下文的576个token。在2048上下文长度下，这为文本留下1472个token。在32k上下文下，这几乎可以忽略不计。

### 第一阶段：投影器对齐

冻结ViT。冻结LLM。仅训练2层MLP。数据集：558k图像-标题对（LAION-CC-SBU）。损失：基于图像token的条件，在标题上进行语言建模。

以128的批次大小训练一个轮次，只需几小时即可完成。投影器学习将ViT空间映射到LLM空间。没有任务特定的监督。

### 第二阶段：视觉指令微调

解冻投影器（仍可训练）。解冻LLM（通常全部解冻，有时使用LoRA）。在158k视觉-指令对话上进行训练。

指令数据是诀窍。Liu等人通过以下方式生成：
1. 取一张COCO图像。
2. 提取文本描述（5个人工标题 + 边界框列表）。
3. 使用三个提示模板发送给GPT-4：
   - 对话：“生成关于这张图像的用户和助手之间的来回对话。”
   - 详细描述：“给出对这张图像丰富、详细的描述。”
   - 复杂推理：“提出一个需要推理图像的问题，然后回答它。”
4. 将GPT-4的输出解析为（指令，响应）对。

整个过程并未直接接触图像——仅使用文本描述。GPT-4会“幻觉”出合理的图像内容。存在一些噪声，但它确实有效：158k对话足以解锁对话能力。

### 社区为何复制此方案

- 无需调整第一阶段特定的损失。全程使用LM损失。
- 投影器训练只需几小时，而非几天。
- 通过仅重新训练投影器即可更换LLM（LLaVA-Llama2, LLaVA-Mistral, LLaVA-Llama3）。
- 视觉-指令数据流水线使用GPT-4，为新领域重新生成成本低廉。

### LLaVA-1.5 与 LLaVA-NeXT

LLaVA-1.5（2023年10月）增加了：
- 将学术任务数据（VQA、OKVQA、RefCOCO）混入指令微调。
- 更好的系统提示。
- 上下文长度从2048扩展到32k。

LLaVA-NeXT（2024年1月）增加了：
- AnyRes：将高分辨率图像分割成2x2或1x3的336x336网格，外加一个全局低分辨率缩略图。每个切片成为576个token；每张图像总共约2880个视觉token。OCR和图表任务性能大幅提升。
- 更好的指令数据混合，包含ShareGPT4V（高质量的GPT-4V描述）。
- 更强的基座LLM（Mistral-7B, Yi-34B）。

### LLaVA-OneVision

课程12.08将深入讲解OneVision。简而言之：相同的投影器，但使用涵盖单图像、多图像和视频的课程进行训练，共享视觉token预算。

### 与Q-Former的比较

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| 每张图像的视觉token数 | 32 | 576（基础）或2880（AnyRes） |
| 可训练参数 | 1.88亿 + LM | 4000万 + LM |
| 第一阶段损失 | ITC+ITM+ITG | 仅LM |
| LLM即插即用 | 需要重新训练 | 仅需最少重新训练即可更换 |
| 多图像 | 不自然 | 自然（拼接） |
| 视频 | 不自然 | 自然（逐帧拼接） |
| Token预算 | 小 | 大 |

MLP在简洁性和token灵活性上胜出。Q-Former在token预算上胜出。到2023年底，token预算不再是主要瓶颈（LLM上下文增长至32k-128k+），简洁性占据了主导地位。

### 提示格式

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> Describe this image in detail. ASSISTANT: The image shows ...
```

`<image>` 是一个占位符token。在分词之前，它会被替换为576个视觉token（使用AnyRes时为2880个）。分词器看到的序列比其训练时稍长，但LLM能够处理这种新输入，因为第一阶段已经教会了它。

### 参数经济性

LLaVA-1.5-7B分解：
- CLIP ViT-L/14 @ 336：3.03亿参数（第一阶段冻结，第二阶段通常解冻）。
- 投影器（2层线性层）：约2200万可训练参数。
- Llama-7B：70亿参数。
- 总计：73亿参数。第二阶段可训练：整个70亿LLM + 2200万投影器。

第二阶段的训练成本：在8块A100上约20小时。这是关键数字——一天，一个节点，可复现。这就是LLaVA得以传播的原因。

## 使用它

`code/main.py` 实现了：

1. 纯Python实现的2层MLP投影器（玩具规模下维度为16 → 32 → 32）。
2. 提示构建流水线：系统提示 + `<image>` 替换为N个投影后的token + 用户轮次 + 助手生成占位符。
3. 一个可视化工具，展示576个token的视觉块在LLM上下文中占据多少比例（相对于2k / 32k / 128k上下文）。

## 部署它

本课程产出 `outputs/skill-llava-vibes-eval.md`。给定一个LLaVA系列的检查点，它将运行一个包含10个提示的“感觉评估”套件（3个图像描述，3个视觉问答，2个推理，2个拒绝），并输出一份人类可读的记分卡。这不是一个基准测试；而是一个冒烟测试，用于确认投影器和LLM连接良好。

## 练习

1. 计算 `1024 → 4096 → 4096` 所述的2层MLP投影器的可训练参数数量。考虑到GELU和偏置，它占LLaVA-13B总参数的百分比是多少？

2. 为一个“拒绝”案例构建一个LLaVA提示——图像中包含私人个体。编写预期的助手响应。为什么LLaVA应该零样本拒绝这个请求？需要什么训练数据来强化这种拒绝？

3. 阅读LLaVA-NeXT博客中的AnyRes部分。计算一张1344x672图像在AnyRes下的视觉token数量。与基础336x336图像下的576个token进行比较。

4. LLaVA第一阶段的投影器是用标题上的LM损失训练的。如果跳过第一阶段，直接进行第二阶段（视觉指令微调）会发生什么？引用Prismatic VLMs（arXiv:2402.07865）的消融实验来回答。

5. LLaVA-Instruct-150k使用GPT-4配合COCO标题生成指令。针对一个新领域（如医学X光、卫星图像），描述生成领域指令的四步数据流水线。每个步骤可能出现什么问题？

## 关键术语

| 术语 | 人们如何称呼 | 它的实际含义 |
|------|----------------|------------------------|
| 投影器 | “MLP桥” | 带GELU激活的2层MLP，将ViT维度映射到LLM维度 |
| 图像token | “<image> 占位符” | 在推理前被N个投影后的视觉token替换的提示标记 |
| 视觉指令微调 | “LLaVA第二阶段” | 使用GPT-4生成的（图像，指令，响应）三元组进行训练 |
| 第一阶段对齐 | “投影器预训练” | 冻结ViT和LLM，用标题上的LM损失训练投影器 |
| AnyRes | “多切片平铺” | 将高分辨率图像分割成网格，并拼接每个切片的视觉token |
| LLaVA-Instruct | “GPT-4生成” | 从COCO标题+GPT-4合成的158k指令-响应对 |
| 视觉编码器冻结 | “骨干网络锁定” | 第一阶段CLIP权重不更新，第二阶段有时也不更新 |
| ShareGPT4V | “更好的标题” | 由GPT-4V生成的100万个密集标题，用于更高质量的对齐 |
| VQA | “视觉问答” | 回答关于图像的自由形式问题的任务 |
| Prismatic VLMs | “设计空间论文” | Karamcheti 2024年消融实验，系统测试投影器和数据选择 |
