# Qwen-VL 家族与动态帧率视频

> Qwen-VL 家族——Qwen-VL (2023)、Qwen2-VL (2024)、Qwen2.5-VL (2025)、Qwen3-VL (2025)——是2026年最具影响力的开源视觉语言模型谱系。每一代都做出了一个关键的架构决策，其余开源生态系统在十二个月内纷纷效仿：通过M-RoPE实现的原生动态分辨率、带有绝对时间对齐的动态帧率采样、ViT中的窗口注意力机制，以及结构化的智能体输出格式。到了Qwen3-VL，其技术方案已趋于稳定：一个采用原生纵横比输入的2D-RoPE-ViT编码器，通过MLP投影器连接到一个大型Qwen3语言模型基础，以及强调将OCR、定位和智能体行为作为首要目标的训练阶段。本课程将按时间顺序解读这个家族，以便你理解每一个关键设计背后的缘由。

**类型：** 学习
**语言：** Python (标准库, M-RoPE编码器 + 动态帧率采样器)
**前置知识：** 第12阶段 · 06 (patch-n'-pack)
**时间：** ~120分钟

## 学习目标

- 计算M-RoPE的三轴旋转（时间、高度、宽度）并解释为何三者缺一不可。
- 为视频选择一种动态帧率采样策略，并推理每秒令牌数与事件检测精度之间的关系。
- 按顺序列举Qwen-VL的四次代际升级及其各自实现的功能。
- 实现一个Qwen2.5-VL风格的JSON智能体输出格式，并从VLM响应中解析结构化的工具调用。

## 问题背景

Qwen-VL于2023年8月发布，直接回应了LLaVA-1.5和BLIP-2。Qwen团队瞄准的差距有三：分辨率、视频和结构化输出。

分辨率：LLaVA-1.5运行在336x336。对于照片尚可，但对于中文发票或密集的电子表格截图则毫无用处。Qwen-VL的首个创新是448x448分辨率并输出带定位的边界框，使模型能够指向具体事物。

视频：Video-LLaVA堆叠逐帧编码器并将其输入LLM。这对短片有效，但对于以时间轴为信号的多分钟视频则力不从心。Qwen团队希望拥有一个能理解时间的单一编码器。

结构化输出：LLaVA输出自由格式的文本。而智能体需要JSON。Qwen-VL在包含边界框坐标等文本的显式JSON输出格式上进行了训练。

Qwen-VL的每一代都在扩展这三个轴中的某一个。

## 核心概念

### Qwen-VL (2023年8月)

第一代：使用OpenCLIP ViT-bigG/14作为编码器（25亿参数），兼容LLama的Q-Former（单步，256个查询），以及Qwen-7B作为基础模型。主要贡献：

- 448x448分辨率（当时开源VLM的SOTA）。
- 定位：在带有显式坐标令牌输出的图像-文本对上进行训练。"猫在 <box>(112, 204), (280, 344)</box>"。
- 从一开始就进行中英文多语言训练。

当时的基准测试：在英文上与GPT-4V竞争力相当，在中文上占主导地位。定位监督是真正的亮点。

### Qwen2-VL (2024年9月) — M-RoPE与原生分辨率

Qwen2-VL用原生的动态分辨率ViT编码器取代了固定分辨率+Q-Former的堆叠。关键变化：

- 原生动态分辨率。ViT接受任何被28整除的HxW（14像素的patch加2倍空间合并）。1120x672的图像（40x24个合并后的patch）产生960个视觉令牌。无需调整大小、切块或缩略图。
- M-RoPE（多模态RoPE）。每个令牌携带一个3D位置 (t, h, w) 而非1D。对于图像，t=0；对于视频，t = 帧索引。RoPE根据每个轴的频率旋转查询/向量键。无需位置编码表。
- MLP投影器。弃用Q-Former；对合并后的patch令牌使用一个2层MLP。
- 支持动态帧率的视频。默认以1-2 FPS采样视频，但模型接受任意帧数。

结果：Qwen2-VL-7B在多个多模态基准测试上匹配GPT-4o，并在DocVQA上超越它（94.5 vs 88.4）。架构变革是决定性的一步。

### Qwen2.5-VL (2025年2月) — 动态帧率 + 绝对时间

Qwen2.5-VL的重大转变在于视频。动态帧率不仅仅是“需要时采样更多帧”。该论文正式定义了：

- 绝对时间令牌。不使用位置索引（第0、1、2帧...），而是使用实际时间戳。"在0:04，猫跳了起来。"模型看到 `<time>0.04</time>` 令牌与帧令牌交错。
- 动态帧率。对慢速镜头以1 FPS采样，对动作场景以4+ FPS采样。用户或训练器选择；M-RoPE自适应。
- ViT中的窗口注意力。空间注意力采用窗口机制（在块内局部进行）以提高吞吐量；每隔几层进行一次全局注意力。
- 显式的JSON输出格式。在工具调用数据上训练："{\"tool\": \"click\", \"coords\": [380, 220]}"。开箱即用的智能体就绪。
- MRoPE-v2缩放。位置随最大输入大小缩放，因此10分钟的视频不会耗尽频率范围。

基准测试：Qwen2.5-VL-72B在大多数视频基准测试上击败GPT-4o，在文档任务上匹配Gemini 2.0，并在GUI定位上创下开源模型SOTA（ScreenSpot：84%准确率 vs GPT-4o的38%）。

### Qwen3-VL (2025年11月)

Qwen3-VL是一次增量升级，旨在巩固而非重新发明：更大的LLM骨干（Qwen3-72B），扩展的训练数据，改进的OCR，以及通过Qwen3的“思考模式”增强推理能力。ViT和M-RoPE保持不变。论文重点在于数据和训练改进，而非架构。

该谱系的启示：到2025年，Qwen-VL架构已趋于稳定。后续的代际升级主要是扩大计算和数据规模，而非改变基本原理。

### M-RoPE 的数学表达

经典RoPE通过配对坐标，根据位置 `m` 旋转维度为 `d` 的查询向量 `q`：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE将隐藏维度分成三个频带。假设 `d = 96`。分配32维给时间，32维给高度，32维给宽度。每个频带根据其轴位置独立旋转。位于 (t=5, h=10, w=20) 的一个patch会对其三个频带分别应用旋转 `R_t(5)`、`R_h(10)`、`R_w(20)`。

文本令牌使用 `t = text_index, h = 0, w = 0`（或归一化的选择），保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

好处：一个位置编码无需分支代码或不同的位置表即可处理文本、图像和视频。

### 动态帧率采样逻辑

给定一个持续 `T` 秒的视频和一个目标令牌预算 `B`：

1. 计算你能够承受的最大帧率：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中选择一个满足 `fps <= fps_max` 的目标帧率。
3. 如果运动剧烈（使用光流启发式算法或用户明确请求），则选择较高的帧率。如果运动平缓，则选择较低的帧率。
4. 以选定的帧率均匀采样；在帧之间插入 `<time>t</time>` 令牌。

Qwen2.5-VL隐式地训练了此逻辑；在推理时，用户通过 `fps` 参数进行控制。一个60秒的动作序列以4 FPS采样，每帧81个令牌 = 19440个令牌，在32k上下文中可控。

### 结构化智能体输出

Qwen2.5-VL的智能体训练显式地针对结构化的工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型的输出进行JSON.parse。与自由格式的“点击在 (1024, 512)”（需要正则表达式和歧义处理）相比，这是一个转变。这也是Qwen2.5-VL在ScreenSpot上的得分从Qwen2-VL的55%跃升至84%的原因。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| M-RoPE | "多模态RoPE" | 3D旋转位置编码，在隐藏维度中包含时间、高度和宽度频带 |
| 动态帧率 | "智能采样" | 基于视频的运动、时长和令牌预算选择的帧采样率 |
| 绝对时间令牌 | "时间戳令牌" | `<time>t</time>` 交错于序列中，使模型看到实际秒数而非帧索引 |
| 窗口注意力 | "局部注意力" | 空间自注意力仅限于小窗口内以提高速度；定期添加全局注意力 |
| 结构化智能体输出 | "JSON模式" | 训练数据监督，教会VLM输出包含坐标和工具名的可解析JSON |
| min_pixels / max_pixels | "分辨率范围" | 每次请求时Qwen2.5-VL控制总像素数，从而控制令牌数的上下界 |
| 定位 | "指向它" | 将边界框坐标作为文本令牌输出；自Qwen-VL v1起使用 |
