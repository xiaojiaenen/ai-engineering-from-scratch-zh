# 开放权重VLM配方：什么真正重要

> 2024-2026年的开放权重VLM文献是一片消融实验表的森林。苹果的MM1测试了图像编码器、连接器和数据混合的13种组合。Allen AI的Molmo证明了详细的人工标注描述优于GPT-4V蒸馏。Cambrian-1进行了20多种编码器比较。Idefics2形式化了五维设计空间。Prismatic VLMs在受控基准上比较了27种训练配方。在所有这些噪音中，有一小部分结果在各论文中保持一致：**图像编码器比连接器架构更重要，数据混合比两者都重要，而详细的人工标注描述优于蒸馏的合成数据**。本课程解读那些表格，这样你就不必亲自研究它们了。

**类型：** 学习 + 实验
**语言：** Python (标准库，消融实验表解析器 + 配方选择器)
**先决条件：** Phase 12 · 05 (LLaVA基线)
**时间：** 约180分钟

## 学习目标

- 命名VLM设计的五个维度：图像编码器、连接器、LLM、数据混合、分辨率调度。
- 阅读MM1 / Idefics2 / Cambrian-1的消融实验表，并预测哪个调节项会影响特定基准。
- 根据给定的计算预算和任务组合，为新VLM选择一个配方（编码器、连接器、数据、分辨率）。
- 解释为什么在相同token数量下，详细的人工标注描述优于GPT-4V蒸馏。

## 问题

存在数百个开放权重VLM。“良好”与“最先进”之间的大部分差距不在架构，而在于数据、分辨率调度和编码器选择。当你的模型表现不佳时，知道先调节哪个调节项，可以为你节省一个500万GPU小时的错误。

2023年的一波（LLaVA-1.5, InstructBLIP, MiniGPT-4）运行于字幕对预训练 + LLaVA-Instruct-150k。良好的基线。在MMMU上大约达到35%。

2024年的一波（MM1, Idefics2, Molmo, Cambrian-1, Prismatic VLMs）进行了详尽的消融实验。结果令人惊讶且具有实践性。

## 概念

### 五维设计空间

Idefics2 (Laurençon 等人, 2024) 命名了这些维度：

1. **图像编码器**。CLIP ViT-L/14, SigLIP SO400m/14, DINOv2 ViT-g/14, InternViT-6B。编码器在patch大小、分辨率和预训练目标上有所不同。
2. **连接器**。MLP (2-4层), Q-Former (32个查询 + 交叉注意力), Perceiver Resampler (64个查询), C-Abstractor (卷积 + 双线性池化)。
3. **语言模型**。Llama-3 8B / 70B, Mistral 7B, Phi-3, Gemma-2, Qwen2.5。LLM大小是主要的参数成本。
4. **训练数据**。字幕对 (CC3M, LAION), 交错数据 (OBELICS, MMC4), 指令数据 (LLaVA-Instruct, ShareGPT4V, PixMo, Cauldron)。
5. **分辨率调度**。固定的224/336/448, AnyRes, 原生动态分辨率。在训练期间递增或保持恒定。

每个生产环境的VLM都在每个维度上做出选择。MMMU分数中的大部分方差由维度1、4和5解释——而不是由你选择的连接器决定。

### 维度1：编码器 > 连接器

MM1第3.2节显示：从CLIP ViT-L/14切换到SigLIP SO400m/14，在MMMU上增加了3分以上。将连接器从MLP切换到Perceiver Resampler，增加了不到1分。Idefics2复制了这一结果：SigLIP > CLIP, 在相同的token数量下，Q-Former ≈ MLP ≈ Perceiver。

Cambrian-1的“Cambrian视觉编码器对决”（Tong 等人, 2024）在以视觉为中心的基准（CV-Bench）上测试了20多种编码器。排行榜顶端是DINOv2和SigLIP的混合；CLIP处于中游；ImageBind和ViT-MAE较低。从CLIP ViT-L到DINOv2 ViT-g/14的差距在CV-Bench上大约是5-7分。

2026年开放VLM的默认编码器是用于语义+密集特征的SigLIP 2 SO400m/14，有时会与DINOv2 ViT-g/14特征连接（Cambrian的“空间视觉聚合器”就这样做）。

### 维度2：连接器设计差异不大

MM1、Idefics2、Prismatic和MM-Interleaved都得出了相同的结论：在固定的视觉token数量下，连接器架构几乎不重要。一个2层MLP在平均池化的patch上，在相同的token预算下，性能与一个32查询的Q-Former相差不超过1分。

真正重要的是**token数量**。更多视觉token = 更多LLM计算 = 性能更好，直到某个点，然后收益递减。每张图像64个token对于OCR来说太少了。576-1024个token是大多数开放VLM的最佳选择。2048个以上仅对文档和图表有帮助。

Q-Former与MLP是一个成本问题，而非质量问题：Q-Former将token数量限制在32-64个，无论图像分辨率如何；MLP输出所有patch token。对于高分辨率输入，Q-Former节省LLM上下文；对于低分辨率，差异可以忽略不计。

### 维度3：LLM大小决定了上限

将LLM从7B加倍到13B，在每篇VLM论文中可靠地为MMMU增加2-4分。在70B时，你已经饱和了大多数基准。VLM的多模态推理上限是LLM的文本推理上限——视觉编码器只能为其提供信息，而不能替代其进行推理。

这就是为什么Qwen2.5-VL-72B和Claude Opus 4.7在MMMU-Pro和ScreenSpot-Pro上表现出色：语言大脑非常强大。一个7B VLM无法通过巧妙的连接器设计来替代70B VLM。

### 维度4：数据——详细的人工标注描述优于蒸馏

Molmo + PixMo (Deitke 等人, 2024) 是2024年所有人都应该阅读的研究结果。Allen AI让人类标注员用1-3分钟的密集语音转文字过程来描述图像，产生了712K张密集标注的图像。训练数据中完全没有GPT-4V蒸馏。

Molmo-72B在11个基准中的11个上击败了Llama-3.2-90B-Vision。差距不在于架构——而在于**描述质量**。详细的人工标注描述每张图像包含的信息量是简短网络描述的5-10倍，并且在GPT-4V蒸馏产生幻觉的地方保持事实准确。

ShareGPT4V (Chen 等人, 2023) 和 Cauldron (Idefics2) 采用了相同的策略，混合使用人工 + GPT-4V描述。趋势很明确：对于2026年的前沿，**描述密度 > 描述数量 > 蒸馏的便利性**。

### 维度5：分辨率及其调度

Idefics2的消融实验：384 -> 448增加1-2分。448 -> 980 带有图像分割（AnyRes）在OCR基准上再增加3-5分。固定分辨率训练在中等准确率上达到平台期；分辨率递增（从224开始，结束于448或原生分辨率）训练更快，最终结果更高。

Cambrian-1进行了分辨率与token数量的权衡：在固定计算量下，你可以在较低分辨率下拥有更多token，或在较高分辨率下拥有更少的token。对于OCR，更高分辨率胜出；对于一般场景理解，更低分辨率+更多token胜出。

2026年的生产配方：第1阶段在384固定分辨率下训练，第2阶段使用最高1280的动态分辨率，用于OCR密集型任务。

### Prismatic受控比较

Prismatic VLMs (Karamcheti 等人, 2024) 是一篇控制了所有维度的论文。相同的13B LLM，相同的指令数据，相同的评估——每次只变化一个维度。结果：

- 每张图像的视觉token数量解释了约60%的方差。
- 编码器选择解释了约20%。
- 连接器架构解释了约5%。
- 其他所有（数据混合、调度器、学习率）占剩余约15%。

这是一个粗略的分解，但它是文献中对于“我应该先消融什么”这个问题最清晰的答案。

### 2026年选择器

根据证据，2026年新项目的默认开放VLM配方：

- **编码器**：SigLIP 2 SO400m/14，原生分辨率并使用NaFlex，如果需要分割/定位，则与DINOv2 ViT-g/14特征连接。
- **连接器**：在patch token上使用2层MLP。除非你受token数量限制，否则跳过Q-Former。
- **LLM**：Qwen2.5 / Llama-3.1 / Gemma 2，7B用于成本考虑，70B用于质量考虑，根据目标延迟选择。
- **数据**：PixMo + ShareGPT4V + Cauldron，用任务特定的指令数据补充。
- **分辨率**：动态（长边最小256像素，最大1280像素）。
- **调度**：第1阶段对齐（仅投影器），第2阶段完全微调，第3阶段任务特定微调。

每一个默认选择都可以追溯到本课程末尾引用的论文中进行的测量消融实验。

## 使用它

`code/main.py` 是一个消融实验表解析器和配方选择器。它编码了（精简的）MM1和Idefics2消融实验表，并允许你查询：

- “给定预算X和任务Y，哪个配方胜出？”
- “如果我将7B Llama上的SigLIP换成CLIP，预期的MMMU差异是多少？”
- “为了得到80%置信度的答案，我应该先消融哪个维度？”

输出是一个带有预期基准差异和“首先消融”建议的配方排名列表。

## 交付它

本课程产生 `outputs/skill-vlm-recipe-picker.md`。给定目标任务组合、计算预算和延迟目标，它会输出一个完整的配方（编码器、连接器、LLM、数据混合、分辨率调度），并引用证明每个选择的消融实验。这可以防止工程师在每次启动新VLM项目时重新发明Idefics2消融实验表。

## 练习

1.  阅读MM1第3.2节。对于在预算5000万张图像下的固定2B LLM，哪个编码器胜出？如果换成13B LLM，答案会改变吗？为什么？
2.  Cambrian-1发现，在以视觉为中心的基准上，连接DINOv2 + SigLIP优于单独使用任何一个，但在MMMU上没有增加任何信号。预测哪些基准会有所提升，哪些会保持平稳。
3.  你的目标是在2B LLM上构建一个移动UI智能体。选择编码器、连接器、分辨率和数据混合。用一个具体的消融实验表证明每个选择。
4.  Molmo发布了4B和72B模型。4B模型与闭源的7B VLM具有竞争力；72B模型在11/11个基准上击败了Llama-3.2-90B-Vision。这告诉你关于LLM大小平台期假说的什么信息？
5.  设计一个消融实验表，以在7B VLM上分离数据混合质量与编码器质量。最少需要多少次训练运行？提出四个维度的设置。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------|----------|
| **消融实验** | “调节一个旋钮” | 训练多个运行，它们在设计空间的一个维度上精确不同，其他所有保持不变 |
| **连接器** | “桥梁” / “投影器” | 可训练模块，将视觉编码器输出映射到LLM的token空间（MLP, Q-Former, Perceiver） |
| **详细人工标注描述** | “密集描述” | 人工编写的多句话描述（通常80-300个token），比网络alt文本更丰富 |
| **蒸馏** | “GPT-4V描述” | 由更强大的专有VLM生成的训练数据；方便但容易继承幻觉 |
| **AnyRes / 动态分辨率** | “高分辨率路径” | 通过分块或M-RoPE馈送大于编码器原生分辨率图像的策略 |
| **分辨率递增** | “课程学习” | 从低分辨率开始并逐渐增加的训练计划，加速对齐学习 |
| **以视觉为中心的基准** | “CV-Bench / BLINK” | 评估重点在细粒度视觉感知而非语言推理 |
| **PixMo** | “Molmo的数据” | Allen AI的712K密集标注图像数据集；人工语音转录为密集描述 |
