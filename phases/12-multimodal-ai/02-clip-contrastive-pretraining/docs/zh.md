# CLIP与对比式视觉-语言预训练

> OpenAI的CLIP（2021）证明了一个足以驱动未来五年的核心思想：仅使用嘈杂的网络图像-描述对和对比损失，将图像编码器和文本编码器对齐到同一向量空间。无需监督标签。4亿对样本。由此产生的嵌入空间支持零样本分类、图像-文本检索，并作为视觉塔集成到2026年所有的VLM中。SigLIP 2（2025）用sigmoid替代softmax，以更低成本实现了超越CLIP的规模。本课将从InfoNCE到sigmoid逐对损失讲解数学原理，并使用标准库Python构建训练步骤。

**类型：** 实践构建
**语言：** Python（标准库，InfoNCE + sigmoid损失实现）
**先修课程：** 第12阶段·01（ViT分块），第7阶段（Transformer）
**时间：** 约180分钟

## 学习目标

- 从互信息推导InfoNCE损失，并实现数值稳定的向量化版本。
- 解释为何sigmoid逐对损失（SigLIP）能扩展到32768+批次，而无需softmax所需的all-gather通信开销。
- 通过构建文本模板（`a photo of a {class}`）并对余弦相似度取argmax，运行零样本ImageNet分类。
- 列举CLIP/SigLIP预训练提供的四个关键杠杆：批大小、温度参数、提示模板、数据质量。

## 问题背景

CLIP之前的计算机视觉依赖监督学习。收集带标签数据集（ImageNet：120万张图像，1000个类别），训练CNN，部署上线。标签获取成本高昂，标签内容受限于标注者的共识范围，且标签无法在不微调的情况下迁移到新任务。

网络上的图像-描述对有数十亿对免费提供，且标注较宽松。一张金毛犬的图片附带替代文本“我在公园的狗狗Max”，这本身就是一个监督信号——文本描述了图像内容。问题在于：能否将其转化为有用的训练数据？

CLIP的解决方案：将图像-描述对视为匹配任务。给定一批N张图像和N段描述，学习将每张图像与其自身描述匹配，以对抗N-1个干扰项。监督信号是“这两者属于一起；其余N-1个不属于”。无需类别标签。无需人工标注。仅需对比损失。

由此产生的嵌入空间能实现的远超CLIP训练目标。ImageNet零样本分类之所以可行，是因为“一张猫的照片”的嵌入位置靠近那些从未被明确标注为“猫”的猫图片。这个假设催生了2026年所有的VLM。

## 核心概念

### 双编码器

CLIP包含两个塔：

- 图像编码器`f`：ViT或ResNet，为每张图像输出一个D维向量。
- 文本编码器`g`：小型Transformer，为每段描述输出一个D维向量。

两个塔都会将输出归一化为单位长度。相似度为`cos(f(x), g(y)) = f(x)^T g(y)`，因为两者均为单位范数。

对于一批N个（图像，描述）对，构建相似度矩阵`S`，形状为`(N, N)`：

```
S[i, j] = cos(f(x_i), g(y_j)) / tau
```

其中`tau`是一个可学习的温度参数（CLIP初始化为0.07；在log空间中学习）。

### InfoNCE损失

CLIP使用一种对称的交叉熵，分别沿行和列计算：

```
loss_i2t = CE(S, labels=identity)     # each image's positive is its own caption
loss_t2i = CE(S^T, labels=identity)   # each caption's positive is its own image
loss = (loss_i2t + loss_t2i) / 2
```

这就是InfoNCE。交叉熵中的softmax迫使每张图像与其自身描述的匹配度高于批次内所有其他描述。“负样本”是批次内所有其他项。批大小越大 = 负样本越多 = 信号越强。CLIP以32k的批大小进行训练；规模至关重要。

### 温度参数

`tau`控制softmax的尖锐程度。低τ → 尖锐分布，难负样本挖掘效果。高τ → 平缓分布，所有样本均有贡献。CLIP学习log(1/τ)，并裁剪以防止坍塌。SigLIP 2固定了初始τ，转而使用可学习的偏置。

### 为何sigmoid更易扩展（SigLIP）

softmax需要整个相似度矩阵同步计算。在分布式训练中，必须将所有嵌入all-gather到每个副本，然后执行softmax。通信成本随节点数量呈平方增长。

SigLIP用逐元素的sigmoid替代softmax：对于每个对`(i, j)`，损失是二分类问题——“这对是否匹配？”。正类标签是矩阵对角线元素，其余均为负类。损失函数为：

```
L = -1/N sum over (i, j) [ y_ij log sigmoid(S[i,j]) + (1-y_ij) log sigmoid(-S[i,j]) ]
```

若`i == j`则`y_ij = 1`，否则为0。每个对的损失相互独立。无需all-gather。每个GPU计算其本地分块并求和。SigLIP 2能以低成本扩展到32k-512k的批大小，而CLIP则需要成比例增加通信量。

### 零样本分类

给定N个类别名称，为每个类别构建文本模板：

```
"a photo of a {class}"
```

用文本编码器嵌入每个模板。用图像编码器嵌入图像。对余弦相似度取argmax即为预测类别。无需在目标类别上训练。

提示模板很重要。CLIP原始论文为每个类别使用了80个模板（纯文本、艺术风格、照片、绘画等）并对嵌入取平均。ImageNet准确率提升3个点。现代用法通常选取一两个模板。

### 线性探测与微调

零样本是基线。线性探测（在冻结的CLIP特征之上为目标类别训练一个线性层）在领域内任务上优于零样本。完全微调在领域内优于线性探测，但可能损害零样本迁移能力。三种模式对应三种权衡。

### SigLIP 2：NaFlex与密集特征

SigLIP 2（2025）新增：
- NaFlex：单一模型处理可变宽高比和分辨率。
- 更好的密集特征，用于分割和深度估计，旨在作为VLM中的冻结骨干。
- 多语言支持：在100+种语言上训练，而CLIP仅支持英语。
- 10亿参数规模，而CLIP最高为4亿。

在2026年的开源VLM中，SigLIP 2 SO400m/14是默认的视觉塔。在纯图像-文本检索任务中，当特定的LAION-2B训练分布与你的查询模式匹配时，CLIP仍是默认选择。

### ALIGN、BASIC、OpenCLIP、EVA-CLIP

ALIGN（Google，2021）：与CLIP思想相同，18亿对样本规模，90%嘈杂数据。证明了嘈杂数据的扩展性。OpenCLIP（LAION）：在LAION-400M/2B上对CLIP的开源复现，多种规模，首选的开源检查点。EVA-CLIP：从掩码图像建模初始化；VLM的强大骨干。BASIC：Google的CLIP+ALIGN混合体。同属一个家族，只是数据和调优策略不同。

### 零样本天花板

CLIP系列模型在ImageNet零样本准确率上上限约为76%（CLIP-G，OpenCLIP-G）。超越此限要么需要更大规模的数据（SigLIP 2达到80%+），要么需要架构变化（监督头、更多参数）。该基准正在饱和；真正的价值在于下游VLM消费的嵌入空间。

## 关键术语

| 术语 | 人们如何说 | 实际含义 |
|------|------------|----------|
| InfoNCE | “对比损失” | 基于批次相似度矩阵的交叉熵；每个样本的正样本是其配对样本，负样本是其他所有样本 |
| Sigmoid损失 | “SigLIP损失” | 逐对二元交叉熵；无softmax，无all-gather，在分布式训练中扩展成本低 |
| 温度参数 | “tau” | 在softmax/sigmoid前缩放logits的标量；控制分布的尖锐程度 |
| 零样本 | “无微调分类” | 使用文本提示构建类别嵌入，通过余弦相似度分类；不在目标类别上训练 |
| 提示模板 | “一张...的照片” | 围绕类别名称的文本支架；影响零样本准确率1-5个点 |
| 双编码器 | “双塔” | 一个图像编码器 + 一个文本编码器，输出在共享的D维空间中 |
| 难负样本 | “困难干扰项” | 与正样本相似度足够高的负样本，模型必须努力将其区分开 |
| 线性探测 | “冻结+一层” | 仅在冻结的特征之上训练线性分类器；衡量特征质量 |
| NaFlex | “原生灵活分辨率” | SigLIP 2在不调整大小的情况下，以任意宽高比和分辨率输入图像的能力 |
| 温度缩放 | “log参数化tau” | CLIP将`log(1/tau)`参数化以使梯度表现良好；裁剪以防止τ坍塌至接近零 |
