# 多模态RAG与跨模态检索

> 视觉原生文档RAG只是冰山一角。生产环境的多模态RAG范围更广——它需要跨越文本、图像、音频和视频进行检索，以支持多种工作流，例如旅行规划（“帮我找一家安静、供应素食早午餐且有自然光的地方”）、医疗分诊（“什么损伤与这张照片和这些笔记相匹配”）、电商（“找出与这张自拍相似、且有我尺码的套装”）以及现场服务（“诊断这个引擎声音并结合零件照片”）。2025年的三项调研——Abootorabi等人、Mei等人、Zhao等人——明确了子问题：跨模态检索、检索融合、生成接地、多模态评估。本课程解读这些调研并设计一个生产级管道。

**类型:** 构建
**语言:** Python（标准库、带融合与接地生成器的跨模态检索器）
**先修课程:** 第12阶段 · 23（ColPali），第11阶段（RAG基础）
**时间:** ~180分钟

## 学习目标

- 设计跨模态检索：文本→图像，图像→文本，音频→视频等。
- 比较三种融合策略：分数融合、基于注意力的融合、MoE（专家混合）融合。
- 解释生成接地：当信息源是混合模态时，“引用你的来源”是什么样的。
- 列出2025年三份权威的多模态RAG调研及其子问题分类法。

## 问题描述

单模态RAG是一个已解决的模式：嵌入查询、嵌入文本块、检索、填入LLM。多模态RAG需要：

1.  多个检索头（每种模态都需要在兼容空间中的嵌入）。
2.  跨模态检索结果的融合。
3.  引用跨模态来源的生成接地。
4.  覆盖跨模态信号的评估指标。

2025年的调研都得出了相同的分类法。

## 核心概念

### 跨模态检索

给定模态A的查询，检索模态B的文档。三种模式：

1.  **共享嵌入空间**。CLIP和CLAP在共享空间中生成文本+图像/文本+音频的嵌入。直接使用跨模态余弦相似度。局限于CLIP训练过的配对。
2.  **每模态编码器 + 转换器**。文本编码器 + 图像编码器 + 一个在空间之间映射的小型转换模块。例如Gupta等人的Sen2Sen以及其他2024年的设计。灵活但增加了复杂性。
3.  **VLM作为编码器**。使用VLM的隐藏状态作为检索表示。VLM支持的任何模态都可使用。质量更高，成本也更高。

选择：对于文本+图像使用CLIP/SigLIP 2；对于文本+音频使用CLAP；对于前沿质量的跨模态检索使用VLM隐藏状态。

### 融合策略

你检索到了10个结果：5张图像、3段文本、2个音频片段。如何合并？

**分数融合（成本最低）**。每种模态有自己的检索器，每个检索器返回分数。在模态内部归一化分数然后求和。简单，通常有效。

**基于注意力的融合**。拼接所有检索到的项目，让一个小型注意力网络对它们进行加权。需要训练。

**MoE融合**。门控网络路由到针对特定模态的专家。不同类型的查询路由方式不同——视觉问题会给予图像更高的权重。

生产默认方案：使用分数融合，并对查询的主要模态稍作偏向。如果A/B测试在你的领域显示明显优势，则升级到MoE。

### 生成接地

LLM应该引用哪些检索到的项目驱动了其每项主张。对于多模态：

- 文本来源：标准引用 `[1]`。
- 图像来源：`[img 3]` 加简短描述。
- 音频来源：`[audio 2 at 0:34]`。

使用带有接地感知的数据训练生成器：训练目标中的每个主张都标记有来源索引。在推理时，模型会自然地输出引用。

### 2025年调研

**Abootorabi等人 (arXiv:2502.08826, "Ask in Any Modality")**：多模态RAG的分类法。涵盖检索、融合、生成。覆盖最广。

**Mei等人 (arXiv:2504.08748, "A Survey of Multimodal RAG")**：聚焦于子任务基准和失败模式。对评估设计有用。

**Zhao等人 (arXiv:2503.18016)**：以视觉为重点的调研。在ColPali系列工作方面很强。

阅读这三份调研，你就能掌握截至2025年春季的最新技术状态。大多数子问题仍然是开放的。

### MuRAG — 奠基性论文

MuRAG (Chen等人, 2022) 是第一个多模态RAG。它从多模态知识库中检索图像+文本，生成答案。在VLM浪潮之前展示了可行性。现代系统（REACT, VisRAG, M3DocRAG）建立在其基础之上。

### 一个生产级旅行规划器示例

查询：“帮我找一家安静、供应素食早午餐且有自然光的地方。”

管道：

1.  分解查询。“安静” → 音频/评论关键词；“素食早午餐” → 菜单项；“自然光” → 图像特征。
2.  按模态检索：
    -   在评论上进行文本检索：“素食早午餐，安静氛围。”
    -   在餐厅照片上进行图像检索：“自然光，通风。”
    -   在环境声音片段上进行音频检索：“低分贝，无音乐。”
3.  融合分数。每家餐厅都有一个综合分数。
4.  前k家餐厅 → 带有所有证据的VLM生成器 → 生成带有引用的答案。

这远远超出了文本RAG。每种模态都增添了仅靠文本会遗漏的信号。

### 智能体式多模态RAG

**多跳**：如果第一次检索没有返回高置信度的答案，LLM会重新制定查询并再次检索。第14阶段的智能体RAG模式在此适用。示例：

- 检索初始top-10 → LLM提出“太吵了，过滤低于40分贝的” → 重新检索。
- 检索图像 → LLM看到其中一张有菜单 → 检索菜单文本 → 生成答案。

增加了复杂性，但能处理单次检索无法应对的查询。

### 评估

跨模态评估仍不成熟。常用代理指标：

- 每模态的Recall@k。
- 融合top-k准确率。
- 人工评判的端到端满意度。
- 任务特定指标（完成的预订、产生的购买）。

尚无标准基准覆盖所有模态。大多数论文在特定领域的任务上进行评估。

## 实现它

`code/main.py`：

- 三个模拟检索器（文本、图像、音频），在共享的餐厅语料库上运行。
- 分数融合，以可配置的权重组合各模态分数。
- 一个生成器存根，输出带有引用的最终答案。
- 一个简单的智能体循环，在置信度低时重新制定查询。

## 练习

1.  提出一个医疗分诊多模态RAG：查询=损伤照片 + 文本症状。哪些模态从哪些知识库检索？
2.  分数融合是简单的加权求和。它有什么失败模式是MoE融合可以避免的？
3.  阅读Abootorabi等人分类法（第3节）。三个规范的子问题是什么？它们如何映射到你选择的产品？
4.  为旅行规划器多模态RAG设计一个评估规格。哪些指标覆盖图像召回率、音频召回率和综合正确性？
5.  智能体式多跳RAG每轮都有延迟税。在什么查询难度下，准确率的提升才能证明延迟是合理的？

## 关键术语

| 术语 | 人们如何说 | 其实际含义 |
|------|------------|------------|
| 跨模态检索 | "查询一种模态，检索另一种" | 文本查询检索图像；图像查询检索文本；需要共享空间或转换器 |
| 分数融合 | "合并分数" | 各模态检索分数的加权求和；最简单的融合方式 |
| MoE融合 | "模态路由的专家" | 门控网络为每个查询选择信任哪个模态的分数 |
| 接地生成 | "引用你的来源" | 答案中的每项主张都标记有来源索引 |
| MuRAG | "第一个多模态RAG" | 2022年确立多模态RAG模式的论文 |
| 智能体式多跳 | "重新制定并重试" | 当首次通过置信度低时，LLM重新查询检索器 |
