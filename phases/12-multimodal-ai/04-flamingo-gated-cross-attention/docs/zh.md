# Flamingo与门控交叉注意力：少样本视觉语言模型

> DeepMind的Flamingo（2022）率先做到了两件事。它展示了单个模型能够处理任意交织的图像、视频和文本序列。它还展示了视觉语言模型能够进行上下文学习——只需一个包含三个示例（图像，描述）对的少样本提示，模型就能描述新图像，而无需任何梯度更新。其机制是：门控交叉注意力层，被插入到冻结的LLM的现有层之间，并配有一个可学习的tanh门，该门初始化为零，从而在初始化时保留了LLM的文本能力。本课程将深入讲解Flamingo的感知器重采样器和门控交叉注意力架构——这是Gemini的交织输入和Idefics2的视觉词元的鼻祖。

**类型：** 学习
**语言：** Python（标准库，门控交叉注意力 + 感知器重采样器演示）
**先修课程：** 阶段 12 · 03 (BLIP-2 Q-Former)
**时间：** ~120 分钟

## 学习目标

- 解释门控交叉注意力如何通过tanh(gate) = 0在初始化时保留冻结LLM的文本能力。
- 逐步讲解感知器重采样器：通过交叉注意力将N个图像块转换为K个固定的“潜在”查询。
- 描述Flamingo如何通过遵循图像位置关系的因果掩码来处理交织的图像-文本序列。
- 复现少样本多模态提示结构（3个图像-描述示例后跟一个查询图像）。

## 问题背景

BLIP-2将32个视觉词元输入到冻结的LLM的输入层。对于每个提示只处理一张图像是可行的。但如果你想提供*多张*图像并与文本交织，例如“这是图像A，描述它；这是图像B，描述它；现在这是图像C，描述它”，该怎么办？LLM的自注意力需要在单个序列中处理图像词元和文本词元，而关于哪些位置可以关注哪些图像的问题会变得很棘手。

Flamingo的答案是：完全不改变LLM的输入序列。在现有的LLM模块之间插入额外的交叉注意力层。文本词元仍然像往常一样流经LLM的因果自注意力。在每几个LLM模块之间，文本词元还会通过一个新的门控层交叉关注图像特征。该门（初始化为零）意味着在步骤零时，新层相当于空操作——模型的行为与预训练的LLM完全相同。随着训练的进行，门会打开，视觉信息开始流动。

Flamingo回答的第二个问题是：如何处理每个提示中可变数量的图像（0、1或许多张）？通过感知器重采样器——一个小型的交叉注意力模块，它接收任意数量的块，并产生固定数量的视觉潜在词元。无论提示中有多少图像，LLM的交叉注意力层看到的形状都相同。

## 核心概念

### 冻结的LLM

Flamingo从一个冻结的Chinchilla 70B LLM开始。所有700亿参数的权重都保持不动。现有的文本自注意力和FFN正常运行。

### 感知器重采样器

对于提示中的每张图像，ViT会产生N个块词元。感知器重采样器具有K个固定的可学习潜在向量（Flamingo使用K=64）。每个重采样器模块包含两个子步骤：

1. 交叉注意力：K个潜在向量关注N个块词元（查询来自潜在向量，键/值来自图像块）。
2. 潜在向量之间的自注意力 + FFN。

经过6个重采样器模块后，输出是K=64个维度为1024的视觉词元，无论ViT产生了多少个块。一张224x224的图像（196个块）和一张480x480的图像（900个块）都会输出64个重采样器词元。

对于视频，重采样器在时间维度上应用：每一帧的块产生64个潜在向量，通过时间位置编码让模型区分t=0和t=N。完整的视频变为T * 64个视觉词元。

### 门控交叉注意力

在冻结LLM的每M层之间（Flamingo使用M=4），插入一个新的门控交叉注意力模块：

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha` 是一个可学习的标量，初始化为零。
- `tanh(0) = 0`，因此在初始化时，门控分支贡献为零。
- 随着 `alpha` 偏离零，交叉注意力的贡献会平滑增长。
- 残差连接意味着即使门完全打开，也不会覆盖LLM的文本表示；它只是在原有基础上添加视觉信息。

这是Flamingo中最重要的设计选择：视觉条件是相加的、有门控的，并且在初始化时为零。在步骤0时的Flamingo模型在纯文本输入上表现得就是完美的Chinchilla 70B。

### 用于交织输入的掩码交叉注意力

在诸如“<图像A> 描述A <图像B> 描述B <图像C> ?”这样的提示中，每个文本词元只能看到序列中在它之前出现的图像。交叉注意力掩码强制规定：位置`t`的文本词元只能关注图像索引`i < i_t`的图像重采样器词元，其中`i_t`是位置`t`之前最近的一张图像。“只看到前一张图像”或“看到所有之前的图像”都是有效的选择；Flamingo选择了前者。

### 上下文少样本学习

一个Flamingo的提示看起来像这样：

```
<image1> A photo of a cat. <image2> A photo of a dog. <image3> A photo of a
```

模型看到补全模式，并输出“鸟”（或任何图像3显示的内容）。无需梯度更新。冻结LLM的上下文学习能力通过门控交叉注意力得以延续——这是该论文的核心要点及其重要性的原因。

### 训练数据

Flamingo在三个数据集上进行训练：

1.  MultiModal MassiveWeb (M3W)：4300万个包含交织图像和文本的网页，重建阅读顺序。
2.  Image-Text Pairs (ALIGN + LTIP)：44亿个图文对。
3.  Video-Text Pairs (VTP)：2700万个短视频片段。

OBELICS（2023）是交织网络语料库的开放复现版本，Idefics、Idefics2以及大多数开放的“类Flamingo”模型都在其上进行训练。

### OpenFlamingo 与 Otter

OpenFlamingo（2023）是开放复现版本。架构相同（感知器重采样器 + 门控交叉注意力，置于冻结的LLaMA或MPT之上）。提供3B、4B、9B的检查点。由于基础LLM较小且数据较少，质量落后于Flamingo。

Otter（2023）在OpenFlamingo基础上，使用MIMIC-IT（一个多模态指令数据集）进行指令微调，表明门控交叉注意力也适用于指令遵循。

### 后续发展

- Idefics / Idefics2 / Idefics3：Hugging Face的门控交叉注意力谱系，逐步简化（Idefics2放弃了重采样器，转而采用带有自适应池化的直接块词元）。
- 从Flamingo到Chameleon的转变：到2024年，许多团队转向早期融合（课程12.11）；Flamingo风格的门控交叉注意力在需要冻结主干模型的场景中仍在生产中使用。
- Gemini的交织输入：概念上继承了Flamingo的交织格式灵活性，但具体机制是专有的。

### 与BLIP-2的对比

| | BLIP-2 | Flamingo |
|---|---|---|
| 视觉桥梁 | 在输入处一次性使用Q-Former | 在每M层使用门控交叉注意力 |
| 视觉词元 | 每张图像32个 | 每张图像每个交叉注意力层64个 |
| 冻结LLM | 是 | 是 |
| 少样本上下文学习 | 较弱 | 强劲——论文的核心 |
| 交织输入 | 无原生支持 | 有，是设计目标 |
| 训练数据 | 1.3亿对 | 13亿对 + 4300万交织网页 |
| 参数量 | 训练了1.88亿参数 | 训练了~100亿参数（交叉注意力层） |
| 计算量 | 在8块A100上训练数天 | 在数千块TPUv4上训练数周 |

选择BLIP-2用于预算内的单图像视觉问答。选择Flamingo/Idefics2用于交织、少样本或多图像推理。

## 动手实践

`code/main.py` 演示了：

1.  对36个假块词元和8个可学习潜在向量应用感知器重采样器（纯Python交叉注意力）。
2.  一个门控交叉注意力步骤，当`alpha = 0`时→输出等于输入（LLM未改变），然后当`alpha = 2.0`时→视觉贡献被混合进来。
3.  一个交织掩码构建器，为“（图像1）（文本1）（图像2）（文本2）”序列生成二维注意力掩码。

## 部署应用

本课程产出`outputs/skill-gated-bridge-diagnostic.md`。给定一个开放VLM的配置（是否使用重采样器、交叉注意力频率、门控方案），它能识别出Flamingo谱系的元素并解释冻结策略。对于调试为什么微调导致文本性能下降很有用（答案是：门开得太快太早了）。

## 练习题

1.  计算Flamingo-9B的视觉参数量：90亿LLM + 14亿门控交叉注意力层 + 6400万重采样器。可训练参数占总参数的比例是多少？

2.  在PyTorch中实现门控残差`y = tanh(alpha) * cross + x`。通过实验证明，在初始化时，当`alpha=0`，`y==x`。

3.  阅读OpenFlamingo第3.2节（arXiv:2308.01390），了解他们如何处理批量中每个提示图像数量不同的情况。描述其填充策略。

4.  为什么Flamingo的交叉注意力掩码只允许一个文本词元关注*最近的*前一张图像，而不是所有之前的图像？阅读Flamingo论文第2.4节并解释其权衡。

5.  上下文少样本：为一个新的Flamingo变体构建一个包含4个“图像→主体颜色”示例的提示。描述当示例数量从0变到8时，预期的准确率模式。

## 关键术语

| 术语 | 常用说法 | 实际含义 |
|------|----------|----------|
| 感知器重采样器 | “固定潜在交叉注意力” | 从可变数量的输入块生成K个固定词元的模块 |
| 门控交叉注意力 | “Tanh门控桥梁” | 残差层`y = tanh(alpha)*cross + x`，可学习的alpha，初始化为0 |
| 交织输入 | “混合序列” | 图像和文本按阅读顺序自由混合的提示格式 |
| 冻结LLM | “无LLM梯度” | 文本LLM的权重不更新；只有重采样器 + 交叉注意力层被训练 |
| 少样本 | “上下文示例” | 在提示中给出几个（图像，答案）对；模型无需微调即可泛化 |
| OBELICS | “交织网络语料库” | 包含1.41亿个图像和文本按阅读顺序排列的网页的开放数据集 |
| Chinchilla | “700亿冻结基座” | Flamingo冻结的文本LLM，来自DeepMind的Chinchilla论文 |
| 门控调度 | “alpha如何变化” | 训练期间交叉注意力门打开的速率 |
| 交叉注意力频率 | “每M层” | 插入门控交叉注意力模块的频率；Flamingo使用M=4 |
| OpenFlamingo | “开放复现版” | MosaicML/LAION提供的3-90亿参数开放检查点；架构与Flamingo相同 |
