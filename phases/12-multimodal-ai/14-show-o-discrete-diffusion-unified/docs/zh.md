# Show-o 与离散扩散统一模型

> Transfusion 混合了连续与离散表征。Show-o（Xie 等人，2024年8月）则采用了相反路径：文本标记使用因果下一标记预测，图像标记使用符合 MaskGIT 精神的掩码离散扩散。两者集成在一个 Transformer 中，并采用混合注意力掩码。该设计在单一主干网络、每种模态一个分词器、一种损失公式（将下一标记预测扩展为掩码预测）的基础上，统一了视觉问答、文本生成图像、图像修复和混合模态生成。本课程将解析 Show-o 的设计——为何掩码离散扩散是一种并行、少步图像生成器——并将其与 Transfusion 和 Emu3 进行对比。

**类型：** 学习
**语言：** Python（标准库，掩码离散扩散采样器）
**先修课程：** 第12·13期（Transfusion）
**时长：** 约120分钟

## 学习目标

- 解释掩码离散扩散：均匀掩码标记，然后要求 Transformer 恢复它们的调度过程。
- 在速度和质量方面，比较并行图像解码（Show-o、MaskGIT）与自回归图像解码（Chameleon、Emu3）。
- 列出 Show-o 在单一检查点中处理的三项任务：文本生成图像、视觉问答、图像修复。
- 选择一种掩码调度（余弦、线性、截断）并推理其对样本质量的影响。

## 问题背景

Transfusion 的双损失训练有效但动态更复杂——连续扩散损失与离散下一标记预测损失处于不同的数值尺度上。平衡损失权重是一项超参数搜索。其架构虽有效但复杂。

Show-o 的解决方案：保持两种模态均为离散（类似 Chameleon），但通过掩码离散扩散而非顺序方式并行生成图像。训练目标变为单一的掩码标记预测，自然地泛化了下一标记预测。

## 核心概念

### 掩码离散扩散（MaskGIT）

Chang 等人（2022）提出的原始 MaskGIT 技巧十分精妙。从完全掩码的图像开始（每个标记均为特殊的 `<MASK>` 标识符）。在每一步中，并行预测所有掩码标记，然后保留置信度最高的前 K 个预测，并重新掩码其余部分。经过约 8-16 次迭代后，所有标记均被填充。每一步解掩码多少标记的调度是经过调优的——余弦调度效果良好。

训练很简单：从 [0, 1] 均匀采样一个掩码率，将其应用于图像的向量量化标记，训练 Transformer 恢复被掩码的标记。这与 BERT 对文本所做的完全相同，只是扩展到了图像生成。

### Show-o：单一 Transformer，混合掩码

Show-o 将 MaskGIT 嵌入一个因果语言模型 Transformer 中。其注意力掩码为：

- 文本标记：因果式（标准大语言模型）。
- 图像标记：在图像块内完全双向（因此掩码标记在预测时可以看到所有其他图像标记）。
- 文本到图像：文本关注先前图像，图像关注先前文本。

训练交替进行：
1. 在文本序列上进行标准下一标记预测。
2. 文本到图像样本：文本 → 图像，图像标记被掩码，使用掩码标记预测损失。
3. 视觉问答样本：图像 → 文本，文本标记被掩码（实质上就是下一标记预测）。

统一损失是基于 `<MASK>` 标记的交叉熵，它涵盖了文本下一标记预测（仅最后一个标记被“掩码”）和图像掩码扩散（随机子集被掩码）。

### 并行采样

Show-o 在约 16 步内生成一张图像，而非约 1000 步（自回归每个标记）或约 20 步（扩散）。每一步，并行预测所有掩码标记；提交置信度最高的前 K 个；重复。

比较：
- Chameleon / Emu3（自回归遍历标记）：N_标记 次前向传播，每张图像通常需要 1024-4096 步。
- Transfusion（连续扩散）：约 20 步，每步一次完整的 Transformer 传播。
- Show-o（掩码离散扩散）：约 16 步，每步一次完整的 Transformer 传播。

Show-o 在相似规模模型上比 Chameleon 更快，步骤数大致与 Transfusion 相当，但每步成本更低（离散词汇对数几率 vs 连续均方误差损失）。

### 单一检查点支持多种任务

Show-o 在推理时支持四种任务，通过提示格式选择：

- 文本生成：标准自回归文本输出。
- 视觉问答：输入图像，输出文本。
- 文本生成图像：输入文本，通过掩码离散扩散输出图像。
- 图像修复：输入部分标记被掩码的图像，进行填充。

图像修复能力来自掩码预测训练，是“免费”的。掩码向量量化标记网格的一个区域，输入其余部分加上文本提示，预测被掩码的标记。

### 掩码调度

每一步解掩码多少标记的调度影响质量。Show-o 推荐余弦调度：

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

在步骤 0，所有标记被掩码（比例 1.0）。在步骤 T，无掩码。余弦将概率质量集中在中段比例上，此时预测最具信息量。线性调度也可以，但会更快达到平台期。

### Show-o2

Show-o2（2025年后续研究，arXiv 2506.15564）对 Show-o 进行了扩展：更大的大语言模型基础、更好的分词器、改进的掩码调度。架构模式相同。

### Show-o 的定位

在2026年的分类体系中：

- 离散标记 + 下一标记预测：Chameleon、Emu3。简单但推理慢。
- 离散标记 + 掩码扩散：Show-o、MaskGIT、LlamaGen、Muse。并行采样，但仍受限于分词器的有损压缩。
- 连续 + 扩散：Transfusion、MMDiT、DiT。质量最高，训练更复杂。
- 在视觉语言模型中使用连续 + 流匹配：JanusFlow、InternVL-U。最新。

根据任务选择：当你需要在一个开放模型中同时实现文本生成图像、图像修复和视觉问答，并兼顾合理速度时，选择 Show-o；当质量至关重要且你能承担双损失训练的复杂性时，选择 Transfusion。

## 练习

1.  掩码离散扩散采样约需 16 步。为何不是 1 步？如果在步骤 0 就解掩码所有内容，会出什么问题？

2.  图像修复在掩码扩散中是免费的。提出一个产品用例（真实或假设的），其中 Show-o 的图像修复优于专用模型。

3.  余弦调度 vs 线性调度：在 T=8 时，追踪每一步解掩码的标记数量。哪种更均衡？

4.  一张 512x512 的 Show-o 图像包含 1024 个标记。词汇量 K=16384 时，模型输出 1024 * log2(16384) = 14,336 比特（约 1.75 KiB）的数据。Stable Diffusion 输出 512*512*24 比特 = 6,291,456 比特（约 768 KiB）的原始像素。压缩比是多少？它带来了什么质量？

5.  阅读 LlamaGen（arXiv:2406.06525）。LlamaGen 的类条件自回归图像模型与 Show-o 的掩码方法有何不同？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------|---------|
| 掩码离散扩散 | “MaskGIT 风格” | 训练以预测被掩码的标记；推理时，迭代解掩码置信度最高的预测 |
| 余弦调度 | “解掩码调度” | 推理步骤中掩码率的衰减；将置信度增长集中在中段范围 |
| 并行解码 | “一次性处理所有标记” | 每一步在一次前向传播中预测整个被掩码标记序列，然后提交前 K 个 |
| 混合注意力 | “因果 + 双向” | 对文本标记为因果式、对图像块内为双向式的掩码 |
| 图像修复 | “填充生成” | 以部分标记被掩码的图像为条件，预测缺失部分；源于训练目标，无需额外成本 |
| 提交率 | “每步前 K 个” | 每次迭代有多少标记被宣布为“完成”；控制推理速度与质量的权衡 |
