# 视频语言模型：时间令牌与定位

> 视频不是照片的堆叠。一段5秒的片段具有图像模型无法表示的因果顺序、动作动词和事件时序。Video-LLaMA (Zhang等人，2023年6月) 发布了首个具有视听定位功能的开源视频-LLM。VideoChat和Video-LLaVA扩展了这一模式。到2025年，Qwen2.5-VL的TMRoPE缩小了与前沿专有模型的差距。每个系统用不同的方式处理时间令牌——Video-LLaMA的每片段Q-former、Video-LLaVA的每帧拼接池化、Qwen2.5-VL的每令牌TMRoPE。本课程解析这些模式，构建统一与动态帧采样器，并在时间定位任务上进行评估。

**类型：** 构建
**语言：** Python (标准库, 帧采样器 + 时间定位评估器)
**前置要求：** 阶段12 · 08 (LLaVA-OneVision)
**时间：** ~180分钟

## 学习目标

- 解释为什么时间位置编码独立于视觉编码器改变视频VLM性能。
- 比较统一、动态FPS和事件驱动帧采样在每秒令牌数与定位精度上的权衡。
- 描述Q-former-per-clip (Video-LLaMA) 与 pooled-per-frame (Video-LLaVA) 与 M-RoPE-per-token (Qwen2.5-VL) 的设计差异。
- 说出四个视频基准测试：VideoMME、TempCompass、EgoSchema、Video-MMMU。

## 问题所在

一个1分钟、30 FPS的视频有1800帧。按每帧196个视觉令牌（ViT-B at 224），那就是35.2万个令牌——比任何2024时代的LLM上下文都大。

存在三种缩减策略：

1. 子采样帧（根据内容，1-8 FPS）。
2. 对每帧的patch令牌进行激进池化（3x3或4x4双线性池化）。
3. 通过Q-former压缩，它接收一个16帧片段，输出64个令牌。

每种权衡都不同。子采样丢失时间细节。池化丢失空间细节。Q-former两者都丢失一点但节省令牌。

时间位置编码是另一个维度：模型如何知道第5帧在第6帧之前？选项包括简单的一维时间RoPE (Video-LLaMA)、学习到的时间嵌入 (Video-LLaVA) 和TMRoPE (Qwen2.5-VL，完整的3D)。

## 概念解析

### Video-LLaMA：每片段Q-former + 音频分支

Video-LLaMA (2023) 是第一个开源视频-LLM。架构：

- 以2 FPS采样16帧片段（即8秒）。
- 每帧的ViT特征 -> 视频Q-former在所有16帧上进行交叉注意力 -> 32个学习到的查询 -> LLM。
- 并行的音频分支：波形 -> ImageBind音频编码器 -> 音频Q-former -> 32个查询 -> LLM。

优势：视听联合推理。劣势：固定片段长度，无法进行任意时间定位。

### VideoChat和Video-LLaVA

VideoChat保留了Video-LLaMA的思想，但去掉了音频并简化了。Video-LLaVA (Lin等人，2023) 在图像和视频帧上训练单个视觉编码器（“投影前对齐”），提供了统一表示。两者都是冻结的CLIP编码器 + MLP + LLM。

两者都无法处理长视频。都是8-16帧的系统。

### Qwen2.5-VL和TMRoPE

Qwen2.5-VL引入了TMRoPE——时间模态旋转位置嵌入。每个patch令牌携带一个(t, h, w)位置，其中t是实际时间戳（不是帧索引）。

与简单时间嵌入的关键区别：

- 绝对时间，而非索引。模型看到的是“在4.2秒”而不是“在第15帧”。
- 每令牌旋转，而非每片段。每个视觉令牌根据其时间戳独立旋转。
- 兼容动态FPS。如果你在这里以2 FPS采样，在那里以4 FPS采样，TMRoPE原生处理不均匀的间隔。

TMRoPE支持“猫在第几秒跳起来？”这样的查询。模型可以输出“在4.2秒”。Video-LLaMA只能说“在片段的早期”。

### 帧采样策略

统一：在持续时间内均匀采样N帧。简单，但会丢失运动峰值。

动态FPS：根据运动强度自适应采样。光流或帧差分法选择高运动片段进行更密集的采样。Qwen2.5-VL在此模式下训练。

事件驱动：运行一个轻量级检测器，在动作发生的地方采样更多。VideoAgent使用此方法。

关键帧 + 上下文：在镜头边界处采样 + 几个相邻帧。用于电影内容。

### 每帧池化

以1 FPS和每帧576个令牌，一个5分钟的片段是172,800个令牌。使用Qwen2.5-VL-72B的128k上下文可以做到，但很昂贵。

3x3双线性池化将每帧减少到64个令牌 -> 5分钟为19,200个令牌。这是大多数任务的最佳平衡点。

更激进的池化（6x6 -> 每帧16个令牌）用于空间细节不太重要的智能体工作流。

### 四个视频基准测试

- VideoMME：全面的视频理解，短、中、长视频。
- TempCompass：细粒度的时间推理，“之前”/“之后”问题。
- EgoSchema：长时域第一人称视频。
- Video-MMMU：多模态多学科视频问题。

一个完整的视频-VLM评估会覆盖全部四个。它们侧重不同维度——TempCompass全部关于顺序，EgoSchema关于3分钟以上的推理，VideoMME跨越不同时长。

### 定位输出格式

时间定位的输出格式：

- 自由文本：“猫在大约4秒时跳起来。”易于解析但不精确。
- 结构化JSON：`{"event": "jump", "start": 4.1, "end": 4.3}`。Qwen2.5-VL训练此格式。
- 基于令牌：特殊的`<time>4.1</time>`令牌穿插在答案中。Qwen2.5-VL的内部格式。

基于令牌对于下游使用最精确。Qwen2.5-VL的JSON输出格式可以直接解析。

### 2026年最佳实践

对于2026年的视频VLM：

- 编码器：带M-RoPE或TMRoPE的SigLIP 2 (Qwen2.5-VL)。
- 帧采样：动态FPS（根据运动，1-4 FPS）并设置最大帧数上限。
- 每帧池化：3x3双线性。
- 输出：带时间和事件字段的结构化JSON。
- 基准测试：通用用VideoMME + TempCompass；长时域用EgoSchema。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|------|-----------------|------------------------|
| 时间定位 | “时间局部化的答案” | VLM输出事件发生的具体时间戳范围 |
| TMRoPE | “时间-多模态RoPE” | 带有绝对时间戳的3D旋转位置，由Qwen2.5-VL使用 |
| 动态FPS | “运动感知采样” | 在高运动片段采样更多帧，在静态片段采样更少帧 |
| 帧池化 | “每帧的空间压缩” | 在LLM之前使用双线性插值减少每帧的patch数 |
| 视频Q-former | “片段压缩器” | 交叉注意力瓶颈，将N帧映射到K个学习到的查询 |
| VideoMME | “视频基准测试” | 全面的短/中/长视频基准测试，2500+样本 |
