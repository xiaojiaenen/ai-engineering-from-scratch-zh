# 多模态智能体与计算机使用（综合项目）

> 2026年前沿产品是一款能读取截图、点击按钮、浏览网页界面、填写表单并端到端完成工作流程的多模态智能体。SeeClick和CogAgent（2024年）证明了图形用户界面定位的基本原理。Ferret-UI扩展了移动端能力。ChartAgent引入了图表的视觉工具使用。VisualWebArena和AgentVista（2026年）是前沿竞争的基准测试——即便是Gemini 3 Pro和Claude Opus 4.7在AgentVista的高难度任务上也仅能获得约30%的分数。本综合项目汇集了第12阶段的所有线索：感知（高分辨率视觉语言模型）、推理（具有工具使用能力的LLM）、定位（坐标输出）、长期记忆和评估。

**类型：** 综合项目
**语言：** Python（标准库、动作模式+智能体循环骨架）
**先决条件：** 第12阶段 · 05（LLaVA），第12阶段 · 09（Qwen-VL JSON），第14阶段（智能体工程）
**时间：** 约240分钟

## 学习目标

- 设计多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建图形用户界面定位输出模式（点击坐标、输入文本、滚动、拖拽），让视觉语言模型能够以JSON格式输出。
- 比较仅使用截图的智能体与使用可访问性树的智能体与混合智能体。
- 在一小部分VisualWebArena切片上建立多模态智能体基准评估。

## 问题描述

一个预订网站的工作流程：“帮我找一张4月15日去东京的机票，靠走道座位，价格低于800美元，然后预订它。”

一个多模态智能体需要：

1. 获取浏览器的截图。
2. 解析截图+URL+目标为一个计划。
3. 发出一个结构化动作：点击（在x,y处）、输入“东京”（在元素E处）、向下滚动、选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一次多模态视觉语言模型调用。视觉语言模型的输出必须是可解析的JSON。错误会在各步骤间累积，因此错误恢复至关重要。

## 核心概念

### 图形用户界面定位——基本原理

图形用户界面定位是：给定一个截图和自然语言指令，输出要点击的（x, y）坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个大规模的开源成果：在合成与真实图形用户界面数据上微调视觉语言模型，将坐标作为纯文本token输出。可行。

CogAgent（arXiv:2312.08914）为密集界面增加了1120x1120高分辨率编码。在网页导航上得分约84%。

Ferret-UI（arXiv:2404.05719）专注于移动设备界面，与iOS可访问性数据集成。

输出格式通常是JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 有助于错误恢复：如果坐标在截图之间漂移，语义提示能让系统重新定位。

### 动作模式

一个典型动作模式有6-10种动作类型：

- `click`：（x, y）
- `type`：（文本, x?, y?）
- `scroll`：（方向, 数量）
- `drag`：（x0, y0, x1, y1）
- `select`：（选项索引）
- `hover`：（x, y）
- `navigate`：（URL）
- `wait`：（毫秒）
- `done`：（成功, 解释）

智能体每一步发出一个动作。浏览器包装器执行并返回新状态。

### 仅截图 vs 可访问性树

两种输入模式：

- 仅截图：完整图像，无结构信息。最通用；适用于任何应用。
- 可访问性树：结构化的DOM/iOS可访问性信息。定位可靠得多；在树可用的地方有效。
- 混合：两者结合，树作为原子动作的可靠定位器，截图提供语义上下文。

生产环境智能体在可能时使用混合模式。浏览器自动化（Selenium+可访问性）总有树；桌面应用有时有。

### 长期记忆

一个20步的工作流程会产生20张截图。视觉语言模型的上下文会很快填满。三种压缩策略：

- 摘要链：每5步后，总结已发生的事情，丢弃旧截图。
- 跳帧：保留第一张、最后一张和每第三张截图。
- 工具记录日志：执行动作，保留所做事情的文本日志；不重新查看旧截图。

Claude的计算机使用API采用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）引入了用于图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以输出“裁剪到区域(100, 200, 300, 400)然后调用OCR”作为工具调用。工具返回文本；视觉语言模型继续推理。

这种模式可推广：集合标记提示、区域注释和外部检测工具都符合相同的“输出工具调用，接收结构化响应”模式。

### 2026年基准测试

- ScreenSpot-Pro。在约1000张网页截图上进行图形用户界面定位。开源最佳模型Qwen2.5-VL-72B约85%。前沿约90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类广告）。开源最佳模型约20%。Gemini 3 Pro约27%。
- AgentVista（arXiv:2602.23166）。2026年最难的基准测试。横跨12个领域的现实工作流程。前沿模型得分27-40%；开源模型10-20%。
- WebArena / WebShop。较旧的基准测试；已被前沿模型饱和。

### 为何仍然困难

智能体性能瓶颈：

1. 细粒度视觉定位。“点击小X”在移动端分辨率下经常失败。
2. 长期规划。10个动作后，智能体偏离目标。
3. 错误恢复。当点击失败（按钮错误）时，检测+恢复很少在训练数据中。
4. 跨页面上下文。在标签页或长表单间跳转会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（用截图匹配来判断动作成功）。

### 综合项目构建

综合项目任务：构建一个计算机使用智能体，它能够：

1. 读取预订网站模拟页面的HTML + 截图。
2. 规划多步骤序列：搜索 → 选择 → 填写表单 → 提交。
3. 发出匹配动作模式的JSON动作。
4. 在固定的10个任务切片上进行评估。

课程提供了易于扩展到真实浏览器的脚手架代码。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|------|-----------------|------------------------|
| 图形用户界面定位 | "点击坐标" | 模型为指令在截图上的目标输出(x,y) |
| 动作模式 | "工具定义" | 对有效动作（点击、输入、滚动、拖拽）的JSON描述 |
| 可访问性树 | "结构化DOM" | 来自浏览器/iOS API的机器可读界面层次结构 |
| 混合智能体 | "截图+树" | 同时使用图像和结构化信息；比单独使用任何一种更可靠 |
| 视觉工具使用 | "缩放/裁剪/检测" | 智能体在计划过程中调用外部视觉工具（OCR、检测） |
| 摘要链 | "记忆压缩" | 定期用文本摘要替代长截图历史 |
| VisualWebArena | "端到端网页基准" | 2024年端到端网页任务的基准测试 |
| AgentVista | "2026年高难度基准" | 12个领域的现实工作流程；即便是Gemini 3 Pro也仅得约30% |
