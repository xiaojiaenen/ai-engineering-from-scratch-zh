# 具身视觉-语言-动作模型：RT-2、OpenVLA、π0、GR00T

> 2023年7月，谷歌DeepMind的RT-2首次实现让模型读取网络食谱并在厨房机器人上执行。RT-2将动作离散化为文本标记，在网页数据和机器人动作数据上联合微调视觉语言模型，证明了网络规模的视觉语言知识可迁移至机器人控制。2024年6月发布的OpenVLA提供了开源的70亿参数参考模型。Physical Intelligence公司的π0系列（2024-2025）引入了流匹配动作专家。英伟达的GR00T N1（2025年3月）为人形机器人实现了大规模双系统（系统1/系统2）控制。视觉-语言-动作（VLA）原语——一种能看、能读、能行动的单一模型——是连接本阶段理解模型与第15阶段自主系统的桥梁。

**类型：** 学习
**语言：** Python（标准库、动作分词器 + VLA推断骨架）
**先决条件：** 第12阶段·第05课（LLaVA），第15阶段（自主系统，已引用）
**时间：** 约180分钟

## 学习目标

- 描述动作分词：离散区间编码（RT-2）、FAST高效动作分词、连续流匹配动作（π0）。
- 解释为何在网页+机器人数据上联合微调能保持通用知识向新任务的迁移。
- 在相同机器人任务上对比OpenVLA（开源70亿参数Llama+VLM）、π0（流匹配）和GR00T N1（双系统）。
- 列举Open X-Embodiment数据集及其作为RT-X训练语料库的作用。

## 问题背景

通过自然语言指令完成家务的机器人自20世纪70年代以来一直是研究目标。2020年代的答案：视觉-语言-动作（VLA）模型。采用与视觉问答（VQA）相同的VLM架构，但输出是动作（关节扭矩、末端执行器位姿、离散指令）而非文本。

VLA特有的挑战：

1. 动作空间是连续的（关节角度、力）且高维的（7自由度机械臂 + 3自由度抓手 = 10维，30Hz）。
2. 机器人专用训练数据稀缺。Open X-Embodiment约有100万条轨迹；网页图文数据超过50亿。
3. 控制频率至关重要。30Hz控制循环意味着每个动作有33毫秒的计算预算。
4. 安全性。错误的动作会损坏硬件、伤害人类或财产。

## 核心概念

### 动作分词（RT-2）

RT-2的诀窍：将每个关节目标表示为量化后的文本标记。将归一化到[-1, 1]的区间离散化为256个区间，每个区间映射到一个词汇表ID。一个10自由度的动作在每个控制步骤产生10个标记。

在混合数据上联合微调PaLM-X VLM：

- 网页图文对（图像描述、视觉问答）。
- 机器人演示，动作作为标记。

模型接收“拿起红色方块”（语言）→ 图像（视觉）→ 10个标记的动作序列（离散化的关节目标）。网页预训练保留了通用知识迁移：RT-2可以执行“向快速移动的物体移动”，即使“快速移动”不在训练数据中。

RT-2论文中推断速度为3-5 Hz，受限于VLM的自回归解码。

### OpenVLA——开源的70亿参数参考模型

OpenVLA（Kim等，2024年6月）是开源权重的RT-2等效模型。采用70亿参数Llama主干网络，DINOv2 + SigLIP双视觉编码器，基于256区间的动作分词。

在Open X-Embodiment（涵盖22种机器人的97万条轨迹）上训练。支持LoRA微调以适配新机器人。

推断：在A100上量化后达到4-5 Hz。足以应对慢速操作，但无法满足高频控制。

### FAST分词器——更快的动作解码

Pertsch等（2024）指出离散区间分词效率低下——大多动作聚集在区间空间的小区域内。FAST（频域动作序列分词器）通过离散余弦变换（DCT）压缩动作序列并量化系数。

一条30步的动作轨迹变为约10个FAST标记，而非300个离散区间标记。推断速度提升3-5倍，且不损失质量。

### π0与流匹配动作

Physical Intelligence公司的π0（Black等，2024年10月）用流匹配动作专家替换了离散动作标记：

- 一个小型动作Transformer读取VLM的隐藏状态，通过整流流输出连续的50步动作序列。
- 动作头使用流匹配损失训练；VLM预训练保持不变。
- 推断：通过约5个去噪步骤发出完整动作序列，实现有效的50Hz控制。

π0声称：在广泛的操作任务中超越OpenVLA和Octo。连续动作公式保留了离散化会破坏的平滑性。

π0.5和π0-FAST是增量升级。π0-FAST结合了FAST分词与流匹配。

### GR00T N1——人形机器人的双系统

英伟达的GR00T N1（2025年3月）专为人形机器人设计（超过30自由度，全身）：

- 系统2：大型VLM解读场景+指令，以约1Hz生成高层子目标。
- 系统1：小型动作头Transformer根据子目标生成底层50-100Hz关节指令。

这种划分对应卡尼曼的快慢思考：系统2规划，系统1行动。优势：慢速的VLM级规划不阻塞快速控制；系统1保持小巧以降低延迟。

GR00T N1.7（2025年末）改进了数据扩展。GR00T使用Omniverse的仿真到真实数据进行微调。

### Open X-Embodiment

训练数据。RT-X（2023年10月）整合了22个数据集，涵盖22种机器人的100万条轨迹。Open X-Embodiment是大家共用的语料库：

- 包含ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table等数据集。
- 每个样本：（机器人状态、相机视角、指令、动作序列）。
- 训练规范：统一动作空间、归一化关节范围、调整相机分辨率。

OpenVLA和π0在Open X-Embodiment上训练。与任何特定机器人的领域差异通过100-1000个任务专用演示的LoRA微调来消除。

### 联合微调 vs. 仅机器人数据训练

联合微调混合网页视觉问答数据与机器人轨迹。比例很重要：视觉问答数据过多会导致模型忘记动作；机器人数据过多会导致模型丧失通用知识。

RT-2的比例：约1:1。OpenVLA：网页与机器人数据约0.5:1。π0：类似。精确比例是需根据数据集大小调整的超参数。

仅用机器人数据训练产生的任务专用模型，在分布外指令下会失败。联合微调是区分“拿起红色方块（演示中的）”和“拿起左边第三大的物体（新表述）”的关键。

### 安全性与动作限制

每个生产级VLA都附带：

- 关节硬限制（扭矩不能超过规格）。
- 速度限制（软限幅）。
- 工作空间边界（末端执行器不能离开工作台）。
- 新任务需经人类在环批准。

这些作为控制层检查存在于VLA之外。VLA的输出是建议，而非命令。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| VLA | “视觉-语言-动作” | 接收图像+指令并输出动作指令的模型 |
| 动作分词 | “离散区间” | 将连续关节目标量化为每维256个区间，每个区间对应一个词汇表ID |
| FAST分词器 | “频域动作标记” | DCT + 量化，将30步轨迹压缩为约10个标记 |
| 联合微调 | “混合网页+机器人” | 在网页视觉问答数据和机器人演示上共同训练，以保留通用知识 |
| 流匹配动作头 | “π0连续输出” | 小型Transformer，通过整流流输出50步动作序列 |
| 系统1/系统2 | “双系统控制” | 大型VLM缓慢规划，小型动作头快速行动；GR00T模式 |
| Open X-Embodiment | “RT-X数据集” | 包含100万条轨迹的跨机器人数据集；训练语料库 |
