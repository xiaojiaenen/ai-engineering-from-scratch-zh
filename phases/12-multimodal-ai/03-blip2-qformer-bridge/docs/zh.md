# 从 CLIP 到 BLIP-2 — Q-Former 作为模态桥梁

> CLIP 对齐了图像和文本，但无法生成描述、回答问题或进行对话。BLIP-2（Salesforce，2023）通过一个小型可训练桥梁解决了这个问题：32个可学习的查询向量通过交叉注意力关注冻结的ViT的特征，然后直接嵌入冻结的LLM的输入流中。这个188M参数的桥梁将一个11B的LLM连接到了一个ViT-g/14。到2026年，所有基于适配器的VLM——MiniGPT-4、InstructBLIP、LLaVA的衍生模型——都是它的后裔。本课解读了Q-Former的架构，解释了其两阶段训练，并构建了一个玩具版本，将视觉token输入冻结的文本解码器。

**类型：** 构建
**语言：** Python（标准库，交叉注意力+可学习查询演示）
**先决条件：** 阶段12 · 02（CLIP），阶段7（Transformers）
**时间：** ~180分钟

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的LLM之间设置一个可训练的瓶颈，比端到端微调在成本和稳定性上更具优势。
- 实现一个交叉注意力块，其中一组固定的可学习查询关注外部图像特征。
- 了解BLIP-2的两阶段预训练：表征阶段（ITC + ITM + ITG）和生成阶段（带冻结解码器的LM损失）。
- 比较Q-Former与LLaVA中使用的更简单的MLP投影器，并论述每种选择的优势场景。

## 问题所在

你有一个冻结的ViT，每张图像生成256个维度为1408的patch token。你有一个冻结的7B LLM，它期望维度为4096的token嵌入。显而易见的桥梁——一个从1408到4096的线性层——可行，但将全部256个patch token输入LLM的上下文，每张图像会消耗额外的256个token。在一个32张图像的批次中，仅视觉模态就会占用8192个token。

BLIP-2提出的问题是：你能否将256个token的图像表示压缩到远少于32个token，同时为LLM保留足够的信息来描述图像、回答问题和进行推理？并且你能否在不触及冻结主干网络的情况下训练这个桥梁，仅将训练成本控制在桥梁参数上？

答案：一个Q-Former。32个可学习的“查询”向量，它们交叉关注ViT的patch token，生成一个32个token的视觉摘要供LLM使用。总共188M参数。在接触LLM之前，使用对比性、匹配性和生成性目标进行训练。

## 核心概念

### 可学习查询

Q-Former的核心技巧：不是让LLM的文本token去关注图像patch，而是引入一组新的32个可学习查询向量`Q`，让*它们*去关注图像patch。这些查询是模型的参数——在训练过程中学习，并且相同的32个查询用于每张图像。

交叉注意力之后，每个查询都包含一个压缩的图像摘要——“描述主要物体”、“描述背景”、“计数物体”等。这些查询并非专门针对语义标签进行特化；它们学习的是使下游损失降低的任何编码方式。

### 架构

Q-Former是一个小型Transformer（12层，约1亿参数），包含两条路径：

1. 查询路径：32个查询向量流经自注意力（在它们之间），然后对冻结的ViT的patch token进行交叉注意力，接着是前馈网络。
2. 文本路径：一个类似BERT的文本编码器与查询路径共享自注意力和前馈网络权重。文本路径禁用交叉注意力。

在训练时，两条路径都运行。查询和文本通过共享的自注意力进行交互，这意味着查询可以根据需要（ITM、ITG）在任务中以文本为条件。在用于VLM交接的推理时，只有查询流经网络，生成32个视觉token。

### 两阶段训练

BLIP-2分为两个阶段进行预训练：

阶段1：表征学习（不涉及LLM）。三种损失：
- ITC（图像-文本对比）：在池化的查询token和文本CLS token之间进行CLIP风格的对比学习。
- ITM（图像-文本匹配）：二元分类器——这个图像-文本对匹配吗？采用硬负例挖掘。
- ITG（基于图像的文本生成）：在文本上使用因果LM头，以查询为条件。迫使查询编码可用于文本生成的内容。

只有Q-Former被训练。ViT是冻结的。没有LLM参与。

阶段2：生成式学习。连接一个冻结的LLM（如OPT-2.7B或Flan-T5-XL）。通过一个小型线性层将32个查询输出投影到LLM的嵌入维度。将它们置于文本提示之前。仅在连接的提示+图像+描述序列上，训练线性投影层和Q-Former的LM损失。

阶段2结束后，Q-Former + 投影层就是完整的视觉适配器。推理时：图像 → ViT → Q-Former → 线性投影 → 置于文本之前 → 冻结的LLM生成输出。

### 参数经济学

BLIP-2 搭配 ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，可训练）= 总共8B参数，训练188M参数。Q-Former本身仅占整个技术栈参数的约2.4%。训练成本也相应体现：在少数几张A100上几天，而端到端训练则需要数周。

质量：BLIP-2在零样本VQA上达到或超过了Flamingo-80B的性能，同时规模小50倍。桥梁奏效了。

### InstructBLIP 和 指令感知的Q-Former

InstructBLIP（2023）扩展了Q-Former，增加了一个额外的输入：指令文本本身。在交叉注意力时，查询现在可以访问图像patch和指令。查询可以针对每条指令（“数一数汽车”、“描述氛围”）进行特化，而不是学习一个固定的摘要。在 held-out 任务上取得了基准测试的提升。

### MiniGPT-4 和 仅投影器方法

MiniGPT-4保留了Q-Former，但仅训练输出线性投影层，同时冻结其他所有部分。成本低，但代价是质量——查询是BLIP-2的，不是你自己的。适合快速迭代，并非最佳架构。

### 为什么LLaVA选择了更简单的方法

LLaVA（2023，课程12.05）用一个简单的2层MLP替换了Q-Former，该MLP将每个ViT patch token投影到LLM空间——对于24x24网格，每张图像产生576个token，全部输入LLM。压缩效果较差，但让LLM可以直接关注原始patch。这在当时备受争议；到2023年底，它成为主流，因为视觉指令数据（LLaVA-Instruct-150k）证明MLP可以被训练以保留足够的信号。权衡是：LLaVA的上下文填充更快，但它能自然地扩展到多图像和视频。

到2026年，该领域出现了分化：在token预算紧张的场景（长视频、多图像）中，Q-Former得以保留；在追求每token原始质量的场景中，MLP投影器占据主导地位。

### 门控交叉注意力：Flamingo，鼻祖

Flamingo（课程12.04）早于BLIP-2，采用了相同的交叉注意力思想，但在每个冻结的LLM层上，而不是作为单一的桥梁。BLIP-2表明你可以仅压缩到输入层并仍然有效。Gemini和Idefics结合了两者：交错的输入token加上可选的门控交叉注意力用于上下文少样本学习。

### 2026年的后继者

- Q-Former：BLIP-2, InstructBLIP, MiniGPT-4, 以及出于token预算考虑的大多数视频-语言模型。
- Perceiver重采样器：Flamingo的变体（课程12.04）；Idefics系列、Eagle、OmniMAE。
- MLP投影器：LLaVA, LLaVA-NeXT, LLaVA-OneVision, Cambrian-1。
- 注意力池化：VILA, PaliGemma。

四种方式都有效。决定性的问题在于你是受token预算限制还是受每token质量限制。

## 使用它

`code/main.py` 构建了一个基于标准库的Q-Former风格的交叉注意力：

1.  模拟256个图像patch token（维度128）。
2.  实例化32个可学习查询（维度128）。
3.  运行缩放点积交叉注意力（Q来自查询，K/V来自patch）。
4.  通过线性层投影到LLM维度（512）。
5.  输出32个LLM就绪的视觉token。

所有数学运算均使用纯Python（向量的嵌套循环）。虽然简单但形状正确。注意力权重矩阵被打印出来，以便你可以看到每个查询从哪些patch中提取信息。

2.  在BLIP-2阶段1中，Q-Former同时运行三种损失：ITC、ITM、ITG。用伪代码写出每种损失的前向传播签名。哪一种需要文本编码器路径处于激活状态？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|------------|----------|
| Q-Former | “查询Transformer” | 带有32个可学习查询向量的小型Transformer，这些向量交叉关注冻结的ViT特征 |
| 可学习查询 | “视觉的软提示” | 一组固定的参数，作为交叉注意力的查询端；每个模型学习一次，所有输入共享 |
| 交叉注意力 | “Q来自这里，K/V来自那里” | 查询、键和值来自不同来源的注意力；查询如何从ViT patch中提取信息 |
| ITC | “图像-文本对比” | 应用于Q-Former池化查询与文本CLS的CLIP风格损失 |
| ITM | “图像-文本匹配” | 基于硬负例挖掘对的二元分类器；迫使查询区分细微的不匹配 |
| ITG | “基于图像的文本生成” | 以查询为条件的因果LM损失生成文本；迫使查询编码可用于文本解码的内容 |
| 两阶段预训练 | “先表征，后生成” | 阶段1单独训练Q-Former（ITC/ITM/ITG）；阶段2连接冻结的LLM，仅训练投影层+Q-Former |
| 冻结主干 | “不要微调” | 视觉编码器和LLM权重是固定的；只有桥梁被训练 |
| 投影头 | “线性投影到LLM维度” | 将Q-Former输出映射到LLM嵌入维度的最终线性层 |
| Perceiver重采样器 | “Flamingo的版本” | 类似的可学习查询交叉注意力，由Flamingo在每一层使用，而不是作为单一桥梁 |
