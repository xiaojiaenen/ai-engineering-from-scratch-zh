# InternVL3：原生多模态预训练

> 在 InternVL3 之前，所有开放的视觉语言模型都遵循相同的三步流程：取一个在数万亿文本 token 上训练的文本 LLM，装上一个视觉编码器，然后微调连接处。这种方法可行，但会产生“对齐债务”——文本 LLM 在其整个预训练过程中只专注于纯文本，并不原生理解视觉 token。当你事后添加视觉能力时，LLM 必须在不遗忘文本的前提下，重新学习如何将视觉输入与其文本推理关联起来。InternVL3 (Zhu 等人, 2025年4月) 拒绝了这种事后补救的方法：一次预训练，从第一步开始就交织文本和多模态数据。结果是，它在 78B 参数规模下，在 MMMU-Pro 上与 Gemini 2.5 Pro 匹敌，且是开放的。本课解读原生预训练的理由，以及当你采用它时会发生什么变化。

**类型：** 学习
**语言：** Python (标准库，训练语料混合器)
**前置条件：** 阶段 12 · 05，阶段 12 · 07 (流程)
**时间：** 约 120 分钟

## 学习目标

- 解释事后 VLM 训练为何会累积对齐债务，并列举三个可衡量的症状（灾难性遗忘、答案漂移、视觉-文本不一致性）。
- 描述 InternVL3 的原生预训练语料构成比例，并解释文本：交织：图文描述的比例为何重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（ViR）和解耦视觉-语言（DvD）部署优化。

## 问题所在

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics —— 都是取一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral），然后添加视觉能力。其训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练的投影器，在图文描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的特定任务微调。

对齐债务的三个症状随之显现：

- **灾难性遗忘**。事后的 VLM 遗忘了纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本智能体性能回退。
- **答案漂移**。对同一视觉问题的微小措辞变化会得到不同的答案。视觉编码器与 LLM 的连接强度弱于 LLM 自身 token 之间的连接。
- **视觉-文本不一致性**。VLM 可以正确描述一张图片，然后回答一个与其自身描述相矛盾的问题。视觉 token 参与 LLM 内部一致性检查的方式与文本不同。

这些症状已有充分记录。MM1.5 第 4 节对其进行了量化。LLaVA-OneVision 的消融实验也暗示了它们。原生预训练是解决方案。

## 核心概念

### 原生多模态预训练

InternVL3 从头开始在一个从第一步就是原生多模态的语料上训练。混合比例是：

- 40% 纯文本数据（FineWeb, Proof-Pile-2 等）
- 35% 交织图文数据（OBELICS, MMC4 风格）
- 20% 图文对数据
- 5% 视频文本数据

从第一个梯度步开始，视觉 token、文本 token 和跨模态交互都参与相同的损失函数计算。没有对齐预训练，没有投影器冻结阶段，无需从灾难性遗忘中恢复。

基础模型的训练是单一阶段的。随后是指令微调，但基础模型已经将视觉 token 理解为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用具有固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：位置编码因模态类型（文本、图像、视频）而异，并具有可学习的缩放因子。实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像块获得 2D 位置（行，列）。
- 视频帧获得 3D 位置（时间，行，列）。

三者共享相同的 RoPE 频率基准，但每个频段的隐藏维度分配是一个可学习参数，而非固定划分。在预训练期间可以自由权衡时间与空间的频率分辨率。

V2PE 的消融实验声称：在相同计算量下，在视频基准上比 M-RoPE 高 1-2 个点。不是革命性的，但更简洁。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张低细节、只有一个物体的照片，以 1280px 原生分辨率编码会浪费 token。ViR 是一个小型分类器，它在编码前预测回答问题所需的最小分辨率。

路由分为三个层级：低分辨率（256 个 token）、中等（576）、高（2048+）。在生产流量的查询中，60% 的情况下低或中等分辨率就足够了。净效果：在同等质量下，吞吐量提升 2-3 倍。

### 解耦视觉-语言部署（DvD）

当你部署一个大型 VLM 时，视觉编码器对每张图像运行一次，但 LLM 为每个输出 token 自回归地运行。这两个组件有不同的瓶颈（视觉 = 用于卷积和注意力的 GPU 显存带宽；LLM = KV 缓存）。DvD 将它们拆分到独立的 GPU 上，并在它们之间进行流式传输。

对于一个 8B + 400M 编码器的模型，DvD 与共置相比，大致使单节点吞吐量翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声称：在 78B 参数下，与 Gemini 2.5 Pro 的 MMMU-Pro 匹敌。在 38B 下，与 GPT-4o 匹敌。在 8B 下，在开放 8B 排行榜上领先。所有这些都是基于单阶段预训练 + 指令微调的流程。

对齐债务假设是可衡量的：InternVL3-8B 每单位视觉基准增益所损失的文本基准点数（MMLU, GSM8K）少于 Qwen2.5-VL-7B。该模型更具通用性，因为训练是一体的，而不是两段式的。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025年8月）扩展了该流程。相同的原生预训练方法，更多的数据，更多的参数。MMMU 的提升是渐进的。

InternVL-U（2026年）增加了统一生成能力——在相同的骨干网络上，通过 MMDiT 头输出图像。“U”代表“理解 + 生成”，追求类似 Transfusion 的统一模型（课程 12.13）。同一个原生预训练骨干网络支持理解和生成两种头。

### 原生预训练的权衡

原生预训练并非没有代价：

- **计算**。从头训练一个新的 VLM 与训练一个文本 LLM 成本相当——数百万 GPU 小时。事后适配则复用现有的 LLM 权重，节省大部分成本。
- **数据**。大规模的交织图文语料稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。仅文本就达到 15T token。多模态预训练数据稀缺是一个硬约束。
- **基础 LLM 复用**。原生预训练放弃了以后更换新 LLM 的选项。事后方法允许你只重训适配器就能用 Llama-3.1 替换 Llama-4。

InternVL3 的赌注是：对齐债务比复用损失更严重。基准测试支持了这一观点。生产成本的门槛阻止了未来实验室的廉价复制。事后 VLM 将继续存在，因为它们对大多数项目来说仍然更便宜。

## 使用它

`code/main.py` 是一个训练语料混合器和 ViR 路由模拟器。它：

- 接收目标语料混合比例（%文本，%交织，%图文描述，%视频），并计算每种模态的预期步数。
- 在一个查询批次上模拟 ViR 路由（分布：50% 低细节，30% 中等，20% 高细节）并报告平均 token 数。
- 根据编码器与 LLM 的 FLOPs 报告 DvD 吞吐量估计。
- 打印事后与原生预训练在参数、计算、数据和预期对齐债务症状方面的对比表。

## 交付它

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个拟定的 VLM 训练计划，它将审计是采用原生还是事后方案，标记对齐债务风险，并推荐语料混合比例。当你在规划一个新的开放 VLM 项目并需要选择训练策略时，请使用它。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后）之间的计算量差异。GPU 小时的大致比例是多少？是什么原因造成了这个差距？
2. InternVL3 报告的比例是 40% 文本 / 35% 交织 / 20% 图文描述 / 5% 视频。如果你的目标任务是视频密集型的，提出一个新的比例，并论证为什么基础模型仍然需要大量的文本和图文描述数据。
3. 阅读 MM1.5 第 4 节关于遗忘的部分。说出事后训练显示出最大回退的具体基准是什么。回退损失了多少？
4. ViR 将 60% 的流量路由到低分辨率编码。它会错误路由（当需要高分辨率时却发送到低分辨率）哪些类型的查询？提出三种路由器故障模式。
5. DvD 将视觉和 LLM 分离到不同的 GPU 上。在什么流量模式下，DvD 会损害吞吐量而不是帮助？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|------------|----------|
| 原生多模态预训练 | "从头开始一起训练" | 文本 + 图像 + 视频 token 从第 1 步就参与损失计算，而不是事后附加 |
| 对齐债务 | "事后惩罚" | 因将视觉能力附加到冻结的 LLM 上而导致的可衡量的文本技能和答案一致性回退 |
| V2PE | "可变视觉位置编码" | 按模态可学习的位置编码分配；InternVL3 的 M-RoPE 后继者 |
| ViR | "分辨率路由器" | 在编码前为每个查询选择最小所需分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一个 GPU，LLM 在另一个，带流式传输；为大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年的后续版本，在原生预训练骨干上添加图像生成头 |
| 交织语料 | "OBELICS / MMC4" | 按自然阅读顺序包含文本和图像的文档；原生预训练的原材料 |
