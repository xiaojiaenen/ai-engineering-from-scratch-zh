# 合规性 — SOC 2, HIPAA, GDPR, PCI-DSS, EU AI Act, ISO 42001

> 多框架覆盖是2026年企业交易的准入门槛。**EU AI Act**：自2024年8月1日生效。大多数高风险要求将于2026年8月2日强制执行。违反高风险系统义务（第99(4)条）的罚款高达€1500万或全球年营业额的3%；违反禁止性AI行为（第99(3)条）的罚款高达€3500万或7%。如果服务于欧盟用户，则全球适用。**Colorado AI Act**：生效日期为2026年6月30日（由SB25B-004从2026年2月推迟）——要求对高风险系统进行影响评估，以及对AI决策提出上诉的权利。弗吉尼亚州在信贷/就业/住房/教育领域有类似规定。**SOC 2 Type II**：事实上的B2B AI要求（金融科技领域需Type II，而非Type I）。**GDPR**：有记录以来最大的AI专项罚款是针对Clearview AI的€3050万（荷兰DPA，2024年9月）；意大利的Garante于2024年12月对OpenAI处以€1500万罚款（后于2026年3月上诉后被推翻）。推理阶段的实时PII编辑是可辩护的标准；事后清理处理并不足够。**HIPAA**：医疗保健领域强制要求——没有BAA不能将PHI发送到外部AI服务。**PCI-DSS**：AI交互层覆盖需要配置加合同协议，并非自动获得。**ISO 42001**：新兴的AI治理标准，正与ISO 27001一起成为日益增长的采购要求。参考概况：OpenAI维护SOC 2 Type 2、ISO/IEC 27001:2022、ISO/IEC 27701:2019、GDPR/CCPA/HIPAA (BAA)/FERPA，以及ChatGPT支付组件的PCI-DSS。跨框架映射可减少审计负担：访问控制映射跨越ISO 27001 A.5.15-5.18、GDPR第32条、HIPAA §164.312(a)。

## 问题所在

一家企业客户的采购部门要求提供SOC 2 Type II、GDPR、HIPAA BAA、ISO 27001以及"EU AI Act合规声明"。你的团队目前有SOC 2 Type I。你们距离获得Type II还有六个月，并且还没有开始准备GDPR第30条要求的记录。

多框架覆盖不是LLM特有问题——它是企业SaaS问题，附带有LLM特定覆盖层。2026年的采购团队想要的是一个矩阵（行对应框架，列对应控制措施），而不是一份PDF。

## 核心概念

### 七个框架

| 框架 | 范围 | LLM特定要求 |
|-----------|-------|--------------------------|
| SOC 2 Type II | B2B SaaS基线 | 流程控制经过6-12个月审计 |
| HIPAA | 美国医疗保健 | 需要BAA；PHI离开基础设施必须签署协议 |
| GDPR | 欧盟用户 | 实时PII编辑；数据主体权利；第30条记录 |
| PCI-DSS | 支付数据 | AI接触支付需配置加合同 |
| EU AI Act | 服务欧盟用户 | 风险等级分类；高风险系统：合规性评估、文档记录、日志记录 |
| Colorado AI Act | 服务科罗拉多州居民 | 影响评估；上诉权 |
| ISO 42001 | AI治理 | 新兴；与ISO 27001配对 |

### EU AI Act时间线

- 2024年8月1日：生效。
- 2025年2月2日：禁止性AI行为开始执行。
- 2026年8月2日：高风险系统开始执行（合规性评估、文档记录、日志记录）。
- 2027年8月：协调立法下产品中的高风险系统。

风险等级：不可接受（禁止）、高风险（合规+日志记录）、有限风险（透明度）、最小风险（无约束）。大多数B2B LLM SaaS属于有限风险；在就业、信贷、教育、执法、移民、基本服务等领域应用时，升级为高风险。

罚款（第99条）：违反高风险系统义务（第99(4)条）最高€1500万或全球年营业额的3%；违反禁止性AI行为（第99(3)条）最高€3500万或7%；适用较高者。

### GDPR — 实时编辑是标准

事后清理处理（在LLM看到数据后编辑PII）不是可辩护的立场——模型已经看到了数据。实时推理层编辑是2026年的标准：

- 在LLM调用前进行实体识别。
- 一致的词元化（Mesh方法）保留语义。
- 仅存储编辑后的提示词 + 经同意选择加入的原始数据。

近期执法：针对Clearview AI的€3050万罚款（荷兰DPA，2024年9月）是迄今记录在案最大的AI专项GDPR罚款；针对OpenAI的€1500万罚款（意大利Garante，2024年12月）是最大的LLM专项罚款，尽管该罚款于2026年3月上诉后被推翻，判决仍在进一步审查中。事后处理的声明在审计中未能通过。

### HIPAA — BAA不可或缺

没有签署《业务伙伴协议》，您不能将PHI发送到外部AI服务。所有三大超大规模云提供商的LLM平台（Bedrock、Azure OpenAI、Vertex）都提供BAA。OpenAI直接API提供BAA。Anthropic直接API提供BAA。发送PHI前请确认。

### SOC 2 Type II

Type I：控制措施已设计并记录。
Type II：控制措施在6-12个月内有效运行。

2026年的B2B采购默认要求Type II。Type I是入门；Type II是门槛。

常见审计驱动因素：访问日志（谁看了什么）、变更管理（如何部署）、风险评估（每季度）、事件响应（是否测试过？）。第17阶段 · 25的审计日志可直接复用。

### 跨框架映射

一项访问控制策略满足多个框架控制要求：

| 控制措施 | 框架 |
|---------|-----------|
| 访问日志记录 | ISO 27001 A.5.15-5.18, GDPR第32条, HIPAA §164.312(a) |
| 变更管理 | ISO 27001 A.8.32, PCI DSS 要求6, HIPAA 违规通知范围 |
| 传输中加密 | ISO 27001 A.8.24, GDPR第32条, HIPAA §164.312(e) |
| 密钥管理 | ISO 27001 A.8.19, PCI DSS 要求8, SOC 2 CC6.1 |

合规工具（Drata、Vanta、Secureframe）可自动化此映射。在规模上物有所值。

### ISO 42001 — 新兴标准

发布于2023年末。正与ISO 27001一起成为日益增长的采购要求。这是一个AI治理框架，包括风险管理、数据质量、透明度、人工监督。

### OpenAI的参考概况

OpenAI维护SOC 2 Type 2、ISO/IEC 27001:2022、ISO/IEC 27701:2019、GDPR/CCPA/HIPAA (BAA)/FERPA，以及ChatGPT支付组件的PCI-DSS。这大致是2026年的企业准入门槛。

### 你应记住的数字

- EU AI Act罚款：高风险义务（第99(4)条）最高€1500万/3%；禁止性行为（第99(3)条）最高€3500万/7%。
- EU AI Act高风险执行日期：2026年8月2日。
- 有记录以来最大的AI专项GDPR罚款：€3050万，Clearview AI（荷兰DPA，2024年9月）。
- 最大的LLM专项GDPR罚款：€1500万，OpenAI（意大利Garante，2024年12月；2026年3月上诉后被推翻）。
- SOC 2 Type II窗口期：控制措施需运行6-12个月。
- Colorado AI Act生效日期：2026年6月30日（由SB25B-004从2026年2月推迟）。
