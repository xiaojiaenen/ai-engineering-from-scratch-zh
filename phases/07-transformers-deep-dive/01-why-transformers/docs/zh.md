# 为何选择 Transformer —— RNN 的困境

> RNN 逐个处理 token，Transformer 则一次性处理所有 token。这个单一的架构决策改变了 2017 年后深度学习的每一条扩展曲线。

**类型：** 学习  
**语言：** Python  
**先修知识：** 第三阶段（深度学习核心）、第五阶段 · 09（序列到序列）、第五阶段 · 10（注意力机制）  
**时间：** 约 45 分钟

## 问题所在

2017 年之前，全球所有最先进的序列模型——无论是用于语言、翻译还是语音——都是循环神经网络。LSTM 和 GRU 在长达五年的时间里赢得了等同于 ImageNet 级别的翻译基准测试。它们是当时唯一可用的工具。

它们有三个致命弱点。顺序计算意味着你无法沿时间轴进行并行化：token `t+1` 需要来自 token `t` 的隐藏状态。一个 1,024 个 token 的序列意味着在每周期可进行 1,000,000 次浮点运算的 GPU 上需要 1,024 个串行步骤。在设计用于并行的硬件上，训练时间随序列长度线性增长。

梯度消失意味着 50 个 token 之前的信息已经通过 50 次非线性压缩。门控循环单元（LSTM、GRU）缓解了这种挤压，但从未彻底消除它。长距离依赖关系——比如“我去年夏天在去京都的飞机上读的那本书……”——通常会失效。

固定宽度的隐藏状态意味着编码器在解码器看到任何内容之前，就将整个源序列压缩成一个单一向量。无论源序列是 5 个 token 还是 500 个 token，瓶颈的形状都是一样的。

2017 年的论文《Attention Is All You Need》提出了一个激进的想法：完全抛弃循环。让每个位置并行地关注其他所有位置。用一个大型矩阵乘法代替 1,024 个顺序乘法进行训练。

到 2026 年，该结果主导了所有模态。语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人学（RT-2）。相同的模块，不同的输入。

## 核心概念

![RNN 顺序计算 vs Transformer 并行注意力](../assets/rnn-vs-transformer.svg)

**循环作为瓶颈。** RNN 计算 `h_t = f(h_{t-1}, x_t)`。每一步都依赖于前一步。在计算 `h_5` 之前，你无法计算 `h_4`。在拥有 10,000 多个并行核心的现代 GPU 上，对于长序列，这会浪费 99% 的硅片资源。

**注意力作为广播。** 自注意力为每对 `(i, j)` 同时计算 `output_i = sum_j(a_ij * v_j)`。整个 N×N 的注意力矩阵在一次批量矩阵乘法中完成。没有任何步骤依赖于其他步骤。GPU 非常适合这种计算。

**速度提升并非恒定。** 这是 `O(N)` 串行深度和 `O(1)` 串行深度之间的差异。实际上，在 N=512 的匹配硬件上，Transformer 每个 epoch 的训练速度快 5-10 倍，并且随着序列长度的增加，差距会扩大，直到触及注意力机制的 `O(N²)` 内存墙（Flash Attention 后来解决了这个问题——见第 12 课）。

**Transformer 的代价。** 注意力内存与 `O(N²)` 成正比。对于 2K 上下文来说没问题。对于 128K 上下文，你需要滑动窗口、RoPE 外推、Flash Attention 分块或线性注意力变体。循环在时间和内存上都是 `O(N)`；Transformer 用内存换时间，然后又通过并行性赢回了时间。

**归纳偏置的转变。** RNN 假设局部性和近期性。Transformer 不做任何假设——每对位置都是注意力的候选对象。这就是为什么 Transformer 需要更多数据才能训练好，但一旦拥有数据就能扩展得更远。Chinchilla（2022）对此进行了形式化：给定足够的 token，Transformer 总能击败参数数量相同的 RNN。

## 动手构建

这里没有神经网络——我们通过数值模拟核心瓶颈，让你在笔记本电脑上感受这种差距。

### 步骤 1：测量串行深度

见 `code/main.py`。我们构建两个函数。一个将序列表示为加法链（串行，类似 RNN）。另一个将其表示为并行归约（广播，类似注意力）。相同的数学运算，不同的依赖图。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # can't parallelize: h depends on previous h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # every x is independent
```

我们对长度高达 100,000 的序列计时。RNN 版本是 O(N) 且只使用单个 CPU 流水线。即使在纯 Python 中，当长度 ≥ 1,000 时，注意力风格的归约也能胜过它，因为 Python 的 `sum()` 是用 C 实现的，并且每次迭代都没有解释器开销。

### 步骤 2：计算理论运算量

两种算法都进行 N 次加法。区别在于*依赖深度*：在下一步可以开始之前，必须顺序完成多少个运算。RNN 深度 = N。使用树形归约时注意力深度 = log(N)，或使用并行扫描时为 1。决定 GPU 时间的是深度，而不是运算次数。

### 步骤 3：长序列上的经验缩放

我们打印一个计时表，使 O(N) 的差距清晰可见。在 2026 年的 Mac 笔记本电脑上，长度低于 1,000 的序列太快，难以测量。长度为 100,000 的序列显示出清晰的线性扫描。将此扩展到拥有 12 层 LSTM 等效模型的 16,384 个 token 的 Transformer，你就会明白为什么在 2016 年训练时间是阻碍因素。

## 何时使用

在 2026 年，何时仍然选择 RNN：

| 场景 | 选择 |
|------|------|
| 流式推理，每次处理一个 token，内存恒定 | RNN 或状态空间模型（Mamba、RWKV） |
| 超长序列（>1M token），注意力内存爆炸 | 线性注意力、Mamba 2、Hyena |
| 没有矩阵乘法加速器的边缘设备 | 深度可分离 RNN 在 FLOPs/瓦特上仍占优 |
| 其他任何情况（训练、批量推理、上下文长度达 128K） | Transformer |

状态空间模型（SSMs）如 Mamba 本质上是具有结构化参数化的 RNN，这赋予了它们两者兼备的优势：`O(N)` 扫描内存，通过选择性扫描实现并行训练。它们以更好的长上下文扩展性，恢复了 Transformer 90% 的质量。在 2026 年，大多数前沿实验室训练混合 SSM+Transformer 模型（例如 Jamba、Samba）——循环并未消亡，它已成为一个组件。

## 练习

1.  **简单。** 从 `code/main.py` 中取 `rnn_style`，并将标量隐藏状态替换为长度为 64 的隐藏状态向量。重新测量。串行开销随隐藏状态维度的增长情况如何？
2.  **中等。** 用纯 Python 实现一个并行前缀和（Hillis-Steele 扫描）。在长度为 1024 的序列上验证其输出与串行扫描的数值结果相同。计算深度。
3.  **困难。** 将注意力风格的归约移植到 PyTorch GPU 上。在序列长度从 64 到 65,536 进行扫描时，对两种方法计时。绘制曲线图并解释其形状。

## 关键术语

| 术语 | 人们常说什么 | 实际含义 |
|------|-------------|---------|
| 循环 (Recurrence) | "RNN 是顺序的" | 计算中步骤 `t` 依赖于步骤 `t-1`，迫使沿时间轴串行执行。 |
| 串行深度 (Serial depth) | "计算图有多深" | 依赖运算的最长链；即使在无限硬件上也限制实际时间。 |
| 注意力 (Attention) | "让 token 相互查看" | 加权求和 `sum_j a_ij v_j`，其中 `a_ij` 来自位置 i 和 j 之间的相似度分数。 |
| 上下文窗口 (Context window) | "模型能看到多少" | 注意力层可以作为输入的位置数量；二次方内存成本在此处缩放。 |
| 归纳偏置 (Inductive bias) | "架构内置的假设" | 关于数据形态的先验知识；CNN 假设平移不变性，RNN 假设近期性。 |
| 状态空间模型 (State-space model) | "背后的代数结构是 RNN" | 通过结构化状态空间矩阵实现并行训练的循环参数化。 |
| 二次瓶颈 (Quadratic bottleneck) | "上下文成本高昂的原因" | 注意力内存 = `O(N²)`（序列长度的二次方）；Flash Attention 隐藏的是常数，而非缩放关系。 |
