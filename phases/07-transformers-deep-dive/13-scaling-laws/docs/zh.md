# 缩放定律

> 卡普兰等人在2020年的论文中指出：模型越大，损失越低。而霍夫曼等人在2022年的论文则指出：你们之前的训练都不充分。计算资源主要分配在两个方面——参数和训练数据——但最优的分配比例并不明显。

**类型：** 学习  
**语言：** Python  
**先决条件：** 第7阶段 · 05（完整Transformer），第7阶段 · 07（GPT）  
**时间：** 约45分钟

## 问题所在

当你拥有 C 浮点运算数（FLOPs）的训练算力，并希望获得最佳模型时，你面临两个关键选择：

1.  **模型参数量（N）？** 模型越大，容量越高。
2.  **训练数据量（D）？** 数据越多，越能充分利用模型容量。

FLOPs 大致按 `6 × N × D` 缩放。你可以提高 N 并降低 D，或者提高 D 并降低 N。哪种方式更好？

在2022年之前，答案是“大幅提升 N”。GPT-3（2020年）拥有1750亿参数，使用约3000亿个token进行训练。参数与token的比例大约是1:1.7。卡普兰的缩放定律支持这一做法。

然而，霍夫曼等人（2022年）在训练一个名为 Chinchilla 的小规模模型家族时，发现了不同的结论：最优比例更接近 **每参数20个token**。GPT-3 的训练量实际不足了10倍。Chinchilla（700亿参数，1.4万亿token）在所有基准测试中都击败了 GPT-3（1750亿参数，3000亿token），而推理成本仅为后者的2.5倍。

2026年是 Chinchilla 的天下——但有一个重要变化。Llama 3 8B 在15万亿个token上训练，参数与token的比例达到了1:1875。这比 Chinchilla 最优比例高出94倍。对于大规模使用的模型，推理成本比训练成本更重要，因此，为了获得一个更小、更易于部署的模型，进行过度训练（超出 Chinchilla 最优范围）已成为2026年的默认选择。

## 核心概念

![Chinchilla 曲线：不同 N/D 比例下损失与计算量的关系](../assets/scaling-laws.svg)

### 霍夫曼定律

根据 Chinchilla 论文，损失遵循以下公式：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数量（不含嵌入层）。
- `D` = 训练 token 数。
- `α ≈ 0.34`, `β ≈ 0.28` （大致对称）。
- `E ≈ 1.69`，不可约损失上限。
- `A ≈ 406`, `B ≈ 411`。

随着规模的扩大，这两个项相互制约。在固定计算量（C = 6ND）下对 `N` 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

计算最优：每参数20个token。

### 为何仍需过度训练

Chinchilla 最优方案旨在最小化每训练 FLOP 的训练损失。但训练成本是一次性的，而推理成本是持续性的。

对于一个每月处理万亿 token 的聊天机器人，推理成本主导了总成本。Llama 的做法是：训练更小的模型，但用更长的时间。8B参数、15万亿token 的方案是深度优化推理的：

-   可部署在消费级 GPU 上。
-   延迟仅为70B Chinchilla 最优模型的几分之一。
-   在大多数任务上质量足够接近。

DeepMind 在2024年的论文（“过度训练成为新的最优”）对此进行了形式化。对于以推理为主的工作负载，根据服务量的不同，最佳比例更接近每参数100-500个token。

### 涌现能力 vs. 平滑性

声称：某些能力（算术、多步推理、遵循思维链）会在某个规模点“突然涌现”。

Schaeffer 等人（2023年）认为这是一种测量假象：所谓的涌现指标使用了不连续的评分方式（精确匹配、准确率阈值），这掩盖了底层 logits 的平滑改进。而使用连续指标（交叉熵）则会呈现平滑的曲线。

到2026年，共识已形成：通过连续损失进行的预测是可靠的。基准测试的跃升往往是评分器造成的结果。应依据连续指标来规划预算。

### 2026年的新图景

缩放定律依然有效，但情况有所变化：

| 因素 | 如何改变 |
|------|----------|
| 数据质量 | 整理“优质” token（如 Phi 风格）可使等效计算量提升超过2倍 |
| MoE | 总参数量与活跃 FLOPs 解耦；缩放定律基于每活跃 FLOP 计算 |
| 后训练 | 某些能力（指令遵循、代码）通过 SFT+RLHF 的提升幅度大于预训练 |
| 多模态 | 图像与文本 token 共同缩放；每种模态有独立的曲线 |
| 合成数据 | 模型生成训练数据；有效计算量可复合增长 |

Muon 优化器（Kimi Moonlight，2024）展示了在匹配数据下比 AdamW 高出约2倍的有效计算量收益。一些2026年的训练运行默认使用 Muon。这改变了缩放定律中的绝对常数，但没有改变其形式。

## 动手实现

参见 `code/main.py`。我们实现了 Chinchilla 损失方程，并在多个计算预算下求解计算最优的 `(N, D)`。

### 步骤 1：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

绘制 `L` 关于 `(N, D)` 在固定 `C = 6ND` 下的等高线图。找到最小值。

### 步骤 2：计算最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的计算预算，找到在约束 `6ND = C` 下使损失最小化的 `(N, D)`。验证比例 `D/N ≈ 20`。

### 步骤 3：过度训练成本

计算使用10倍更小模型（最优 N 的1/10，最优 D 的10倍）训练所产生的额外损失。报告为此节省的推理 FLOP（与 N 成正比）。

### 步骤 4：与真实模型对比

代入已知的 `(N, D)` 对，包括 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数），比较预测损失与报告损失。

## 学以致用

你不太可能自己去训练一个前沿模型。但缩放定律告诉你：

1.  **你的微调数据是否足够。** 如果你的任务特定数据少于基础模型每参数20个token，预计损失会停滞在某个下限。
2.  **是否应选择更大的基础模型。** 如果你将全部预算用于推理，应优先选择更小、训练更久的模型。
3.  **收益递减点在哪里。** 超过 Chinchilla 最优比例1000倍后，对数损失的变化将变为噪音。

**2026年的研究轨迹：**

-   **数据受限环境。** 高质量 token 的数量是有限的（过滤后约5-10万亿英文 token）。前沿预训练正在逼近这个上限。合成数据、多语言、多模态和 RLHF 规模化的微调是接下来的杠杆。
-   **计算量增益技巧。** MoE、Muon 优化器、更好的数据整理——每种都改变了绝对常数，但没有改变渐近线。
-   **RL 的缩放定律。** 这是个开放性问题。早期证据表明在强化学习样本中存在幂律，但其指数与预训练截然不同。

## 关键术语

| 术语 | 通俗说法 | 精确含义 |
|------|----------|----------|
| 参数量 (N) | “模型大小” | 非嵌入层的权重数量；决定了模型容量。 |
| Token 数 (D) | “训练数据” | 训练中使用的 token 总数；决定了参数的利用程度。 |
| 计算量 (C) | “消耗的算力” | 对于标准 Transformer，近似为 `6 × N × D`。 |
| Chinchilla 最优 | “D/N ≈ 20” | 最小化预训练每 FLOP 损失的比率。 |
| 过度训练 | “超越 Chinchilla” | 投入额外的训练 FLOP 以节省推理 FLOP；D/N >> 20。 |
| 不可约损失 | “下限” | 缩放定律中的 `E` 项；数据本身的熵。 |
| 涌现能力 | “规模点的跃升” | 通常是评分器假象；连续损失是平滑的。 |
| 有效计算量 | “训练效率乘数” | 更好的数据/优化器/架构能使每 FLOP 发挥更大效用。 |
