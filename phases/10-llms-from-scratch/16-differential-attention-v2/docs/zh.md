# 差分注意力 (V2)

> Softmax注意力会将少量概率分散到每一个非匹配token上。在超过10万个token时，这种噪声会累积并淹没信号。差分Transformer (Ye et al., ICLR 2025) 通过计算两个softmax的差值来修复此问题，减去了共享的噪声下限。DIFF V2 (Microsoft, 2026年1月) 是生产级的重写版本：将解码延迟匹配到基线Transformer，无需自定义内核，兼容FlashAttention。本课程将从V1到V2进行端到端讲解，并包含一个可在标准Python库中运行的差分操作的可用示例实现。

**类型：** 构建
**语言：** Python (标准库)
**前提条件：** Phase 7 · 02 (自注意力)，Phase 7 · 15 (注意力变体)，Phase 10 · 14 (架构详解)
**时间：** 约60分钟

## 学习目标

- 精确说明为什么softmax注意力存在噪声下限，以及它为什么随上下文长度增长而增长。
- 推导差分注意力公式，并解释为什么相减操作能在保留信号的同时抵消共享的噪声分量。
- 详细讲解V1到V2的差异：哪些方面变快了，哪些变简单了，哪些更稳定了，以及为什么每个改变对于生产预训练都是必要的。
- 在纯Python中从头实现差分注意力，并通过合成信号加噪声查询实验验证其噪声抵消特性。

## 问题所在

标准softmax注意力具有一个在规模化时会演变成操作难题的数学特性。对于一个查询`q`，注意力权重为`softmax(qK^T / sqrt(d))`。Softmax永远无法产生精确的零——每个非匹配token都会获得一些正的质量。这些残余质量就是噪声，它随上下文长度缩放。在128k个token时，即使每个非匹配token只获得0.001%的概率，其中127,999个的总和也会贡献大约12%的总概率。模型必须学会绕过这个随上下文增长的噪声下限。

经验上，这表现为注意力头的干扰：长上下文RAG中的幻觉引用、100k-token检索任务中的“中间丢失”失败，以及超过32k后“大海捞针”基准测试中的微妙准确率下降。差分Transformer论文 (arXiv:2410.05258, ICLR 2025) 测量了这一差距：DIFF Transformer比相同规模的基线模型达到了更低的困惑度、更高的长上下文准确率和更少的幻觉。

DIFF V1存在三个问题，阻碍了其进入前沿预训练流程。它的值缓存需要在每个解码步骤加载两次，它需要自定义CUDA内核从而破坏了FlashAttention兼容性，并且其逐头RMSNorm在70B以上规模的长期训练中不稳定。DIFF V2 (Microsoft unilm博客，2026年1月20日) 修复了所有这三个问题。本课程将讲解两个版本，构建差分算子，并在玩具查询上对噪声抵消进行基准测试。

## 概念

### Softmax的噪声下限

对于一个查询`q`和键`K = [k_1, ..., k_N]`，注意力权重为：

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

没有`w_i`会是零。如果`k_i`与`q`完全无关，得分`q . k_i`也不是0——它在零附近波动，方差为`||q||^2 / d`。经过softmax归一化后，每个不相关的token仍然对加权和贡献`O(1/N)`。不相关token的总贡献是`O((N-1)/N) = O(1)`——这并非一个小数目。

模型想要的是类似于硬性top-k的机制：对匹配的token赋予高权重，对其他所有位置赋予接近零的权重。Softmax过于平滑，无法直接做到这一点。

### 差分思想

将每个头的Q和K投影分成两部分：Q = (Q_1, Q_2) 和 K = (K_1, K_2)。计算两个注意力图：

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

输出：

```
DiffAttn = (A_1 - lambda * A_2) V
```

相减操作抵消了两个注意力图共享的任何噪声分布。如果两个注意力图在127k个不相关token上大致具有均匀的权重（在随机初始化时它们确实会如此），那么这些噪声会被抵消。信号——在少数真正相关token上出现的峰值权重——只有在以相同幅度出现在两个注意力图中时才会被抵消，而这在模型训练后不会发生。

`lambda`是每个头一个可学习的标量，参数化为`lambda = exp(lambda_q1 dot lambda_k1) - exp(lambda_q2 dot lambda_k2) + lambda_init`。它可以是负数。`lambda_init`默认是一个较小的正数，如0.8。

### 为什么这符合定向降噪

想象两个嘈杂的麦克风录制同一个声音。两者都拾取了说话者加上相关的背景噪声。将一个从另一个中减去，共享的噪声就消失了。声音保留下来，因为两个信号在相位或幅度上的差异足以防止完全抵消。每个头的`lambda`学习的正是这种平衡。

### V1与V2：差异

V1保持了与基线Transformer相同的参数量。为了每个头获得两个查询，它将头维度减半。这损害了头的表达能力，并且更痛苦的是，每个头的值缓存也减半。解码时每个步骤必须加载两次值缓存（每个softmax分支一次）。结果：解码比基线慢，尽管参数量匹配。

V2将查询头的数量加倍，并保持KV头数量不变（从上投影中借用参数）。头维度与基线保持一致。相减后，额外的维度被投影回以匹配基线Transformer的O_W投影。三件事同时发生：

1.  解码速度匹配基线（KV缓存只加载一次）。
2.  FlashAttention无需更改即可运行（无需自定义内核）。
3.  解码时的算术强度提高（每从HBM加载一个字节有更多的计算）。

V2还移除了V1中用于稳定相减操作的逐头RMSNorm。在70B级别的预训练规模下，该RMSNorm在后期训练中不稳定。V2用一个更简单的初始化方案取代了它，该方案能在没有额外模块的情况下保持训练稳定。

### 何时采用它

| 工作负载 | 益处 |
|----------|------|
| 长上下文RAG (64k+) | 更干净的注意力图，更少的幻觉引用 |
| 大海捞针基准测试 | 超过32k后准确率大幅提升 |
| 多文档问答 | 跨文档干扰减少 |
| 8k代码补全 | 边际效益，不值得更改架构 |
| 短对话 (< 4k) | 与基线几乎无法区分 |

收益随上下文长度增长。在4k个token时，噪声下限足够小，标准注意力仍表现良好。在128k时，它会损害性能。

### 它如何与2026年的其他调整项配合

| 特性 | 与DIFF V2兼容吗？ |
|---------|-------------------|
| GQA | 是（V2增加了Q头，而非KV头） |
| MLA (DeepSeek) | 原则上是，尚无已发表的结合论文 |
| MoE | 是（注意力与MLP块独立） |
| RoPE | 是（未改变） |
| YaRN / 长上下文缩放 | 是（正是DIFF帮助最大的地方） |
| FlashAttention | 在V2中是（在V1中不是） |
| 推测解码 | 是（注意力更改对推测解码循环不可见） |

## 构建它

`code/main.py`在纯Python中实现了差分注意力。一个具有已知信号加噪声结构的玩具查询，让你可以直接测量噪声抵消率。

### 步骤1：标准softmax注意力

标准库矩阵运算：列表的列表，手动矩阵乘法，带数值稳定性最大值减法的softmax。

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### 步骤2：将Q, K分成两半

V1风格：将头维度减半。V2风格：保持头维度并增加头数。玩具实现使用V1以求教学清晰——数学原理相同，只是记账方式不同。

### 步骤3：两个softmax分支 + 相减

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

注意：输出权重可以是负数。这是可以接受的——值缓存仍然处理有符号的贡献。随后的V投影会吸收符号。

### 步骤4：噪声抵消测量

构建一个长度为1024的合成序列。将信号token放在已知位置，其余部分用噪声填充。计算(a) 标准softmax注意力对信号位置的权重，和(b) 差分注意力权重。测量每种注意力中的信噪比。差分注意力可靠地产生更高的信噪比，高出3倍-10倍，具体取决于两个分支被训练得有多不同。

### 步骤5：V1与V2参数核算

给定一个配置 (hidden=4096, heads=32, d_head=128)，打印：

- 基线Transformer：Q、K、V各大小为`hidden * hidden`，MLP为 4 * hidden。
- DIFF V1：Q、K各大小为`hidden * hidden`，V大小为`hidden * hidden`（不变），头维度内部减半。增加逐头`lambda`参数（O(heads * d_head)）。
- DIFF V2：Q大小为`2 * hidden * hidden`，K大小为`hidden * hidden`，V大小为`hidden * hidden`。在O_W之前将额外维度投影回。增加相同的`lambda`参数。

玩具测量了V2的额外参数成本（大约每个注意力块`hidden * hidden`）并打印出来。

## 使用它

截至2026年4月，DIFF V2尚未在所有生产推理服务器中部署，但正在vLLM和SGLang中进行集成。同时，该模式出现在：

- Microsoft内部的长上下文生产模型。
- 在几个针对256k及以上上下文的开源模型训练运行中的研究复现。
- 混合架构，在交替层上结合DIFF注意力与滑动窗口注意力。

2026年何时你会采用它：

- 从头训练一个针对64k及以上有效上下文的新模型。从一开始就添加差分注意力；后期重新训练代价高昂。
- 微调一个长上下文模型，其中“中间丢失”失败主导了你的评估。对Q投影进行LoRA可以近似DIFF结构。

何时你不会采用：

- 你正在服务一个具有稳定长上下文性能的预训练密集模型。在现有权重上重新训练的成本很少能收回。
- 你的上下文始终低于16k。噪声下限可以忽略不计。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|------|------------|--------------|
| 差分注意力 | "两个softmax相减" | 将Q、K分成两半，计算两个softmax图，将第二个（乘以lambda）从第一个中减去，然后乘以V |
| 噪声下限 | "softmax的非零尾部" | softmax对每个不相关token施加的O(1/N)权重，在长上下文中总和为O(1) |
| lambda | "相减的缩放因子" | 每个头可学习的标量，参数化为`exp(lq1.lk1) - exp(lq2.lk2) + lambda_init`；可以是负数 |
| DIFF V1 | "ICLR 2025版本" | 原始差分Transformer；为保持参数量而将头维度减半，需要自定义内核，解码较慢 |
| DIFF V2 | "2026年1月修复版" | 在保持KV头的同时将Q头加倍；匹配基线解码速度，并兼容FlashAttention |
| 逐头RMSNorm | "V1的稳定器" | V1在差值后应用的额外归一化；V2移除它以防止后期训练不稳定 |
| 信噪比 | "注意力浪费了多少" | 真实信号位置上的权重与不相关位置上平均权重的比率 |
| 中间丢失 | "长上下文失败模式" | 经验现象，检索准确率在长上下文中间部分的文档上下降——DIFF注意力减少了这种现象 |
| 算术强度 | "每加载一个字节的FLOPs" | V2通过加倍每次KV加载的查询数在解码时提高了该比率；对于内存受限的解码很重要 |
