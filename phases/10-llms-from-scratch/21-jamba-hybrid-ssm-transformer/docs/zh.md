# Jamba — 混合SSM-Transformer架构

> 状态空间模型(SSMs)与Transformer追求的目标不同。Transformer通过注意力机制以二次成本换取质量。SSMs通过递归实现线性时间推理和恒定内存消耗，但质量稍逊。AI21的Jamba（2024年3月）与Jamba 1.5（2024年8月）将两者置于同一模型中：每7个Mamba层配置1个Transformer层，每隔一个模块使用MoE，并拥有256k上下文窗口，可在单块80GB GPU上运行。Mamba-3（ICLR 2026）通过复值状态空间和MIMO投影优化了SSM端。本课程将深入解读这两种架构，并解释为何在纯SSM与纯Transformer的长上下文方案未能延续时，这种混合配方能经受三年的扩展考验。

**类型：** 学习
**语言：** Python（标准库，层混合计算器）
**前置条件：** 阶段10 · 14（开源模型架构），阶段10 · 17（原生稀疏注意力）
**时间：** 约60分钟

## 学习目标

- 解释Jamba模块中的三个基本组件——Transformer层、Mamba层、MoE——以及1:7:偶数层交错配方。
- 从高层描述SSM的递归形式，并解释其为何能实现恒定内存推理。
- 计算Jamba模型在256k上下文下的KV缓存占用，并与纯Transformer模型的需求进行比较。
- 列举Mamba-3的三项创新（指数梯形离散化、复值状态更新、MIMO）及各自针对的问题。

## 问题背景

注意力机制在序列长度上呈二次方复杂度。状态空间模型呈线性复杂度。这种差异在长序列中会累积：在256k token时，Transformer的注意力图每个头有650亿个条目；而SSM的递归状态大小固定，与序列长度无关。

纯SSM模型（Mamba、Mamba-2）在小规模上能匹配Transformer的困惑度，但在状态跟踪任务上表现滞后，且在某些上下文检索类别中失败。直觉上：SSMs将历史压缩到固定状态中，当历史较长时，信息会泄露。注意力机制能精确记住一切，但需付出二次成本。

显而易见的解决方案：两者结合。在精确召回至关重要的地方使用Transformer层。在其他地方使用SSM层。调整比例。Jamba是首个大规模投入生产的混合配方模型（总参数52B，激活参数12B，256k上下文，单块80GB GPU）。Jamba 1.5将系列扩展至总参数398B / 激活参数94B。Mamba-3（ICLR 2026）是当前最优的纯SSM基线，混合架构可围绕其重建。

本课程将解读所有三篇论文，并建立“选择正确比例”的思维模型。

## 核心概念

### 一页纸理解SSM

状态空间模型通过固定大小的状态`h`处理序列`x_1, ..., x_N`：

```
h_t = A h_{t-1} + B x_t
y_t = C h_t
```

在每一步，状态通过线性动力学`A`演化，接收输入`B x_t`，并输出`C h_t`。`A, B, C`可学习。注意关键属性：计算`y_t`仅需`h_{t-1}`和`x_t`，而非任何更早的`x`。内存恒定。推理时每个token的复杂度为O(1)。

建模质量的关键在于`A`的结构。S4（Gu 2021）使用高度结构化的矩阵，在训练期间可作为长卷积高效评估。Mamba（Gu, Dao 2023）用数据相关的`A, B, C`替代了固定的（即“选择性”部分）。Mamba-2（2024）进一步简化了结构。Mamba-3（2026）则在特定位置重新引入了复杂性。

关键属性：对于解码器LLM，SSM层可直接替代注意力层，其每层状态大小固定，而非像KV缓存那样增长。

### Jamba模块

Jamba模块根据两个数字交错层：

- `l`：注意力与Mamba的比例。Jamba使用`l = 8`，即每7个Mamba层配置1个Transformer层（7个Mamba + 1个注意力 = 每组8层）。
- `e`：MoE频率。Jamba使用`e = 2`，即每隔一个层应用MoE。

一个模块内的层序列：

```
M  M  M  M  M  M  M  A    (7 Mamba + 1 Attention)
|  M  |  M  |  M  |  M    (where | marks MoE applied)
```

每个Jamba模块有8层。深度为4个模块（共32层），你会得到28个Mamba层和4个注意力层。其中16层使用MoE。

### 为何是1:7比例

AI21进行了消融实验：注意力与Mamba的比例在他们的长上下文评估中，能给出最佳的每参数困惑度和上下文内召回率？

- 注意力过多（1:1）：质量提升，但内存和速度下降。
- 注意力过少（1:15）：内存很好，但上下文内检索失败。
- 甜蜜点：1:7或1:8。

直觉上：Transformer层处理精确召回和状态跟踪。Mamba层处理廉价的大量计算。

### 位置编码

Mamba层本身通过递归具有位置感知能力。基于Mamba的早期混合模型中的注意力层未使用RoPE——SSM层提供了位置信息。Jamba 1.5为注意力层添加了RoPE以支持更长上下文的泛化，这是基于经验性长上下文评估的事后改进。

### 内存预算

以Jamba-1的配置为例（32层：28个Mamba + 4个注意力，隐藏维度4096，32个注意力头）：

- KV缓存（仅注意力层贡献）：在256k BF16下为`2 * 4 * 32 * 128 * 256k * 2 = 8.4 GB`。只有4个注意力层有贡献。
- SSM状态：`28 * hidden * state_size`每token前缀，但这是每层固定大小，不随序列长度缩放。典型的Mamba状态为每个特征16，隐藏维度4096：总计`28 * 4096 * 16 * 2 = 3.7 MB`。

与32层、相同隐藏维度、全多头注意力（32头）的纯Transformer相比：在256k BF16下为`2 * 32 * 32 * 128 * 256k * 2 = 128 GB`。KV缓存减少了8倍。即使与大多数2024模型使用的GQA(8)基线（`2 * 32 * 8 * 128 * 256k * 2 = 32 GB`）相比，Jamba的1:7混合架构在16GB时仍小2倍。

这就是AI21所说的“在单块80GB GPU上实现256k上下文”的含义。全MHA纯Transformer的KV缓存放不下；即使是GQA基线，也留不出空间给权重和激活值；而Jamba可以。

### Mamba-3：2026年的纯SSM基线

Mamba-3（ICLR 2026, arXiv:2603.15569）在纯SSM侧引入了三项创新：

1.  **指数梯形离散化。** 用更具表达力的递归替代了Mamba-2中的欧拉方法离散化。卷积类操作应用于核心递归内的状态-输入上，而非作为`x_t`上的外卷积。

2.  **复值状态更新。** 之前的Mamba版本将状态矩阵从复数（S4）简化为实数对角（Mamba），再到缩放单位阵（Mamba-2）。Mamba-3重新引入复值——相当于在状态上进行数据相关的旋转嵌入。这恢复了之前实值简化所损失的状态跟踪能力。

3.  **多输入多输出（MIMO）投影。** 使用矩阵值投影替代每特征标量投影。提升了建模能力和推理时硬件利用率，且不增加解码延迟。

在15亿参数规模下，Mamba-3相比Gated DeltaNet将平均下游准确率提升了0.6点；MIMO变体额外提升1.2点，总计提升1.8点。在相同状态大小下，Mamba-3以一半的状态大小匹配了Mamba-2。

Mamba-3尚未在大规模生产级混合模型中部署——但它显然是下一代Jamba级模型SSM侧的候选方案。

### 何时选择混合架构

混合架构在以下情况胜出：

- 上下文足够长，纯Transformer的KV缓存变得难以承受（64k以上）。
- 任务混合了短期结构（适合SSM）和长期召回（需要Transformer）。
- 希望部署在单GPU内存预算下，而仅Transformer的KV缓存放不下。

混合架构在以下情况不占优：

- 上下文较短（低于16k）。SSM的开销被浪费；纯Transformer即可。
- 任务需要全方位注意力（深度推理、多文档交叉引用）。混合架构中注意力层的稀疏性会带来伤害。
- 正在扩展至万亿参数前沿模型。纯Transformer + MLA + MoE（DeepSeek-V3风格）目前在能力竞赛中领先。

### 竞争格局

| 模型 | 家族 | 规模 | 独特主张 |
|-------|--------|------|-------------|
| Mamba-2 | 纯SSM | 3B | 线性时间，恒定内存 |
| Jamba | 混合 | 52B/12B | 在80GB上实现256k上下文 |
| Jamba 1.5 Large | 混合 | 398B/94B | 企业级长上下文 |
| Mamba-3 | 纯SSM | 1.5B（论文） | 恢复状态跟踪能力 |
| DeepSeek-V3 | 纯Transformer + MoE | 671B/37B | 前沿能力 |

2026年格局：纯Transformer MoE主导前沿领域，但混合架构占据256k以上上下文细分市场。Mamba-3的状态跟踪优势可能推动下一代混合架构的比例向更低（更多SSM，更少注意力）发展。

## 实践应用

`code/main.py`是一个用于混合架构的内存计算器。给定SSM-Transformer比例及隐藏尺寸/层配置，它能计算：

- 目标上下文下的KV缓存。
- SSM状态内存。
- 一系列模型形状在上下文N下的总内存。

计算器支持：

- 纯Transformer基线（KV缓存随N增长）。
- Jamba风格的1:7混合。
- 纯SSM（无KV缓存）。

数字直接来自Jamba-1和Jamba-1.5论文中已发布的形状，并对假设变体进行了外推。

实际部署的集成考虑：

- 大多数生产推理服务器（vLLM, SGLang）支持Jamba和Mamba。请检查具体版本。
- 在256k上下文下，Jamba的内存优势体现在并发请求吞吐量上。在相同显存下，你能比Transformer序列承载更多Jamba序列。
- Mamba-3作为独立模型尚未投入生产部署——目前为1.5B规模的研究预览版。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------------|------------------------|
| 状态空间模型（SSM） | “具有固定状态的递归” | 一个具有学习递归`h_t = A h_{t-1} + B x_t`的层；每个token恒定内存 |
| 选择性SSM | “Mamba的技巧” | 数据相关的A, B, C参数，赋予模型类似门控的选择性，且时间为线性 |
| 注意力与Mamba比例 | “有多少注意力层” | 在Jamba中，`l = 8`表示每7个Mamba层对应1个注意力层 |
| Jamba模块 | “8层组” | 一个注意力层 + 七个Mamba层 + 在交替位置使用MoE |
| SSM状态 | “隐藏缓冲区” | 替代Mamba层KV缓存的每层固定大小状态 |
| 256k上下文 | “Jamba的旗舰指标” | Jamba-1能在单块80GB GPU上运行的序列长度；同规模纯Transformer无法做到 |
| Mamba-3 | “2026纯SSM” | 当前最优纯SSM架构，具有复值状态 + MIMO；混合架构重建的基线 |
| MIMO | “多输入多输出” | Mamba-3创新，使用矩阵值投影而非每特征标量投影 |
| 指数梯形离散化 | “Mamba-3的递归” | 更具表达力的递归，囊括了Mamba-2的欧拉方法离散化 |
| 混合架构 | “混合注意力与SSM” | 任何交错Transformer和SSM层的模型；Jamba是生产原型 |
