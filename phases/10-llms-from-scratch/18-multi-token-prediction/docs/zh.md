# 多token预测（MTP）

> 从GPT-2到Llama 3的每个自回归LLM都在每个位置训练一个损失函数：预测下一个token。DeepSeek-V3在每个位置增加了第二个损失函数：预测其后的token。额外的140亿参数（在671B模型上）通过梯度流蒸馏回主模型，并且训练好的MTP头在推理时被重新用作投机解码草稿生成器，接受率超过80%。免费获得了1.8倍的生成吞吐量。本课程基于DeepSeek技术报告构建顺序MTP模块，计算损失函数和共享头参数布局，并解释为何MTP保留了因果链而Gloeckle等人的原始并行MTP破坏了它。

**语言：** Python（标准库）
**先决条件：** 第10阶段 · 04（预训练迷你GPT），第10阶段 · 15（投机解码）

## 学习目标

- 阐述MTP训练目标并推导跨预测深度的联合损失。

## 问题所在

下一个token预测是标准的LLM训练目标。每个隐藏状态都受监督以预测一个确切的事物：紧随其后的token。这是一个令人惊讶的弱信号。序列中的大部分信息延伸到单个token之外——结构、连贯性、事实准确性、算术流程。模型必须通过数万亿token积累许多单个token信号来学习这些内容。

MTP提出：如果每个隐藏状态同时受监督以预测多个未来token会怎样？Gloeckle等人（Meta，2024）表明这很有帮助。他们的实现是在主干网络之上放置几个独立的输出头，每个头预测不同的偏移。并行、简单，但这些头看到的是相同的隐藏状态，没有任何层次细化——并且预测之间没有因果链接，因此不能用于投机解码。

DeepSeek-V3（2024年12月）将MTP重新设计为顺序模块，在每个预测深度保持因果链。模型从`h_i^(0)`预测`t+1`，然后从一个新的隐藏状态`h_i^(1)`预测`t+2`，该状态将`h_i^(0)`与`E(t+1)`嵌入相结合，依此类推。每个深度都是其自己的小型Transformer块。共享嵌入和共享输出头将参数开销保持在适中水平。在DeepSeek-V3的规模下，MTP模块在主模型权重基础上额外增加了140亿参数。这2%的开销带来了更密集的训练信号，以及推理时现成的投机解码草稿。

本课程从头构建一个MTP模块和D深度损失。数学很简洁。实现大约150行代码。

## 核心概念

### 顺序MTP方案

DeepSeek-V3在主模型之上添加`D`个MTP模块。每个模块`k`（对应`k = 1..D`）预测深度`k`处的token——即，给定直到位置`i`的前缀，预测`t_{i+k}`。

模块`k`由以下部分组成：

- 一个Transformer块`T_k`，包含自己的注意力机制和MLP。
- 一个投影矩阵`M_k`，将前一深度的隐藏状态与下一深度真实token的嵌入相结合。
- 共享嵌入`E`（与主模型相同）。
- 共享输出头`Out`（与主模型相同）。

在训练期间，对于直到位置`i`的前缀，每个深度的隐藏状态为：

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

每个深度的预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

每个深度的损失是针对真实值`t_{i+k}`的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失为：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda`是一个小的加权因子——DeepSeek-V3在训练的前10%使用0.3，之后使用0.1。总训练损失为`L_main + L_MTP`。

### 为何是顺序而非并行

Gloeckle的原始并行MTP有D个输出头，每个头直接应用于`h_i^(0)`。每个头从相同的主干隐藏状态预测`t_{i+k}`。这样训练效果不错，但预测之间没有条件关系。你无法使用`head_1`的输出来辅助`head_2`——这些头并行工作。

DeepSeek-V3的顺序设计从`h_i^(k-1)`加上实际的下一个token嵌入`E(t_{i+k})`构建`h_i^(k)`。这保留了因果链：要预测`t_{i+k+1}`，深度`k+1`的模块会看到`t_{i+k}`处的内容。这在结构上与自回归解码器消费其自身输出的方式完全相同——使得MTP模块可以直接用作投机解码的草稿生成器。

在推理时：将`h_i^(k-1)`和草拟的`t_{i+k}`输入模块`k+1`，获得`t_{i+k+1}`的预测。重复此过程。这正是EAGLE风格的草稿生成方式，使用训练好的MTP模块作为草稿网络。DeepSeek-V3报告第一个MTP模块的接受率超过80%，速度提升约1.8倍。

### 参数核算

对于隐藏维度`h`、词汇表大小`V`的模型：

- 主模型：数十亿参数，加上一个大小为`V * h`的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享嵌入：复用主模型的嵌入。无额外参数。
- 每个MTP模块：
  - 投影层`M_k`：`(2h) * h = 2h^2`。
  - Transformer块`T_k`：注意力层（MHA为`4h^2`）加上MLP（对于SwiGLU，比例8/3时通常为`8h^2`）。每个块约`12h^2`。

每个模块的总额外参数：`~14h^2`。对于DeepSeek-V3的`h = 7168`，D=1个模块：理论上`~14 * 7168^2 = ~720M`参数。DeepSeek-V3报告为140亿——差异主要在于MTP模块中的专家层也采用了MoE结构。

### 投机解码的收益

在预训练期间，MTP模块会使训练速度降低约10%（更多前向计算，额外损失）。收益是两方面的：

1.  更密集的训练信号。每个隐藏状态看到D+1个监督目标。对MMLU、GSM8K、MATH、HumanEval的测量效果：DeepSeek-V3的消融实验显示一致提升几个百分点。
2.  推理时免费的投机解码草稿。MTP模块已被训练为预测接下来的几个token。将其重新用作草稿网络，可提供80%以上的接受率。在这一水平下，N=3或N=5的投机解码可带来1.8倍的吞吐量提升。10%的训练时间成本在首次运行推理时就能收回。

### 与EAGLE的关系

EAGLE在预训练之后单独训练一个小型草稿模型。MTP将草稿集成到预训练过程中。两种方法通过不同的流程达到相似的接受率：

| 维度 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 训练时间 | 预训练之后 | 预训练期间 |
| 与现有权重的向后兼容性 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2个Transformer层 | 1个Transformer块 + 投影层 |
| 接受率 | 0.88-0.92 | 深度1时为0.80+ |
| 超越加速的收益 | 仅投机解码 | 更密集的训练信号 + 加速 |

## 动手构建

`code/main.py`从头到尾构建一个MTP模块：共享嵌入、投影、Transformer块、共享输出头。然后在一个短的合成序列上计算每个深度的交叉熵损失，并按组件打印参数计数。使用32个token的玩具词汇表以保持数字可读。

### 步骤1：共享嵌入表

单个`vocab_size x hidden`表被主模型以及每个深度的每个MTP模块使用。不是第二个副本——完全是同一个张量。

### 步骤2：每个深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # concat along feature dim, then project down to hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # vector addition stand-in
    projected = matvec(M_k, concat)
    return projected
```

真实的DeepSeek-V3将这两个RMSNorm后的向量拼接成`[2h]`，并用`h x 2h`矩阵投影。为了保持标准库的简洁性，玩具版本使用向量加法。

### 步骤3：深度k的Transformer块

自注意力加MLP。在玩具版本中，一个单层线性注意力块和一个SwiGLU MLP在没有numpy的情况下保持结构可见。

### 步骤4：共享输出头

复用主模型的输出投影层。生成词汇表上的logits。

### 步骤5：每个深度的损失

将softmax(logits)与偏移`k`处的真实token计算交叉熵。使用`lambda / D`缩放因子跨深度聚合。

### 步骤6：参数核算

打印总参数计数、共享部分（嵌入、头）的计数以及每个模块的额外计数。显示MTP额外参数与主模型大小的比率。

## 如何使用

MTP已集成到DeepSeek-V3（2024年12月）和DeepSeek-R1系列中。在推理时：

- DeepSeek自己的服务栈开箱即用地将MTP模块作为投机解码器使用。
- 截至2026年4月，vLLM和SGLang已有针对DeepSeek-V3 MTP的集成路径。
- AMD的ROCm SGLang教程展示了一个特定的MTP投机解码配置，在V3检查点上测得1.8倍的速度提升。

在新的预训练运行中何时使用MTP：

- 你控制整个预训练流程，并希望获得更密集的训练信号。
- 你将大规模部署模型，并希望免费获得投机解码。
- 你的隐藏大小至少为4096。在1B规模下，开销的损害大于收益的帮助。

何时不使用：

- 微调现有的预训练稠密模型。MTP模块未被训练。
- 研究模型，需要干净的基线进行比较。MTP改变了架构。

## 关键术语

| 术语 | 人们怎么说 | 其实际含义 |
|------|----------------|------------------------|
| MTP模块 | “额外损失块” | 一个小型Transformer块加投影层，用于预测超前主模型`k`个位置的token |
| 预测深度 | “哪个偏移” | 整数`k`，使得模块`k`根据直到位置`i`的前缀预测`t_{i+k}` |
| 并行MTP | “Gloeckle风格” | D个独立的头作用于相同的主干隐藏状态，无条件链 |
| 顺序MTP | “DeepSeek-V3风格” | 每个模块以先前深度的隐藏状态加上下一个token的嵌入为条件；保留因果链 |
| 共享输出头 | “复用主头” | MTP模块调用主模型的语言模型头，而非单独的输出投影 |
| 共享嵌入 | “复用主表” | 各处使用相同的词汇嵌入表；无重复参数 |
| 投影矩阵M_k | “组合隐藏状态+下一个token” | 一个`h x 2h`线性层，将先前的隐藏状态和目标token嵌入折叠成下一深度的输入 |
| 联合损失L_MTP | “平均额外损失” | 各深度交叉熵损失的算术平均值，按`lambda`缩放 |
| 深度1接受率 | “MTP草稿正确的频率” | D=1 MTP模块的top-1预测与主模型的top-1预测相等的比率；DeepSeek-V3上为80%+ |
| Lambda加权 | “额外损失的重要性” | 每个深度的缩放因子；DeepSeek-V3在训练开始时为0.3，之后为0.1 |
