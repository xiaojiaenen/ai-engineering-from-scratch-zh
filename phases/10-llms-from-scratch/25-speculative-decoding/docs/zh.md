# 推测解码与EAGLE

> 一个前沿的大语言模型每生成一个token都需要对数十亿参数进行一次完整的前向传播。这种前向传播存在严重的资源过度配置：大多数情况下，一个更小的模型就能正确预测接下来的3-5个token，而大模型只需*验证*这个猜测。当猜测正确时，你相当于用一次计算的成本获得了5个token。推测解码（Leviathan等人, 2023）使这一机制精确化，而EAGLE-3（2025）则将接受率提升至每次验证约4.5个token——在输出分布匹配的情况下实现了4-5倍的加速。

**类型：** 构建
**语言：** Python（使用numpy）
**前置课程：** 第10阶段第12课（推理优化），第10阶段第4课（预训练Mini-GPT）
**时间：** 约75分钟

## 问题所在

一个70B级别的模型在H100上的解码吞吐量通常为每秒40-80个token。每个token都需要从HBM读取所有模型权重进行一次完整的前向传播。你无法在不改变模型输出的情况下缩小模型规模。你无法在内存限制下增加批处理大小。你陷入了困境——除非能让模型在一次前向传播中输出多个token。

自回归生成看起来本质上是串行的：`x_{t+1} = sample(p(· | x_{1:t}))`。但存在并发的可能性。如果你有一个廉价的预测器，它能告诉你“接下来的4个token很可能是[a, b, c, d]”，那么你就可以通过**大模型的一次前向传播**来验证所有5个位置，并接受最长的匹配前缀。

Leviathan、Kalai、Matias（2023，“通过推测解码实现Transformer的快速推理”）通过一个巧妙的接受/拒绝规则精确实现了这一点，该规则保留了目标模型的采样分布。相同的输出分布，速度提升2-4倍。

## 核心概念

### 双模型设置

- **目标模型** `M_p`：大型、缓慢、高质量的模型，你真正希望从中获得采样。分布：`p(x)`。
- **草稿模型** `M_q`：小型、快速、质量较低的模型。分布：`q(x)`。体积缩小5-30倍。

每一步的操作：

1. 草稿模型自回归地提议`K`个token：`x_1, x_2, ..., x_K ~ q`。
2. 目标模型对所有`K+1`个位置并行运行一次前向传播，为每个提议的token生成`p(x_k)`。
3. 按照下面修改后的拒绝采样规则，从左到右接受/拒绝每个token。接受最长的匹配前缀。
4. 如果有任何token被拒绝，则从修正后的分布中采样替代token并停止。否则，从`p(· | x_1...x_K)`中采样一个额外的token作为奖励。

如果草稿与目标完全匹配，你将在一次目标前向传播中获得K+1个token。如果草稿在第1个位置出错，你将只获得1个token。

### 精确性规则

推测解码在分布上**被证明等价于从p中采样**。拒绝规则如下：

```
For each drafted token x_t:
    r ~ Uniform(0, 1)
    if r < p(x_t) / q(x_t):
        accept x_t
    else:
        sample replacement from residual: (p - q)+ / ||(p - q)+||_1
        stop
```

其中`(p - q)+`表示逐点差异的正部。当草稿和目标一致时（`p ≈ q`），接受率接近1。当它们不一致时，会构建残差分布，以确保整体样本仍然精确地服从`p`。

**贪心情况。** 对于温度=0的采样，只需检查`argmax(p) == x_t`。如果是，则接受；如果否，则输出`argmax(p)`并停止。

### 预期加速比

如果草稿模型的token级接受率为`α`，则每次目标前向传播预期产生的token数为：

```
E[tokens] = (1 - α^{K+1}) / (1 - α)        # K = draft length, α in [0, 1]
```

当`α = 0.8, K = 4`时：每次前向传播产生`(1 - 0.8^5)/(1 - 0.8) = 3.36`个token。单次目标前向传播的成本大约为`cost_q * K + cost_p`（K次草稿步骤加上一次目标验证）。如果`cost_p >> cost_q * K`，则吞吐量的加速比为`3.36× / 1 = 3.36×`。

唯一的真正参数是`α`，它完全取决于草稿与目标的匹配程度。一个好的草稿模型至关重要。

### 训练草稿：蒸馏

一个随机的小模型会成为糟糕的草稿模型。标准方法是通过目标模型进行蒸馏：

1. 选择一个小型架构（对于70B目标模型，选择约1B参数的模型；对于7B目标模型，选择约500M参数的模型）。
2. 在大型文本语料库上运行目标模型；存储其下一个token的分布。
3. 使用KL散度对草稿模型进行训练，使其接近目标分布（而不是接近真实token）。

结果：`α`在代码生成任务上通常为0.6-0.8，在自然语言聊天任务上为0.7-0.85。在生产环境中加速2-3倍。

### EAGLE：树状草稿生成 + 特征复用

Li、Wei、Zhang、Zhang（2024，“EAGLE：推测采样需要重新思考特征不确定性”）发现了标准推测解码的两个低效之处：

1. 草稿模型执行K次串行步骤，每步都是全栈操作。但草稿模型可以复用目标模型在最近一次验证中的特征（隐藏状态）——目标模型已经计算了丰富的表征，而草稿模型却在从头重新推导。
2. 草稿模型输出一个线性序列。如果草稿模型能输出一个*候选树*（每个节点有多个猜测），那么目标模型的一次前向传播就可以通过树状注意力掩码并行验证多条候选路径，并选择最长的被接受分支。

EAGLE-1的改变：
- 草稿输入 = 目标模型在位置t的最终隐藏状态，而不是原始token。
- 草稿架构 = 1个Transformer解码器层（而不是一个独立的小模型）。
- 输出 = 每层深度K=4-8个候选节点的树，深度为4-6层。

EAGLE-2（2024）增加了动态树拓扑：在草稿模型不确定的地方，树长得更宽；在草稿模型自信的地方，树保持狭窄。在不增加验证成本的情况下提高了`α_effective`。

EAGLE-3（Li等人, 2025，“EAGLE-3：通过训练时测试扩展大语言模型推理加速”）移除了对顶层特征的固定依赖，并采用新的“测试时模拟”损失来训练草稿模型——草稿模型是在匹配目标模型测试时分布的输出上训练的，而不是在教师强制的训练分布上。接受率从0.75（EAGLE-2）提升到0.82（EAGLE-3），平均每次验证的token数从3.0提升到4.5。

### 树状注意力验证

当草稿模型输出一棵树时，目标模型使用**树状注意力掩码**在一次前向传播中进行验证——这是一种编码了树拓扑结构（而非纯线性序列）的因果掩码。每个token只关注其在树中的祖先节点。验证过程仍然是一次前向传播，一次矩阵乘法；拓扑掩码只增加了少量额外的KV缓存条目。

```
        root
       /    \
      a      b
     / \    / \
    c  d   e   f
```

如果`a, b`是竞争的首token候选者，而`c, d, e, f`是第二token候选者，那么所有6个位置都在一次前向传播中得到验证。输出是沿任何被接受路径的最长前缀。

### 适用与不适用场景

**适用场景：**
- 预测性文本的聊天/补全（如代码、常见英语、结构化输出）。`α`较高。
- 解码期间GPU计算有闲置的设置（内存受限阶段）。树状草稿生成利用了可用的FLOPs。

**不适用场景/无优势：**
- 高度随机的输出（如高温度下的创意写作）。`α`会下降接近`1/|vocab|`。
- 非常高并发度的批处理服务——批处理已经填满了FLOPs，留给树状验证的空间很小。
- 目标模型本身很小，草稿模型与之相差不大。

生产环境通常报告：聊天任务加速2-3倍，代码生成加速3-5倍，创意写作几乎无加速。

## 动手构建

`code/main.py`：

- 一个实现精确拒绝规则并验证其保留目标分布的参考`speculative_decode(target, draft, prompt, K, temperature)`（与纯目标采样相比，经验KL散度 < 0.01）。
- 一个EAGLE风格的树状草稿生成器，构建深度为K的树，使用top-p分支策略。
- 一个树状注意力掩码构建器，为验证器生成正确的因果模式。
- 一个接受率测试工具，在一个小型语言模型（从GPT-2-medium目标模型蒸馏一个GPT-2-small）上运行两者。

```python
def speculative_step(p_target, q_draft, K, temperature=1.0):
    """One round of speculative decoding. Returns list of accepted tokens."""
    # 1. Draft K tokens
    draft_tokens = []
    q_probs = []
    state = draft_state_init()
    for _ in range(K):
        probs = softmax(q_draft(state) / temperature)
        t = np.random.choice(len(probs), p=probs)
        draft_tokens.append(t)
        q_probs.append(probs[t])
        state = draft_step(state, t)

    # 2. Target computes p at every drafted position + 1 extra
    p_probs_all = target_forward_batched(p_target, draft_tokens, temperature)

    # 3. Accept/reject left-to-right
    accepted = []
    for k, tok in enumerate(draft_tokens):
        r = np.random.uniform()
        if r < p_probs_all[k][tok] / q_probs[k]:
            accepted.append(tok)
        else:
            residual = np.maximum(p_probs_all[k] - q_probs[k], 0)
            residual /= residual.sum()
            accepted.append(np.random.choice(len(residual), p=residual))
            return accepted
    # 4. All K accepted → sample bonus token from target
    accepted.append(np.random.choice(len(p_probs_all[-1]), p=p_probs_all[-1]))
    return accepted
```

## 如何使用

- **vLLM** 和 **SGLang** 提供一流的推测解码支持。标志位：`--speculative_model`、`--num_speculative_tokens`。通过`--spec_decoding_algorithm eagle`标志支持EAGLE-2/3。
- **NVIDIA TensorRT-LLM** 原生支持Medusa和EAGLE树结构。
- **参考草稿模型**：`Qwen/Qwen3-0.6B-spec`（用于Qwen3-32B的草稿模型）、`meta-llama/Llama-3.2-1B-Instruct-spec`（用于70B的草稿模型）。
- **Medusa头**（Cai等人, 2024，“Medusa：具有多个解码头的简易LLM推理加速框架”）：不是使用独立的草稿模型，而是在目标模型本身上添加K个并行预测头。部署更简单，但接受率略低于EAGLE。

## 部署应用

本课程产出`outputs/skill-speculative-tuning.md`——一种技能，用于分析目标模型的工作负载并选择：草稿模型、K（草稿长度）、树宽度、温度，以及何时回退到纯解码。

## 练习

1. 实现精确拒绝规则并进行经验验证。通过`speculative_decode`和纯目标采样运行10K个样本；计算两种输出分布之间的总变差距离。应小于0.01。

2. 计算加速公式。给定固定的`α`和`K`，绘制每次目标前向传播的预期token数。找到在α ∈ {0.5, 0.7, 0.9} 时的最优K值。

3. 训练一个微型草稿模型。取一个124M GPT-2目标模型，在1亿token上通过KL损失蒸馏一个30M GPT-2草稿模型。在留出文本上测量`α`。预期值：0.6-0.7。

4. 实现EAGLE风格的树状草稿生成。不是输出一个链，而是在每个深度输出top-3分支。构建树状注意力掩码。验证目标模型接受最长正确分支的过程。

5. 测量失败模式。在温度=1.5（高随机性）下运行推测解码。展示α如何崩溃，以及由于草稿开销，算法比纯解码更慢。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|------------|----------|
| 目标模型 | "大模型" | 你希望从中采样的慢速、高质量模型（p分布） |
| 草稿模型 | "推测器" | 小型、快速的预测器（q分布）；比目标模型小5-30倍 |
| K / 草稿长度 | "前瞻长度" | 每次验证步骤中推测的token数量 |
| α / 接受率 | "命中率" | 草稿模型提议的每个token被接受的概率 |
| 精确拒绝规则 | "接受测试" | 比较 r < p/q，该规则保留目标模型的分布 |
| 残差分布 | "修正后的p-q" | (p - q)+ / \|\|(p - q)+\|\|_1，拒绝时从此分布采样 |
| 树状草稿生成 | "分支推测" | 草稿模型输出一棵候选树，使用树状注意力掩码在一次验证中完成验证 |
| 树状注意力掩码 | "拓扑掩码" | 编码树拓扑结构的因果掩码，使每个节点只关注其祖先节点 |
| Medusa头 | "并行头" | 目标模型本身的K个额外预测头；无需独立的草稿模型 |
| EAGLE特征复用 | "隐藏状态草稿" | 草稿模型的输入是目标模型的最后隐藏状态，而非原始token，从而缩小了草稿模型 |
| 测试时模拟损失 | "EAGLE-3训练法" | 在匹配目标模型测试时分布（而非教师强制）的输出上训练草稿模型 |
