# 推测解码与EAGLE-3

> 第7阶段 · 第16课证明了数学原理：利维坦拒绝规则完全保留了验证器的分布。本课是从训练栈视角看2026年生产环境中的推测解码。EAGLE-3将草稿模型从廉价的近似变成了一个专门构建的小型网络，该网络在验证器自身的隐藏状态上训练，并添加了一个训练时测试循环来对齐其训练和推理分布。结果：端到端3倍至6.5倍的加速，聊天任务上每个token的接受率高于0.9，且无分布权衡。2026年的每一个生产推理栈都默认启用它。

## 学习目标

- 用一句话陈述利维坦定理，并证明推测循环产生的样本与验证器分布相同。
- 走过从原始推测解码（利维坦2023）到EAGLE、EAGLE-2、EAGLE-3的两年发展历程，并说出每一步移除了哪个具体限制。
- 根据接受率 `α` 和草稿与验证器成本比 `c` 计算预期加速，并为每种情况选择最优草稿长度 `N`。
- 从头实现完整的推测循环：草拟、验证、从残差分布中拒绝采样、在拒绝时回滚KV缓存、在完全接受时输出奖励token。

## 问题所在

在H100上运行70B模型的自回归解码，速度大约为每秒35个token。GPU远未饱和。内存带宽是瓶颈：每个token从HBM加载70B的权重，进行一次算术运算，然后产生一个浮点数。计算单元大部分时间处于空闲状态。

推测解码将其转变为一个可以通过的吞吐量问题。一个廉价的草稿模型在 `N` 次小前向传递中提出 `N` 个token。验证器在前缀加上所有 `N` 个草稿上运行一次。如果验证器在位置 `i` 的分布与草稿一致（我们将精确定义这种统计意义上的一致），我们接受；如果不一致，我们拒绝并从残差分布中采样一个修正token。单次大模型前向传递可以产生最多 `N+1` 个被接受的token，而不是一个。

重要的定理是利维坦、卡尔曼、马蒂亚斯（ICML 2023）：输出分布与直接从验证器中采样产生的分布完全相同。不是近似，是完全相同。这是推测解码能在生产中使用的全部原因——它是一种无质量权衡的纯延迟优化。

第7阶段 · 第16课给你的是数学。本课给你的是训练栈。一个好的草稿比廉价草稿多带来2倍的加速。EAGLE、EAGLE-2 和 EAGLE-3（Li et al., 2024–2025）将"草稿=相同模型的较小版本"变成了一个精确的工程学科。2026年的生产推理服务器默认使用EAGLE-3。

## 核心概念

### 不变量：利维坦拒绝采样

设 `p(t)` 为草稿在给定前缀下对下一个token的分布，`q(t)` 为验证器的分布。采样一个草稿token `d ~ p`。以概率 `min(1, q(d) / p(d))` 接受。在拒绝时，从残差分布 `(q - p)_+ / ||(q - p)_+||_1` 中采样。得到的样本服从 `q` 分布。无论 `p` 多差，这都成立——它越差，你拒绝得越频繁，但输出保持精确。

将 `N` 次这样的调用串联起来，使用一次验证器前向传递处理 `prefix + d_1 + ... + d_N`。验证器同时返回 `q_1, q_2, ..., q_{N+1}` 个结果。从左到右遍历。在位置 `j` 首次拒绝时，从 `residual(q_j, p_j)` 中采样并停止。在完全接受时，从 `q_{N+1}` 中采样一个奖励token。

### 什么决定了加速

设 `α` 为每个草拟token的预期接受率。设 `c = cost(draft) / cost(verifier)` 为成本比。每次验证器前向传递预期接受的token数为：

```
E[accepted] = (1 - α^(N+1)) / (1 - α)
```

每个被接受token的预期总墙钟时间是 `(N * c + 1) / E[accepted]`。对 `N` 求最小值，你会得到最佳点。对于 `α = 0.8, c = 0.05`：最优 `N` 大约为5-7，加速为3.2倍。对于 `α = 0.95, c = 0.02`：最优 `N` 大约为8-10，加速达到5倍。

单一最大的杠杆是 `α`。在固定的 `N = 5` 下，从 `α = 0.6`（原始草稿）到 `α = 0.9`（EAGLE-3），每次验证器前向传递预期接受的token数从2.2个增加到4.1个。仅通过相同的验证器就获得了近2倍的吞吐量提升。

### 两年的演进历程

**原始推测解码（利维坦，2023）。** 草稿模型是独立训练的同系列较小LLM。易于实现，`α ≈ 0.6`，最多约2倍加速。

**EAGLE-1（Li et al., 2024）。** 草稿是一个微小的transformer——通常一到两层——它以验证器最后一层的隐藏状态作为输入，并直接预测下一个token。因为草稿看到了验证器的特征表示，其分布更接近验证器的分布。`α` 提升到0.7–0.8。

**EAGLE-2（Li et al., 2024）。** 增加了动态草稿树：不是提出单个 `N` token的序列，而是提出一个小的候选树，在一次验证器前向传递中使用树注意力机制对每个候选进行评分，并选择概率最高的路径。草稿长度在每一步都是自适应的。`α` 每个被接受路径token提升到0.85以上。

**EAGLE-3（Li et al., 2025, NeurIPS）。** 两个主要变化。首先，完全放弃特征预测损失——EAGLE-1/2训练草稿以匹配验证器的隐藏状态，这限制了数据带来的提升。EAGLE-3直接在token预测上训练。其次，训练时测试（TTT）：在草稿训练期间，像推理时一样，将草稿自身之前的预测作为多步输入反馈。这使训练和测试分布保持一致，并阻止了误差累积。测量的加速：在聊天任务上最高达6.5倍，在H100上使用SGLang进行批次为64的推理时，吞吐量提升38%。

### KV缓存回滚

验证在一次传递中将验证器的KV缓存扩展了 `N` 个条目。如果在位置 `j` 发生拒绝，则位置 `j-1` 之后的缓存内容现在是错误的。两种常见实现：写入临时缓冲区并在接受时提交（vLLM，TensorRT-LLM），或者保留物理KV缓存加上逻辑长度，并在拒绝时截断。无论哪种方式，回滚成本是每层每头的字节数，与前向传递成本相比可忽略不计。

对于EAGLE-2树搜索，验证器使用一个符合树拓扑的非因果掩码来运行注意力。工程实现很繁琐，但计算是标准flash-attention调用加上自定义掩码。

### 2026年的草稿架构

| 策略 | 草稿类型 | `α` | 加速 | 训练成本 |
|------|-----------|-----|---------|---------------|
| 原始 | 独立的小型LLM | 0.55-0.70 | 1.8-2.3倍 | 无（复用现有小型模型） |
| Medusa | 在验证器上增加额外的LM头 | 0.65-0.75 | 2-3倍 | ~10亿SFT tokens |
| EAGLE-1 | 基于隐藏状态的单层transformer | 0.70-0.80 | 2.5-3倍 | ~600亿 tokens |
| EAGLE-2 | EAGLE-1 + 动态草稿树 | 0.80-0.88 | 3-4倍 | ~600亿 tokens |
| EAGLE-3 | 多层特征融合 + TTT | 0.88-0.92 | 3.5-6.5倍 | ~600-2000亿 tokens |
| 前瞻（Lookahead） | 无草稿（雅可比迭代） | N/A | 1.3-1.6倍 | 无 |

在2026年生产环境中：vLLM和SGLang在可用时默认使用EAGLE-3，否则使用EAGLE-2。TensorRT-LLM为Meta和NVIDIA公开模型提供了最快的Medusa路径。llama.cpp为CPU部署提供原始草稿。

## 动手构建

参见 `code/main.py`。这是完整的利维坦推测循环，包含所有部分：N个草稿、验证器并行传递、逐位置拒绝、残差采样、奖励token、KV回滚，以及通过经验验证输出分布与从 `q` 直接采样一致。

### 步骤1：拒绝规则

```python
def accept(q_prob, p_prob, u):
    if p_prob <= 0:
        return True
    return u < min(1.0, q_prob / p_prob)
```

### 步骤2：残差分布

```python
def residual(q, p):
    raw = [max(0.0, qi - pi) for qi, pi in zip(q, p)]
    s = sum(raw)
    if s == 0:
        return list(q)
    return [r / s for r in raw]
```

### 步骤3：一个完整的推测步骤

`spec_step` 函数从 `p` 草拟 `N` 个token，然后在一次并行 `q` 评估中验证所有token。对于每个草拟的token，它应用拒绝规则，并在第一次拒绝时从残差分布中采样修正。如果全部接受，则从 `q_{N+1}` 发出一个奖励token。

### 步骤4：KV回滚簿记

模拟器为每个工作线程跟踪一个逻辑 `kv_length`。当接受 `k` 个草稿时，`kv_length += k`。当在位置 `j` 发生拒绝时，缓存已经写入了 `j` 之后的内容，但逻辑长度被设置为 `prefix_length + j + 1`——比修正token多一个位置。后续读取会截断到逻辑长度。

### 步骤5：利维坦检验

运行50,000个推测步骤。统计被接受token的经验分布。与从 `q` 直接采样50,000次进行比较。卡方统计量应远低于临界值。该定理在实践中成立。

### 步骤6：加速与α的关系

通过以不同幅度扰动 `p` 远离 `q` 来扫描草稿质量。测量 `α`，然后绘制每次验证器调用预期token数作为 `α` 和 `N` 的函数关系。代码会打印一个表格，展示EAGLE-3级别草稿质量（`α ≈ 0.9`）如何解锁每次验证器调用4-5个token。


何时使用推测解码：
- 任何p50延迟比峰值吞吐量更重要的交互式聊天工作负载。
- 代码生成和结构化输出（JSON、SQL）。`α` 高于0.9，因为目标分布高度可预测。
- 长篇生成（数千个token）。摊销后的加速持续有效。

何时不用：
- 非常小的模型（< 3B）。草稿并不比验证器便宜多少。
- 极小的批次为1的CPU部署。草稿模型的内存开销可能不值得。
- `α` 崩溃的非常高温的创造性采样。

## 关键术语

| 术语 | 人们说法 | 实际含义 |
|------|----------------|------------------------|
| 利维坦规则 | "min(1, q over p)" | 以概率 `min(1, q(d)/p(d))` 进行伯努利接受/拒绝，当拒绝时从残差分布采样时，精确保留验证器分布 |
| 残差分布 | "(q minus p) plus, normalized" | `(q - p)_+` 钳位到零并重新归一化——拒绝时应采样的正确分布 |
| 接受率 α | "草稿正确的频率" | 拒绝规则下每个token的预期伯努利成功概率；决定所有加速计算 |
| EAGLE-1 | "隐藏状态草稿" | 以验证器最后一层隐藏状态为条件的微小transformer草稿（Li et al., 2024） |
| EAGLE-2 | "动态草稿树" | EAGLE-1加上一个用树注意力在一次验证器传递中评分的候选延续树 |
| EAGLE-3 | "训练时测试" | 放弃特征预测损失，在训练期间草稿使用自身输出作为输入，直接在token预测上训练 |
| 训练时测试 (TTT) | "曝光偏差修复" | 在训练期间自回归地运行草稿，使训练和测试输入分布匹配——计划采样的直接类比 |
| KV缓存回滚 | "撤销被拒绝的草稿" | 在拒绝后，将验证器的KV缓存重置为被接受前缀长度的簿记 |
| 奖励token | "免费的那个" | 当所有 `N` 个草稿被接受时，从 `q_{N+1}` 无额外验证器成本地采样一个额外token |
| 树注意力 | "一次验证多个候选" | 使用符合草稿树拓扑的非因果掩码进行注意力；在一次前向传递中为树中的每个节点计算 `q_i` |
