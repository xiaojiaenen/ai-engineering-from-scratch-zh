# 视频生成

> 图像是2-D张量。视频是3-D张量。理论是相同的；计算量则大10-100倍。OpenAI的Sora（2024年2月）证明了其可行性。到2026年，Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0和WAN 2.2已能从文本生成1080p的生产级视频——而开源权重技术栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）落后12个月。

**类型：** 构建
**语言：** Python
**前置要求：** 阶段8 · 07（潜在扩散），阶段7 · 09（ViT），阶段8 · 06（DDPM）
**时间：** ~45分钟

## 问题

一个10秒、24fps的1080p视频包含240帧，每帧1920×1080×3像素。这相当于每个片段约1.5 GB的原始数据。像素空间的扩散是不可行的。你需要：

1.  **时空压缩。** 一个能编码视频（而非单帧）为空间-时间块序列的VAE。
2.  **时间连贯性。** 帧需要在数秒内共享内容、光照和物体同一性。网络必须建模运动。
3.  **计算预算。** 在相同模型规模下，视频训练比图像训练昂贵10-100倍。
4.  **条件输入。** 文本、图像（首帧）、音频或另一个视频。大多数生产模型支持全部四种。

解决此问题的架构是应用于时空块的**扩散Transformer（DiT）**，在大型（提示词、描述、视频）数据集上进行训练。使用与课程06相同的扩散损失。

## 概念

![视频扩散：分块化，DiT，解码](../assets/video-generation.svg)

### 分块化

使用3D VAE（学习到的时空压缩）编码视频。潜在表示的形状为`[T_latent, H_latent, W_latent, C_latent]`。将其分割成大小为`[t_p, h_p, w_p]`的块。对于Sora风格的模型，`t_p = 1`（每帧分块）或`t_p = 2`（每两帧分块）。一个10秒的1080p视频被压缩到约20,000-100,000个块。

### 时空DiT

一个Transformer处理展平的块序列。每个块具有3D位置嵌入（时间 + y + x）。注意力通常是分解的：

- **空间注意力** 在每个帧的块内。
- **时间注意力** 跨越相同空间位置的帧。
- **完整3D注意力** 昂贵16-100倍；仅用于低分辨率或研究中。

### 文本条件

与大型文本编码器进行交叉注意力（Sora使用T5-XXL，CogVideoX-5B使用T5-XXL）。长提示词很重要——Sora的训练集包含了GPT生成的密集重新描述，每个片段平均200个token。

### 训练

标准扩散损失（ε或v预测）应用于时空潜在表示。数据：网络视频 + ~1亿精选片段 + 合成文本描述。计算：即使是小规模研究运行也需要10,000+ GPU小时；Sora规模则需要100,000+。

## 构建

`code/main.py` 模拟了核心时空DiT思想：对一个小型合成视频进行分块化，添加每个块的位置嵌入，并使用基于Transformer的注意力对整个序列进行去噪。不使用numpy；纯Python。我们展示了即使在一维情况下，当相邻帧的块共享去噪器和位置嵌入时，时间连贯性也会涌现。

### 步骤1：对一个合成的1-D“视频”进行分块化

```python
def make_video(T_frames=8, rng=None):
    # a "video" is a sequence of 1-D values following a smooth trajectory
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤2：为每帧添加位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 步骤3：去噪器看到整个序列

我们的小型网络不是独立地去噪每一帧，而是将所有帧的值及其位置嵌入连接起来，并联合预测所有帧的噪声。

### 步骤4：时间连贯性测试

训练后，采样一个视频。测量帧间差值。如果模型学到了时间结构，差值将保持在比独立采样每一帧更小的范围内。

## 陷阱

- **独立采样每帧 = 闪烁。** 如果你对每一帧单独运行图像扩散，输出会闪烁，因为每帧的噪声是独立的。视频扩散通过注意力或共享噪声耦合帧来解决此问题。
- **朴素的3D注意力 = 内存溢出。** 对一个10秒1080p的潜在表示进行全3D注意力操作，计算量达数千亿次。应分解为空间+时间注意力。
- **数据描述比数据量更重要。** Sora相比之前工作的主要升级是使用了约10倍更详细的描述（GPT-4重新标注了片段）。OpenAI的技术报告明确指出了这一点。
- **首帧条件。** 大多数生产模型也接受一张图像作为第一帧。这是“图生视频”模式；训练包含了这种变体。
- **物理漂移。** 长片段（>10秒）会积累微妙的不一致性。滑动窗口生成 + 关键帧锚定有所帮助。

## 生产提示：视频潜在表示是内存带宽问题

一个10秒、24fps的1080p片段有240帧 × 1920 × 1080 × 3 ≈ 1.5 GB的原始像素。经过4倍视频VAE压缩（`2 × spatial × 2 × temporal`）后，每次请求的潜在表示约100 MB。将其通过时空DiT运行30步，批次大小为1，每步通过HBM传输约3 GB数据——瓶颈在于内存带宽，而非浮点运算次数。

三个来自生产推理文献的生产级调优参数：

- **在DiT上使用张量并行(TP)。** 文生视频模型通常参数≥10B。跨4个H100使用TP=4是标准配置；对于405B级模型使用PP=2 × TP=2。每步延迟大致随TP线性下降，直到遇到all-reduce墙。
- **帧批处理 = 持续批处理。** 在生成时，视频概念上是由注意力连接的帧批次。持续批处理（在途调度）适用：如果模型架构支持滑动窗口生成，则在返回 `t-1` 帧的同时开始渲染 `t+1` 帧。
- **片段级预填充缓存。** 对于图生视频，首帧条件类似于LLM的提示词预填充：计算一次，在时间解码器传递中重复使用。这实质上是视频的KV缓存。
