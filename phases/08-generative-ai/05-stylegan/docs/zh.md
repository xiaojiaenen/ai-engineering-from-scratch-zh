# StyleGAN

> 大多数生成器同时将 `z` 注入到每一层。StyleGAN 将其分开：首先将 `z` 映射到一个中间 `w`，然后通过 AdaIN 在每个分辨率级别 *注入* `w`。这单一的改变解开了潜在空间的纠缠，并使得照片级逼真人脸成为过去七年来一个已解决的问题。

**类型：** 构建
**语言：** Python
**前置条件：** 第 8 阶段 · 03 (GANs), 第 4 阶段 · 08 (归一化), 第 3 阶段 · 07 (CNNs)
**时间：** ~45 分钟

## 问题

一个 DCGAN 通过一堆转置卷积将 `z` 映射到图像。问题是：`z` 控制着一切——姿势、光照、身份、背景——都纠缠在一起。沿着 `z` 的一个轴移动，所有四个属性都会改变。你无法让模型"同一个人，不同姿势"，因为其表征并非以这种方式分解的。

Karras 等人 (2019, NVIDIA) 提出：停止将 `z` 直接送入卷积层。而是将一个常量 `4×4×512` 张量作为网络输入。学习一个 8 层 MLP 来映射 `z ∈ Z → w ∈ W`。通过 *自适应实例归一化* (AdaIN) 在每个分辨率注入 `w`：归一化每个卷积特征图，然后通过 `w` 的仿射投影进行缩放和偏移。为每一层添加噪声以产生随机细节（皮肤毛孔、发丝）。

结果是：`W` 大致拥有正交的轴，分别对应"高级风格"（姿势、身份）和"精细风格"（光照、颜色）。你可以通过使用图像 A 的 `w` 用于低分辨率级别，图像 B 的 `w` 用于高分辨率级别，来交换两张图像之间的风格。这解锁了编辑、跨域风格化以及整个"StyleGAN 反演"研究线。

## 概念

![StyleGAN: 映射网络 + AdaIN + 逐层噪声](../assets/stylegan.svg)

**映射网络。** `f: Z → W`，一个 8 层 MLP。`Z = N(0, I)^512`。`W` 无需强制为高斯分布——它学习一种数据自适应的形状。

**合成网络。** 从一个学习到的常量 `4×4×512` 开始。每个分辨率块：`upsample → conv → AdaIN(w_i) → noise → conv → AdaIN(w_i) → noise`。分辨率倍增：4, 8, 16, 32, 64, 128, 256, 512, 1024。

**AdaIN。**

```
AdaIN(x, y) = y_scale · (x - mean(x)) / std(x) + y_bias
```

其中 `y_scale` 和 `y_bias` 来自 `w` 的仿射投影。对每个特征图进行归一化，然后重新调整风格。这里的"风格"是指特征图的一阶和二阶统计量。

**逐层噪声。** 添加到每个特征图的单通道高斯噪声，由一个学习到的逐通道因子缩放。它控制随机细节而不影响全局结构。

**截断技巧。** 在推理时，采样 `z`，计算 `w = mapping(z)`，然后 `w' = ŵ + ψ·(w - ŵ)`，其中 `ŵ` 是多个样本的平均 `w`。`ψ < 1` 在多样性和质量之间进行权衡。几乎所有的 StyleGAN 演示都使用 `ψ ≈ 0.7`。

## StyleGAN 1 → 2 → 3

| 版本 | 年份 | 创新 |
|------|------|------|
| StyleGAN | 2019 | 映射网络 + AdaIN + 噪声 + 渐进式增长。 |
| StyleGAN2 | 2020 | 权重解调取代 AdaIN（修复滴状伪影）；跳跃/残差架构；路径长度正则化。 |
| StyleGAN3 | 2021 | 无别名卷积 + 等变核；消除纹理粘附于像素网格的现象。 |
| StyleGAN-XL | 2022 | 类条件，1024²，ImageNet。 |
| R3GAN | 2024 | 使用更强正则化重新命名；在 FFHQ-1024 上以少 20 倍的参数弥合与扩散模型的差距。 |

在 2026 年，StyleGAN3 仍然是以下情况的默认选择：(a) 高 FPS 下的窄域照片级真实感，(b) 少样本域适应（用 100 张图像在新数据集上训练，冻结映射），(c) 基于反演的编辑（找到重建真实照片的 `w`，然后编辑该 `w`）。对于开放域文本到图像，它不是合适的工具——扩散模型才是。

## 构建它

`code/main.py` 实现了一个玩具版的"精简 StyleGAN"，用于一维数据：一个映射 MLP，一个合成函数，该函数接受一个学习到的常向量并用 `w` 派生的缩放/偏移进行调制，以及逐层噪声。它表明，通过仿射调制注入 `w` 的效果与或优于将 `z` 连接到生成器输入的效果。

### 步骤 1：映射网络

```python
def mapping(z, M):
    h = z
    for i in range(num_layers):
        h = leaky_relu(add(matmul(M[f"W{i}"], h), M[f"b{i}"]))
    return h
```

### 步骤 2：自适应实例归一化

```python
def adain(x, w_scale, w_bias):
    mu = mean(x)
    sd = std(x)
    x_norm = [(xi - mu) / (sd + 1e-8) for xi in x]
    return [w_scale * xi + w_bias for xi in x_norm]
```

逐特征图的缩放和偏移来自 `w` 的线性投影。

### 步骤 3：逐层噪声

```python
def add_noise(x, sigma, rng):
    return [xi + sigma * rng.gauss(0, 1) for xi in x]
```

Sigma（标准差）是逐通道可学习的。

## 陷阱

- **滴状伪影。** StyleGAN 1 在特征图中产生了块状滴状物，因为 AdaIN 将均值归零。StyleGAN2 的权重解调通过缩放卷积权重（而不是激活值）来修复它。
- **纹理粘附。** StyleGAN 1 和 2 的纹理跟随像素坐标，而不是物体坐标（在插值时可见）。StyleGAN3 的无别名卷积通过加窗 sinc 滤波器修复了这个问题。
- **模式覆盖。** 截断 `ψ < 0.7` 看起来干净，但采样自一个狭窄的锥体；如果你需要多样性，请使用 `ψ = 1.0`。
- **反演是有损的。** 将真实照片反演为 `W` 通常通过优化或编码器（e4e, ReStyle, HyperStyle）完成。结果在多次迭代后会漂移。

## 使用它

| 用例 | 方法 |
|------|------|
| 照片级真实人脸（动漫、产品、窄域） | StyleGAN3 FFHQ / 自定义微调 |
| 从照片进行人脸编辑 | e4e 反演 + StyleSpace / InterFaceGAN 方向 |
| 人脸交换 / 重现 | StyleGAN + 编码器 + 混合 |
| 虚拟形象流程 | StyleGAN3 搭配 ADA 进行低数据量微调 |
| 从少量图像进行域适应 | 冻结映射网络，微调合成部分 |
| 多模态或文本条件生成 | 不要——请使用扩散模型 |

对于答案是"人脸照片"的产品级演示，StyleGAN 在推理成本（单次前向传播，在 4090 上 <10ms）和相同质量标准下的清晰度方面击败了扩散模型。

## 练习

1.  **简单。** 使用 `adain_on=True` 和 `adain_on=False` 运行 `code/main.py`。比较固定潜在向量和扰动潜在向量的输出分布范围。
2.  **中等。** 实现混合正则化：对于一个训练批次，计算 `w_a`，`w_b`，并在合成的前半部分应用 `w_a`，后半部分应用 `w_b`。解码器是否学习到了解耦的风格？
3.  **困难。** 获取一个预训练的 StyleGAN3 FFHQ 模型 (ffhq-1024.pkl)。通过在标记样本上训练 SVM，找到控制"微笑"的 `w` 方向；报告在身份开始漂移之前你能推动多远。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|------|------------|--------------|
| 映射网络 | "那个 MLP" | `f: Z → W`，8 层，将潜在几何与数据统计解耦。 |
| W 空间 | "风格空间" | 映射网络的输出；大致上是解耦的。 |
| AdaIN | "自适应实例归一化" | 归一化特征图，然后通过 `w` 投影进行缩放 + 偏移。 |
| 截断技巧 | "Psi" | `w = mean + ψ·(w - mean)`，ψ<1 在多样性和质量之间进行权衡。 |
| 路径长度正则化 | "PL 正则化" | 惩罚每单位 `w` 变化引起的图像大变化；使 `W` 更平滑。 |
| 权重解调 | "StyleGAN2 的修复" | 归一化卷积权重而不是激活值；消除滴状伪影。 |
| 无别名 | "StyleGAN3 的技巧" | 加窗 sinc 滤波器；消除纹理粘附于像素网格的现象。 |
| 反演 | "为真实图像找 w" | 优化或编码 `x → w` 以使 `G(w) ≈ x`。 |

## 生产说明：为什么 StyleGAN 在 2026 年仍在使用

在 4090 上，StyleGAN3 生成 1024² FFHQ 人脸用时不到 10 毫秒——`num_steps = 1`，没有 VAE 解码，没有交叉注意力传递。用生产术语来说，这是任何图像生成器的底限延迟。在相同分辨率下，50 步的 SDXL + VAE 解码管线大约需要 3 秒。这是一个 **300 倍的差距**，对于窄域产品（虚拟形象服务、身份证件流程、库存人脸生成）来说，它在总体拥有成本 (TCO) 上胜出。

两个运营后果：

- **无调度器，无批处理器。** 在目标占用率下进行静态批处理是最优的。连续批处理（对 LLM 和扩散模型至关重要）不提供任何好处，因为每个请求消耗的计算量相同。
- **截断 `ψ` 是安全旋钮。** `ψ < 0.7` 采样自映网络输出范围的一个狭窄锥体。这是服务层控制样本方差的唯一杠杆。在峰值负载时降低 `ψ`，为高级用户提高它。
