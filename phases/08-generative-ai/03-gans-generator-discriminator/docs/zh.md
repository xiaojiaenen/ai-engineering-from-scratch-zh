# GANs — 生成器 vs 判别器

> 古德费洛在2014年的巧妙思路是完全绕过密度估计。两个网络：一个制造赝品，一个识破赝品。两者不断对抗，直到赝品与真品无法区分。这理论上不应奏效，实践中也常失败。但一旦成功，其生成样本的清晰度在狭窄领域内仍是文献中最佳的。

**类型：** 构建型
**编程语言：** Python
**先修知识：** 阶段 3 · 02（反向传播），阶段 3 · 08（优化器），阶段 8 · 02（VAE）
**预计用时：** ~75 分钟

## 问题所在

VAE 生成的样本模糊，因为其均方误差解码器损失对*均值*图像而言是贝叶斯最优的——而众多合理数字的平均值就是一个模糊的数字。你需要一种奖励*合理性*而非像素级接近某个特定目标的损失函数。合理性没有封闭形式的表达，必须通过学习获得。

古德费洛的构想：训练一个分类器 `D(x)` 来区分真实图像和伪造图像。训练一个生成器 `G(z)` 来欺骗 `D`。`G` 的损失信号来自于 `D` 当前判断什么是“真实”的依据。这个信号随着 `G` 的进步而更新，追逐一个移动的目标。如果两个网络都收敛，`G` 就学到了数据分布，而无需显式写出 `log p(x)`。

这就是对抗训练。其数学形式是一个极小极大博弈：

```
min_G max_D  E_real[log D(x)] + E_fake[log(1 - D(G(z)))]
```

到2026年，GANs 不再是顶尖的生成器（扩散模型和流匹配夺走了王冠）。但 StyleGAN 2/3 仍是已发布的面部模型中效果最清晰的，GAN 的判别器在扩散模型训练中被用作*感知损失*，对抗训练驱动着快速的单步蒸馏技术（SDXL-Turbo, SD3-Turbo, LCM），使得实时扩散成为可能。

## 核心概念

![GAN 训练：极小极大博弈中的生成器与判别器](../assets/gan.svg)

**生成器 `G(z)`。** 将噪声向量 `z ~ N(0, I)` 映射为样本 `x̂`。一个解码器形状的网络（全连接层或转置卷积层）。

**判别器 `D(x)`。** 将样本映射为一个标量概率（或分数）。真实 → 1，伪造 → 0。

**损失函数。** 交替进行两种更新：

- **训练 `D`：** `loss_D = -[ log D(x) + log(1 - D(G(z))) ]`。对真实样本（标签=1）和伪造样本（标签=0）计算二元交叉熵。
- **训练 `G`：** `loss_G = -log D(G(z))`。这是古德费洛使用的*非饱和*形式（原始形式 `log(1 - D(G(z)))` 在 `D` 自信时会饱和并导致梯度消失）。

**训练循环。** 进行一步 `D`，然后进行一步 `G`。重复此过程。

**为何能奏效。** 如果 `G` 完美匹配了 `p_data` 的分布，那么 `D` 的表现不会优于随机猜测，处处输出 0.5；`G` 获得的梯度消失。达到平衡。

**为何会失败。** 模式坍塌（`G` 找到一种 `D` 无法分类的模式并一直生成它），梯度消失（`D` 学得太快导致 `log D` 饱和），训练不稳定（学习率、批次大小等任何因素导致）。

## 使GANs成功运作的变体

| 年份 | 创新点 | 解决的问题 |
|------|--------|------------|
| 2015 | DCGAN | 使用卷积/转置卷积、批归一化、带泄露的ReLU —— 首个稳定的架构。 |
| 2017 | WGAN, WGAN-GP | 用Wasserstein距离 + 梯度惩罚替代二元交叉熵。解决梯度消失问题。 |
| 2017 | 谱归一化 | 对判别器进行Lipschitz约束。在2026年的判别器中仍在使用。 |
| 2018 | 渐进式GAN | 先在低分辨率下训练，逐步增加层数。首次生成百万像素级结果。 |
| 2019 | StyleGAN / StyleGAN2 | 映射网络 + 自适应实例归一化。固定领域真实感图像生成的当时最佳技术。 |
| 2021 | StyleGAN3 | 无混叠、平移等变 —— 至2026年仍是面部生成的黄金标准。 |
| 2022 | StyleGAN-XL | 条件化、类感知、更大规模。 |
| 2024 | R3GAN | 采用更强正则化的重命名版本；无需技巧即可处理1024²图像。 |

## 动手构建

`code/main.py` 在一维数据（两个高斯分布的混合）上训练一个微型GAN。生成器和判别器都是单隐藏层的多层感知机。我们手动实现前向传播、反向传播和极小极大循环。目的是观察两种关键失败模式（模式坍塌 + 梯度消失）是如何发生的。

### 步骤1：非饱和损失

原始的古德费洛损失 `log(1 - D(G(z)))`，当D以高置信度将G生成的伪造样本分类为假时，损失值趋向于0。此时，G的梯度基本为零——G无法改进。非饱和形式 `-log D(G(z))` 具有相反的渐近行为：当D非常自信时，损失值会爆炸式增长，从而给G一个强烈的梯度信号。

```python
def g_loss(d_fake):
    # maximize log D(G(z))  <=>  minimize -log D(G(z))
    return -sum(math.log(max(p, 1e-8)) for p in d_fake) / len(d_fake)
```

### 步骤2：每个生成器步骤对应一个判别器步骤

```python
for step in range(steps):
    # train D
    real_batch = sample_real(batch_size)
    fake_batch = [G(z) for z in sample_noise(batch_size)]
    update_D(real_batch, fake_batch)

    # train G
    fake_batch = [G(z) for z in sample_noise(batch_size)]  # fresh fakes
    update_G(fake_batch)
```

为G生成新的伪造样本，否则梯度会过时。

### 步骤3：观察模式坍塌

```python
if step % 200 == 0:
    samples = [G(z) for z in sample_noise(500)]
    mode_a = sum(1 for s in samples if s < 0)
    mode_b = 500 - mode_a
    if min(mode_a, mode_b) < 50:
        print("  [!] mode collapse: one mode is starved")
```

典型症状：两个真实模式中的一个停止被生成。判别器停止纠正它，因为它从未被当作伪造样本见过。

## 常见陷阱

- **判别器过强。** 将D的学习率降低2-5倍，或为D的输入添加实例/层噪声。如果D的准确率超过95%，G就失效了。
- **生成器记忆住一种模式。** 给D的输入添加噪声，使用小批次判别层，或改用WGAN-GP。
- **批归一化泄露统计信息。** 真实批次和伪造批次流经同一个BN层，混合了它们的统计量。应改用实例归一化或谱归一化。
- **刷Inception Score。** FID和IS在样本量少时噪声很大。评估时请使用≥10k个样本。
- **条件任务中单次采样是不现实的。** 你仍然需要分类器自由引导缩放、截断技巧和重采样才能获得可用的输出。

## 如何使用

2026年的GAN技术栈：

| 场景 | 推荐选择 |
|------|----------|
| 真实感人物面部，固定姿势 | StyleGAN3（最清晰，最轻量） |
| 动漫/风格化面部 | StyleGAN-XL 或 Stable Diffusion LoRA |
| 图像到图像转换 | Pix2Pix / CycleGAN（阶段 8 · 04）或 ControlNet（阶段 8 · 08） |
| 快速单步文生图 | 扩散模型的对抗蒸馏（SDXL-Turbo, SD3-Turbo） |
| 扩散模型训练器内的感知损失 | 用于图像裁剪的小型GAN判别器 |
| 任何多模态、开放式任务 | 不要使用GAN——请用扩散模型或流匹配 |

GANs效果锐利但应用范围窄。一旦你的领域扩大——照片、任意文本提示、视频——请转向扩散模型。对抗性技巧作为组件（感知损失、蒸馏）得以延续，而非作为独立的生成器。

## 练习

1. **简单。** 使用默认设置运行 `code/main.py`。然后设置 `D_LR = 5 * G_LR` 并重新运行。G的损失多快会坍塌为一个常数？
2. **中等。** 将古德费洛的二元交叉熵损失替换为WGAN损失：`loss_D = E[D(fake)] - E[D(real)]`, `loss_G = -E[D(fake)]`，并将D的权重裁剪到 `[-0.01, 0.01]`。训练是否更稳定？比较墙钟时间的收敛速度。
3. **困难。** 将一维示例扩展到二维数据（环形分布的8个高斯混合）。跟踪在1k、5k、10k步时，生成器捕捉到了8个模式中的多少个。实现小批次判别并重新测量。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------|----------|
| 生成器 (Generator) | "G" | 从噪声到样本的网络，`G: z → x̂`。 |
| 判别器 (Discriminator) | "D" | 分类器 `D: x → [0, 1]`，区分真与假。 |
| 极小极大 (Minimax) | "那个博弈" | 对联合目标函数进行 `min_G max_D`。 |
| 非饱和损失 (Non-saturating loss) | "那个修复方法" | 为G使用 `-log D(G(z))` 而非 `log(1 - D(G(z)))`。 |
| 模式坍塌 (Mode collapse) | "G只记住了一种东西" | 生成器尽管数据多样，却只产出少量不同的输出。 |
| WGAN | "Wasserstein" | 用推土机距离 + 梯度惩罚替代二元交叉熵；梯度更平滑。 |
| 谱归一化 (Spectral norm) | "Lipschitz技巧" | 约束D的权重范数以限制其斜率；稳定训练。 |
| StyleGAN | "那个好使的" | 映射网络 + 自适应实例归一化；面部生成的最佳选择，至2026年依然如此。 |

## 生产注意事项：单步推理是GAN的持久优势

在开放域生成中，GANs 不再在样本质量上胜出，但在推理成本上依然领先。用生产-推理文献的术语来说，GAN具有：

- **无预填充，无解码阶段。** 一次 `G(z)` 前向传播。TTFT（首个token生成时间）≈ 总延迟。
- **无KV缓存压力。** 唯一的状态是模型权重。批次大小受限于激活内存，而非缓存。
- **简单的连续批处理。** 由于每个请求消耗相同的固定FLOPs，在服务器目标占用率下的静态批处理通常是最优的。无需复杂的在途调度器。

这就是为什么在2026年，GAN蒸馏（SDXL-Turbo, SD3-Turbo, ADD, LCM）是快速文生图的主导技术：它将20-50步的扩散流程压缩为1-4次类GAN的前向传播，同时保持了扩散基础模型的分布。对抗性损失作为一种训练时的调节手段得以延续，用于将慢速生成器转变为快速生成器。
