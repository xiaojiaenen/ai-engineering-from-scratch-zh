# 红队测试：PAIR与自动化攻击

> Chao, Robey, Dobriban, Hassani, Pappas, Wong (NeurIPS 2023, arXiv:2310.08419)。PAIR——提示自动迭代精炼——是标准化的自动化黑盒越狱方法。攻击者LLM（配备红队系统提示）为一个目标LLM迭代生成越狱提示，并将尝试次数和回应累积到其自身聊天记录中作为上下文反馈。PAIR通常在20次查询内成功，效率比GCG（Zou等人的token级梯度搜索）高出数个数量级，并且无需白盒访问权限。PAIR现在是JailbreakBench（arXiv:2404.01318）和HarmBench中的标准基线，与GCG、AutoDAN、TAP和说服性对抗提示并列。

## 学习目标

- 描述PAIR算法：攻击者系统提示、迭代精炼、上下文反馈。

## 问题背景

红队测试曾经是一项手动活动。少数专家测试人员构建对抗性提示并跟踪哪些有效。这种方法无法扩展：攻击成功率需要统计样本，而且随着每次模型发布，目标也在不断变化。PAIR将红队测试构建为一个针对黑盒目标的优化问题。

## 核心概念

### PAIR算法

输入：
- 目标LLM T（我们正在攻击的模型）。
- 评判LLM J（评估回应是否构成越狱）。
- 攻击者LLM A（红队优化器）。
- 目标字符串G：“回应[有害指令]”。
- 预算K（通常为20次查询）。

循环，对于k从1到K：
1. 使用目标G和迄今为止的（提示，回应）对历史记录提示A。
2. A生成一个新的提示p_k。
3. 将p_k提交给T；接收回应r_k。
4. J根据目标对（p_k, r_k）进行评分。
5. 如果分数 >= 阈值，停止——找到越狱方法。
6. 否则，将（p_k, r_k）附加到A的历史记录；继续。

实证结果（NeurIPS 2023）：针对GPT-3.5-turbo、Llama-2-7B-chat的攻击成功率超过50%；成功所需的平均查询次数在10-20次之间。

### 为什么PAIR高效

GCG（Zou等人 2023）通过梯度搜索对抗性token后缀；它需要白盒模型访问权限，并产生难以阅读的后缀。PAIR是黑盒的，并产生可跨模型转移的自然语言攻击。PAIR的上下文反馈让攻击者能从每次拒绝中学习；GCG没有等价机制（每次新的token更新都必须重新发现先前的进展）。

### 相关的自动化攻击

- **GCG（Zou等人 2023, arXiv:2307.15043）。** 针对抗性后缀的token级梯度搜索。白盒、可转移，产生难以阅读的字符串。
- **AutoDAN（Liu等人 2023）。** 在提示上进行进化搜索，由分层目标指导。
- **TAP（Mehrotra等人 2024）。** 带剪枝的攻击树——分支多个类似PAIR的展开。
- **PAP（Zeng等人 2024）。** 说服性对抗提示——将人类说服技巧编码为提示模板。

### JailbreakBench和HarmBench

两者（2024）均标准化了评估：

- JailbreakBench（arXiv:2404.01318）。跨10个OpenAI政策类别的100种有害行为。攻击成功率（ASR）作为主要指标。需要一个评判（GPT-4-turbo、Llama Guard或StrongREJECT）。
- HarmBench（Mazeika等人 2024）。跨7个类别的510种行为，包含语义和功能危害测试。比较18种攻击与33个模型。

ASR通常在固定的查询预算下报告。比较攻击需要匹配预算；在200次查询下90%的ASR与20次查询下85%的ASR无法直接比较。

### 对于2026年部署的重要性

每个前沿实验室现在都在发布前针对生产模型运行PAIR和TAP。ASR轨迹出现在模型卡（课程26）和安全案例附录（课程18）中。这种攻击并不奇特——它是标准基础设施。

## 动手实践

`code/main.py`构建了一个玩具PAIR循环。目标是一个拒绝“明显”有害提示（关键词过滤器）的模拟分类器。攻击者是一个基于规则的精炼器，尝试改述、角色扮演框架化和编码。评判对回应进行评分。你可以看到攻击者在大约5-15次迭代内成功突破关键词过滤器，但在语义过滤器前失败。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------|----------|
| PAIR | “自动化越狱” | 提示自动迭代精炼；攻击者LLM + 评判LLM循环 |
| GCG | “梯度越狱” | 针对抗性后缀的白盒token级梯度搜索 |
| 攻击成功率 (ASR) | “k次查询时的越狱百分比” | 主要指标；必须与查询预算和评判标识一起报告 |
| 评判LLM | “评分者” | 对回应是否满足有害目标进行评分的LLM |
| JailbreakBench | “评估” | 带标签类别的标准化有害行为集 |
| HarmBench | “更广泛的基准” | 510种行为，功能+语义危害测试 |
| TAP | “攻击树” | 带分支+剪枝的PAIR；在更高计算成本下获得更好的ASR |
