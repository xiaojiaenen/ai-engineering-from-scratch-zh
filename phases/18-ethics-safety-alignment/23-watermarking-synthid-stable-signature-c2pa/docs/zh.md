# 水印技术 —— SynthID、Stable Signature、C2PA

> 三项技术构建了2026年AI生成内容的溯源体系。SynthID（谷歌DeepMind）—— 图像水印于2023年8月推出，文本+视频水印于2024年5月推出（Gemini + Veo），文本水印于2024年10月通过负责任生成式AI工具包开源，统一的多媒体检测器于2025年11月随Gemini 3 Pro一同发布。文本水印通过微不可察地调整下一个token的采样概率实现；图像/视频水印能抵抗压缩、裁剪、滤镜、帧率变化。Stable Signature（Fernandez等人，ICCV 2023，arXiv:2303.15435）—— 微调解码器的隐式扩散解码器，使得每个输出都包含固定信息；裁剪（保留10%内容）后的生成图像在假阳性率FPR<1e-6下，检测率>90%。后续研究“Stable Signature并不稳定”（arXiv:2405.07145，2024年5月）—— 微调可在保持质量的同时移除水印。C2PA —— 采用密码学签名、防篡改的元数据标准（C2PA 2.2说明文档2025）。水印与C2PA是互补的：元数据可被剥离，但承载更丰富的溯源信息；水印在转码过程中持续存在，但携带的信息较少。

## 问题背景

2023-2024年，深度伪造和AI生成内容大规模进入政治和消费领域。水印是所提出的技术溯源信号：在生成时标记，后续检测。2025年的证据表明：没有任何水印具有绝对鲁棒性，但与C2PA元数据结合使用时，这种组合提供了可用的溯源故事。

## 核心概念

### 文本水印（SynthID文本风格）

Kirchenbauer等人2023年的机制，由谷歌实现生产化：

1. 在每个解码步骤，对前K个token进行哈希，生成词汇表的一个伪随机划分，分为“绿色”和“红色”两个集合。
2. 通过对绿色集合的logit值增加δ，使采样偏向绿色集合。
3. 生成内容包含比随机情况更多的绿色token。

检测：对每个前缀重新哈希，计算生成内容中绿色token的数量，计算z分数。对于带水印的文本，z分数 >0；对于人类文本，z分数 ≈0。

特性：
- 读者不可感知（δ足够小，质量损失轻微）。
- 在已知词汇表划分函数的情况下可检测。
- 对转述不鲁棒——重写文本会破坏信号。

SynthID文本于2024年10月通过谷歌的负责任生成式AI工具包开源。

### Stable Signature（图像）

Fernandez等人，ICCV 2023。微调隐式扩散解码器，使每个生成图像都在隐式表示中嵌入固定二进制信息。检测通过神经解码器从隐式表示中解码。裁剪（至10%内容）后的图像在FPR<1e-6下，检测率>90%。

2024年5月“Stable Signature并不稳定”（arXiv:2405.07145）：微调解码器可在保持图像质量的同时移除水印。对抗性的生成后微调成本低廉；水印的对抗鲁棒性有限。

### SynthID统一检测器（2025年11月）

随Gemini 3 Pro一同发布：一个多媒体检测器，可通过一个API读取来自文本、图像、音频和视频的SynthID信号。统一了谷歌的溯源技术栈。

### C2PA

内容溯源与真实性联盟。采用密码学签名的防篡改元数据标准。C2PA 2.2说明文档（2025）。C2PA清单记录溯源声明（创建者、时间、进行了哪些转换），由创建者的密钥签名。

与水印互补：
- 元数据可被剥离；水印（通常）不能。
- 元数据信息丰富（完整的溯源链）；水印携带比特信息。
- C2PA依赖平台采纳；水印自动嵌入。

谷歌在搜索、广告和“关于此图片”功能中整合了两者。

### 局限性

- **模型特定性。** SynthID为启用该功能的模型生成的文本添加水印。没有启用SynthID的模型生成的文本不带水印，因此“没有SynthID信号”不能证明内容是真实的。
- **转述。** 文本水印无法在保持原意的转述中存活。
- **转换攻击。** arXiv:2508.20228（2025）展示了能同时破坏文本水印和许多图像水印的保义攻击。
- **微调移除。** 根据“Stable Signature并不稳定”，生成后的微调可以移除嵌入的水印。

## 练习

1. 运行`code/main.py`。报告带水印的1000-token生成内容与人类创作文本的z分数。在95%置信阈值下识别假阳性率。
2. 实现一个转述攻击，替换30%的token为同义词。重新测量z分数。
3. 阅读Kirchenbauer等人2023年论文第6节关于鲁棒性的内容。为什么文本水印在转述下失效，而图像水印能抵抗裁剪？
4. 设计一个使用SynthID文本+C2PA元数据的部署方案。描述消费者看到的溯源链。识别每个组件的一个失效模式。
5. 2024年的“Stable Signature并不稳定”结果显示微调可以移除图像水印。设计一个限制此攻击的部署控制措施——例如，要求微调后的检查点必须经过签名发布。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|------------|----------|
| SynthID | "谷歌的水印" | 跨模态溯源信号；覆盖文本、图像、音频、视频 |
| Token水印 | "Kirchenbauer风格" | 偏置采样文本水印，可通过绿色token z分数检测 |
| Stable Signature | "图像水印" | 微调解码器水印；ICCV 2023 |
| C2PA | "元数据标准" | 采用密码学签名、防篡改的溯源元数据 |
| 转述鲁棒性 | "改写是否会破坏它" | 文本水印特性；目前有限 |
| 微调移除 | "对抗性去水印" | 通过解码器微调移除图像水印的攻击 |
| 跨模态检测器 | "统一的SynthID" | 2025年11月跨模态的统一API |
