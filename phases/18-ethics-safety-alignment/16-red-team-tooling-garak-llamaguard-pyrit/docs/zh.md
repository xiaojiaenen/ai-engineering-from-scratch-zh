# 红队工具 — Garak、Llama Guard、PyRIT

> 三个生产级工具构成了2026年的红队技术栈。Llama Guard（Meta） — 一个基于 Llama-3.1-8B 微调的分类器，针对 MLCommons 的14类危险内容；2025年的 Llama Guard 4 是一个原生多模态的12B分类器，由 Llama 4 Scout 裁剪而来。Garak（NVIDIA） — 开源的 LLM 漏洞扫描器，具备针对幻觉、数据泄露、提示注入、毒性和越狱攻击的静态、动态和自适应探测器。PyRIT（Microsoft） — 通过 Crescendo、TAP 和自定义转换器链进行多轮红队演练，实现深度漏洞利用。Llama Guard 3 的文档见 Meta 的 “Llama 3 Herd of Models” (arXiv:2407.21783)；Llama Guard 3-1B-INT4 见 arXiv:2411.17713；Garak 的探测器架构见 github.com/NVIDIA/garak。这些工具是红队研究（课程12-15）与部署（课程17+）之间的2026年生产接口。

## 核心概念

### Llama Guard (Meta)

Llama Guard 3 是一个基于 Llama-3.1-8B 微调的模型，用于针对 MLCommons AILuminate 的14个类别进行输入/输出分类：
- 暴力犯罪、非暴力犯罪、性相关、儿童性虐待材料、诽谤
- 专业建议、隐私、知识产权、无差别武器、仇恨
- 自杀/自残、性内容、选举、代码解释器滥用

支持8种语言。使用方法：可置于 LLM 之前（输入审核）、之后（输出审核），或两者兼有。这两种用法产生不同的训练数据分布 — Llama Guard 3 作为单一模型同时处理两者。

Llama Guard 3-1B-INT4（arXiv:2411.17713，440MB，移动CPU上约30 tokens/s）是量化后的边缘版本。

Llama Guard 4（2025年4月）为12B，原生多模态，由 Llama 4 Scout 裁剪而来。它用一个能够处理文本和图像的分类器，取代了之前的8B文本和11B视觉模型。

### Garak (NVIDIA)

开源的漏洞扫描器。架构：
- **探测器。** 针对幻觉、数据泄露、提示注入、毒性、越狱的攻击生成器。包括静态（固定提示）、动态（生成提示）和自适应（根据目标输出响应）类型。
- **检测器。** 根据预期的失败模式（如有毒、泄露、被越狱）对输出进行评分。
- **测试工具集。** 管理探测器-检测器对，运行演练，生成报告。

TrustyAI 将 Garak 与 Llama-Stack 的防护栏（Prompt-Guard-86M 输入分类器，Llama-Guard-3-8B 输出分类器）集成，用于端到端的受防护目标评估。基于层级的评分（TBSA）取代了二元的通过/失败 — 同一个探测器下，模型可能在严重性3级通过，在5级失败。

### PyRIT (Microsoft)

Python 风险识别工具包。多轮红队演练。核心组件：
- **转换器。** 变换种子提示 — 改述、编码、翻译、角色扮演。
- **编排器。** 运行演练：Crescendo（逐步升级）、TAP（分支探索）、RedTeaming（自定义循环）。
- **评分。** LLM 作为评判器或分类器作为评判器。

PyRIT 是 Garak 更重量级的同类。Garak 运行数千个单轮探测；PyRIT 运行旨在突破特定失败模式的深度多轮演练。

### 技术栈

将 Llama Guard 放置在模型的两侧。每晚运行 Garak 进行回归测试。发布前运行 PyRIT 演练。这是2026年大多数生产部署的默认配置。

### 评估陷阱

- **评判器身份。** 三个工具都可以使用 LLM 评判器；评判器的校准会影响报告的攻击成功率（课程12）。需在工具旁注明使用的评判器。
- **探测器陈旧。** 随着模型针对攻击进行修补，Garak 的探测器会逐渐失效。自适应探测器（如 PAIR 形式）比静态探测器更耐久。
- **Llama Guard 对良性内容的误报率。** 早期 Llama Guard 版本对政治和 LGBTQ+ 内容过度标记；Llama Guard 3/4 的校准已改进，但并未针对每个部署进行校准。

## 练习

1. 运行 `code/main.py`。比较 Llama-Guard 风格分类器在单轮与多轮攻击上的检测率。
2. 实现一个新的 Garak 探测器：一个 base64 编码的有害请求。测量其被 Llama-Guard 风格分类器检测到的效果。
3. 用一个“先翻译成法语，再改述”的转换器扩展 PyRIT 风格的转换器链。重新测量攻击成功率。
4. 阅读 Llama Guard 3 的危险类别列表。找出两个类别，在这些类别上，训练数据很可能对合法的开发者内容产生高误报率。
5. 比较 Garak 和 PyRIT 的设计原则。论证一种部署场景，在其中每个工具都是最合适的选择。
