# 以遵循指令作为对齐信号

> 每一项后续对RLHF的批评都在反对此流程。在研究优化压力如何扭曲代理指标之前，你必须先看清这个代理指标本身。InstructGPT（Ouyang等人，2022）定义了参考架构：基于指令-响应对的有监督微调、在成对偏好排序上训练的奖励模型，以及针对奖励模型进行的PPO优化，并附加了对SFT策略的KL惩罚。一个13亿参数的InstructGPT模型在人类偏好评估中胜过了1750亿参数的GPT-3。这一单一结果是2026年每个前沿实验室仍在发布RLHF形态后训练流程的原因。

**类型:** 学习
**语言:** Python (标准库, 玩具级三阶段流程)
**前置知识:** 第10章 · 06 (SFT), 第10章 · 07 (RLHF), 第10章 · 08 (DPO)
**时间:** ~45分钟

## 学习目标

- 列举InstructGPT流程的三个阶段及每个阶段使用的损失函数。
- 解释为什么一个13亿参数的指令微调模型在人类偏好评估中击败了原始的1750亿参数GPT-3。
- 说明第三阶段中KL惩罚所防范的问题，以及移除它为何会导致模式寻求行为。
- 描述对齐税以及Ouyang等人用于缓解此问题的PPO-ptx方法。

## 问题所在

预训练语言模型进行文本补全。它们不回答问题。让GPT-3 "编写一个反转列表的Python函数"，你通常会得到另一个提示，因为训练分布的大部分是继续生成更多网页文本的网页文本。模型在完成它的工作——只是这项工作本身是错的。

每个严肃实验室用来解决这个问题的代理指标是人类偏好。两个补全被送到评分员处；评分员选择更好的一个；一个奖励模型学习评分员的偏好。然后一个强化学习循环将策略调整为奖励模型打分较高的输出。这就是完整的InstructGPT论文主旨，三句话概括。论文的其余部分是工程细节。

## 核心概念

### 第一阶段：有监督微调 (SFT)

收集提示-响应对，其中响应是善意的人类会写的内容。Ouyang等人使用了来自标注员和OpenAI API的13000个提示。使用标准交叉熵损失在此数据上微调基础模型。

SFT带来的改变：模型现在回答问题，而不是继续问题。它未提供的：当存在多个合理答案时，关于评分员更偏好哪个答案的信号。

### 第二阶段：奖励模型 (RM)

对于每个提示，从SFT模型采样K个补全。标注员对它们进行排序。训练一个奖励模型，使其对任何提示-响应对进行评分，满足以下条件：对于`y_w`比`y_l`更受偏好的对：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

这是Bradley-Terry成对偏好损失。奖励模型通常从SFT模型初始化，但将语言模型头替换为标量头。

奖励模型通常较小：60亿参数就足以用于1750亿参数的InstructGPT。它们也很脆弱——论文的第5节主要讨论了在小规模上出现的奖励黑客行为。

### 第三阶段：带KL惩罚的PPO

定义目标：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

使用PPO最大化该目标。KL项防止`pi`偏离SFT策略太远。没有它，优化器会找到对抗性样本——在奖励模型下得分很高的字符串，因为奖励模型从未见过它们，而不是因为人类实际偏好它们。

KL系数`beta`是RLHF最重要的单一超参数。太低：导致奖励黑客。太高：相对于SFT没有改进。

### 对齐税

RLHF之后，模型更受人类偏好，但在标准基准测试（SQuAD, HellaSwag, DROP）上性能下降。Ouyang等人称其为对齐税，并用PPO-ptx解决：将预训练梯度混入RL目标，使模型不会遗忘它从未因之获得奖励的下游任务。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx已成为标准做法。Anthropic、DeepMind和Meta都使用了某种变体。

### 结果

一个13亿参数的InstructGPT（SFT + RM + PPO-ptx）在约70%的情况下被标注员评为优于1750亿参数的基础GPT-3。在来自生产流量的隐藏测试提示上，差距更大。可以从这个数字中看出两点：

1.  对齐是与能力不同的维度。1750亿参数模型能力更强；13亿参数模型对齐程度更高；标注员更偏好对齐的那个。
2.  能力下限由基础模型决定。你无法通过RLHF让基础模型知道它从未见过的事实。

### 为何这是第18章的参考点

后续课程中的每一项批评——奖励黑客（第2课）、DPO（第3课）、谄媚（第4课）、CAI（第5课）、潜伏代理（第7课）、对齐伪装（第9课）——都在反对此流程的某个部分。奖励黑客攻击第二阶段。DPO合并了第二和第三阶段。CAI取代了人类标注员。谄媚表明标注员是一个有偏的信号。对齐伪装表明策略可以完全绕过第三阶段。在不先了解这个流程的情况下，你无法理解这些批评中的任何一项。

## 实践应用

`code/main.py` 在玩具偏好数据上模拟了三个阶段。基础"策略"是一个在动作{A, B, C}上的有偏硬币。第一阶段SFT在200个提示上模仿标注员的动作。第二阶段从500个成对排序中拟合一个Bradley-Terry奖励模型。第三阶段运行一个简化的PPO更新，带有对SFT策略的KL惩罚。你可以观察奖励上升、KL散度增长以及策略漂移——并且可以关闭KL项，在50步更新内看到奖励黑客现象出现。

观察要点：

- `beta = 0.1` 与 `beta = 0.0` 的奖励轨迹。
- 训练步骤上的 KL(pi || pi_SFT)。
- 与标注员偏好相比的最终动作分布。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| SFT | "指令微调" | 第一阶段：在提示-响应对上进行交叉熵微调 |
| 奖励模型 | "the RM" | 在 (提示, 响应) 上的标量回归器，使用Bradley-Terry损失在成对标签上训练 |
| Bradley-Terry | "成对偏好损失" | -log sigmoid(r_w - r_l)；将成对排序简化为二分类 |
| KL惩罚 | "正则化器" | `beta * KL(pi || pi_SFT)` —— 将RL策略限制在SFT锚点附近 |
| PPO-ptx | "带预训练混合的PPO" | 将一部分预训练对数似然添加到PPO目标中，以抵消对齐税 |
| 对齐税 | "RLHF回归" | RLHF之后在标准基准测试上的性能下降，而这些基准并非RLHF的优化目标 |
| 标注员偏好 | "基本事实" | 人类排序的样本；RM是对此的统计代理，而非对“人类价值观”的代理 |
