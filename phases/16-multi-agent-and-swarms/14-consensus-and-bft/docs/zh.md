# 代理共识与拜占庭容错

> 经典分布式系统拜占庭容错遇上随机大语言模型。在2025-2026年，三个研究方向崭露头角：**CP-WBFT**（arXiv:2511.10400）通过置信度探测对每一票加权；**DecentLLMs**（arXiv:2507.14928）采用无领导并行工作提案与几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与分层结构聚类来区分核心与边缘节点。来自“AI代理能达成一致吗？”（arXiv:2603.01213）的诚实实证结果是：即便标量共识在今天也十分脆弱——一个欺骗性代理就能破坏混合代理系统。拜占庭容错是必要的，但非充分的。本课程构建一个最小化拜占庭容错协议，注入三种代理特定攻击（拜占庭谎言、谄媚一致性、关联错误单一文化），并测量每种共识变体的应对效果。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**先决条件：** 第16阶段 · 07（心智社会与辩论），第16阶段 · 13（共享内存）
**时间：** 约75分钟

## 问题

你有N个大语言模型代理，每个都给出一个答案。它们意见不一。多数投票选择了错误答案，因为两个代理是关联的（相同的基础模型、相同的训练数据、相同的失败模式）。第三个代理碰巧以一种新颖的方式出错——因此多数票成了虚假多数。

现在加入一个欺骗性代理：它故意撒谎。或者一个谄媚性代理：它同意最后发言者的观点。在经典拜占庭容错中，假设拜占庭节点占总节点的比例为`f < n/3`，且行为任意。2026年的现实是，大语言模型节点即便在诚实状态下也具有随机性，在不同模型间存在关联，并且受彼此输出的影响。你不能将它们视为独立的伯努利投票者。

经典拜占庭容错（PBFT，1999年）并非错误——它是不完整的。它能处理任意的比特翻转。它不能处理“三个诚实代理因共享训练数据而共享幻觉”的情况。本课程在PBFT基础上构建，并叠加三层2025-2026年的适应性方案。

## 概念

### 经典拜占庭容错的贡献

实用拜占庭容错（Castro & Liskov, OSDI 1999）能容忍`f < n/3`个拜占庭节点。该协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定人数证书）。在`n >= 3f + 1`个诚实或恶意节点间就单一值达成共识。

其保障很强，但假设：

1. **故障独立。** 拜占庭节点不会串通。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有真实答案。** 即使对错误事实达成共识，也仍是共识。

大语言模型代理违反了所有三条假设。两个运行相同基础模型的代理共享故障。一个“诚实”的大语言模型仍会产生幻觉。在模棱两可的问题上，“真相”就是代理们决定的东西——没有外部预言机。

### 三种针对大语言模型的攻击

**拜占庭谎言。** 一个代理故意输出错误答案。如果`f < n/3`，经典拜占庭容错可以处理此情况。

**谄媚一致性。** 一个代理在投票前阅读其他代理的答案，并与最后发言者保持一致。这并非恶意，但与最响亮的声音相关联。经典拜占庭容错无法阻止这一点，因为该代理通过了所有签名检查。

**关联错误单一文化。** 三个代理共享一个基础模型。它们对同一错误答案产生幻觉。多数票是错的。经典拜占庭容错无能为力，因为三个代理都“诚实地”同意了。

### 2025-2026年的应对方案

**CP-WBFT**（arXiv:2511.10400）— 置信度探测加权拜占庭容错。每位投票者在答案上附加一个置信度探测（一个自我报告的概率，或一个独立校准模型的预测）。投票权重随置信度缩放。报告在完整图上实现了+85.71%的拜占庭容错提升。缓解：谄媚一致性（谄媚代理倾向于在主动提出的意见上具有低置信度）。

**DecentLLMs**（arXiv:2507.14928）— 无领导者。工作者代理并行提出提案，评估代理对提案评分，最终答案是评分位置的几何中位数。在`f < n/2`时具有鲁棒性。缓解：拜占庭谎言和关联错误（几何中位数对离群值具有鲁棒性，并且倾向于密集集群，而非有偏的模型平均值）。

**WBFT**（arXiv:2505.05103）— 带分层结构聚类的加权拜占庭容错。投票权重根据回答质量加上从历史学习的信任分数来分配。将代理聚类为核心和边缘；核心代理必须先达成共识，边缘代理跟进。缓解：可扩展性（核心共识规模小、速度快）以及部分缓解单一文化（核心可选择多样性代理）。

### 实证研究：“AI代理能达成一致吗？”（arXiv:2603.01213）

该论文测量了跨多个前沿模型的标量一致性（大语言模型代理对单一数值达成一致）。结果令人不安：

- 即使在没有对抗者的情况下，大语言模型代理在许多基准测试上对标量问题的分歧率也超过30%。
- 一个采用欺骗性人格的单一代理，可以将混合代理的共识拉离诚实基准线40多个百分点。
- 分歧率与模型多样性相关——异构集成比同构集成分歧更多（好事：错误不相关），但漂移更慢（坏事：达成一致所需时间更长）。

启示是：拜占庭容错为你提供了对齐输出的机制，但它并不告诉你对齐的输出是否正确。应与验证（第16阶段 · 08 角色专业化）、多样性（第16阶段 · 15 辩论变体）和评估代理（第16阶段 · 24 基准测试）相结合。

### 核心协议，精简版

一个为大语言模型代理设计的最小化拜占庭容错轮次：

```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

语义聚类步骤是大语言模型特有的转折点。两个答案“研究报告为4.2%”和“4.2%的提升”属于同一聚类。朴素的字符串相等检查会遗漏这一点。在生产环境中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold`参数决定了何时接受，何时重试。阈值过低：你接受了微弱的多数。阈值过高：你永远无法接受任何结果。经验范围：对于`n=5-7`个代理，阈值为0.5-0.67；对于更小的`n`，阈值应更高。低于阈值时，升级给人工处理或交给不同的代理集成。

### 共识无能为力的情况

- **模棱两可的问题。** 如果问题没有真实答案，共识只是一种观点。就称之为观点。
- **复合问题。** “编写代码并解释它”——两个答案。对每个答案独立投票。
- **对抗性多轮次。** 如果代理能观察先前轮次并模仿（Du 2023 辩论），它们会开始相互同意，而不顾事实真相。限制轮次（通常2-3轮）。

## 构建它

`code/main.py` 实现了：

- `AgentVoter` — 带有（答案，置信度）的脚本化策略。
- `MajorityVote` — 经典多数票决。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 对评分提案进行几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个代理以高置信度撒谎。
2. `sycophancy`：一个代理复制它看到的第一个答案，并匹配置信度。
3. `monoculture`：三个代理共享一个错误答案（关联错误），并带有中等置信度。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）-> 最终答案的表格，其中正确答案被高亮。多数票决在单一文化案例中失败。CPWBFT的置信度加权缓解了谄媚性。DecentLLMs的几何中位数在单一文化少于半数人口时，会拉向诚实集群。

## 上线

在上线任何共识机制之前：

- **至少使用上述三种模式进行攻击测试。** 你的协议应可预测地失败，而非静默失败。
- **记录每个带有来源信息的少数集群。** 少数集群是你关联错误的早期预警系统。
- **强制限定轮次。** 不要“持续辩论直到达成一致”——那会助长谄媚。
- **将一致性与正确性分离。** 共识输出交给验证器；验证器独立于集成体。
- **监控一致率。** 急剧上升意味着一致性偏见；急剧下降意味着模型漂移。

## 练习

1. 运行`code/main.py`。确认多数票决在单一文化攻击中失败，但当单一文化置信度低于0.7时，CPWBFT部分缓解了该问题。
2. 添加第四种攻击模式：**静默弃权** — 一个代理拒绝回答（“我不知道”）。每个聚合器应如何处理弃权票？实现你的选择。
3. 将语义聚类从字符串规范化交换为嵌入相似度（使用任何开源嵌入模型）。这对谄媚性攻击有何影响？
4. 阅读CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（一个独立的校准模型检查每个代理的自我报告置信度）。测量在单一文化场景下的准确率提升。
5. 阅读“AI代理能达成一致吗？”（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个代理，一个标量问题，欺骗性人格提示。CPWBFT或DecentLLMs能捕捉到吗？

## 关键术语

| 术语 | 人们怎么说 | 其实际含义 |
|------|----------------|------------------------|
| BFT | “拜占庭容错” | Castro-Liskov 1999协议，用于在`f < n/3`个任意故障下达成共识。 |
| 拜占庭 | “任何不良行为” | 一个可以说谎、丢弃消息、静默故障的节点——除了安全崩溃外的任何行为。 |
| 置信度探测 | “你有多确定？” | 附加在投票上的自我报告或校准器预测的概率。 |
| 语义聚类 | “相同答案，不同措辞” | 在计票前对等价答案进行分组。 |
| 几何中位数 | “鲁棒中心” | 使到样本点距离之和最小的点。与平均值不同，它对离群值具有鲁棒性。 |
| 单一文化 | “相同模型，相同失败” | 当代理共享训练数据或基础模型时发生的关联错误。 |
| 谄媚一致性 | “同意响亮的声音” | 一个代理的投票偏向于第一个/最响亮发言的节点。 |
| 核心/边缘 | “分层拜占庭容错” | WBFT拆分：小规模核心共识先行，边缘节点跟进。限制延迟。 |
