# 投票、自一致性与辩论拓扑

> 最廉价的聚合方式：采样 N 个独立智能体，进行多数投票。Wang 等人在 2022 年提出的自一致性方法就是用同一个模型采样 N 次来实现的。多智能体方法通过引入**异构**智能体来扩展它，从而摆脱单一文化——不同的模型、不同的提示、不同的温度、不同的上下文。在多数投票之外，辩论拓扑也很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星型/链型/树型/图型协调方式，发现**图型在研究任务上表现最佳**，但超过约 4 个智能体后会出现"协调税"。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿者行为和从众行为——而从众行为既是一种特性（寻求共识），也是一种风险（群体思维，第 24 课）。本课将映射拓扑空间，构建每种变体，并衡量协调税。

**类型：** 学习 + 实践
**语言：** Python（标准库）
**前置知识：** 第 16 阶段 · 第 07 课（心智社会与辩论），第 16 阶段 · 第 14 课（共识与拜占庭容错）
**时间：** 约 75 分钟

## 问题

辩论可以提高准确性（Du 等人，arXiv:2305.14325）。也可能降低准确性。辩论是否有益取决于四个结构性选择：

1.  谁与谁交谈（拓扑结构）。
2.  进行多少轮（Du 2023：轮次和智能体数量都是独立的重要因素）。
3.  智能体是否异构（不同的基础模型可以打破单一文化）。
4.  是否存在对抗性声音（强化论证 vs. 虚拟论证）。

那些简单地在任务上附加"运行 5 个智能体并投票"的团队，其表现常常不如单个智能体。这些失败并非随机。它们与拓扑和异构性相关。本课就是拓扑地图。

## 概念

### 自一致性，单模型基线

Wang 等人在 2022 年（"自一致性改进思维链推理"）在温度 > 0 时对同一模型进行 N 次采样，并对推理路径的答案进行多数投票。在 GSM8K 上的结果：N=40 次采样比单次贪婪解码有显著收益。自一致性是单智能体多智能体投票的前身。

局限：自一致性使用一个基础模型。错误在结构上是相关的。如果模型存在系统性偏差，所有 N 个采样都会共享它。

### 多智能体投票，异构扩展

用 N 个**不同的**智能体替换 N 个采样。不同的基础模型（Claude、GPT、Llama），不同的提示，不同的工具访问权限。好处：不相关的错误。代价：不同的智能体成本不同；协调它们会增加开销。

异构辩论的 2026 年规范名称是 **A-HMAD** —— 对抗性异构多智能体辩论。并未被普遍采用，但论文用该术语指代"不同模型进行辩论，这减少了单一文化崩溃导致的相关错误"。

### 四种拓扑结构

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

**星型：** 一个中心，所有其他节点只与中心通信。等同于没有反馈通道的监督者-工作者模式。
**链型：** 线性，每个智能体看到前一个智能体的输出。类似流水线。
**树型：** 层级式，由层级智能体系统使用（第 06 课）。
**图型：** 任意对任意。包括全连接团和任意有向无环图。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在一套包含研究、编码和规划的任务套件上，对星型、链型、树型、图型进行了基准测试。关键测量结果：

-   **图型**拓扑在研究任务上获胜。信息任意对任意流动；智能体可以互相批评。
-   **星型**在快速回答的事实性任务上获胜。中心过滤并整合。
-   **链型**在逐步流水线（分阶段优化）上获胜。
-   **协调税**在图型拓扑中，超过约 4 个智能体后出现。挂钟时间和 token 成本增长快于质量。

4 个智能体的上限是经验性的，不是根本性的。它反映了 2026 年 LLM 的上下文容量：每个智能体的上下文都填满了其他智能体的输出，并且一旦每个人都能看到所有人，添加第 N+1 个智能体的边际价值就会下降。

### 多智能体辩论策略（"我们应该采用 MAD 吗？"）

arXiv:2311.17371 是 2023 年关于 MAD 策略的综述。关键发现（被其他研究证实）：与自一致性在**结构上相似**的 MAD 变体（独立采样 + 聚合）在使用相同预算时，其表现常常不如自一致性。MAD 在智能体真正异构且辩论具有对抗性结构（一个智能体进行反驳）时帮助最大。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了即使没有显式设计，也会从多智能体辩论中涌现的两种行为：

-   **志愿者。** 智能体主动提供帮助（"我可以进行下一步"）。有用：它将工作分配给子任务能力最强的智能体。
-   **从众。** 智能体调整立场以迎合批评者，即使批评者是错误的。这是辩论中的阿谀奉承（第 14 课）。

从众行为是为什么"辩论直到达成一致"会奖励霸凌者的原因。限制轮次并设置独立的裁判可以缓解。

### 异构性：真正影响准确性的旋钮

2024-2026 年实用文献中的一个模式：将你的 N 个智能体中的一个替换为不同的基础模型，比增加 1 个 N 带来的准确性提升更大。直觉是单一文化——每个新的独立错误源比一个额外的相关采样更有价值。

在极限情况下，异构性优于数量。在大多数具有明确真实答案的任务上，三个不同的模型胜过五个同一模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了"陪审团"——一组小型的专业智能体，通过在每个阶段投票来优化答案。与简单的多数投票不同，陪审团有角色分工：一个智能体进行交叉询问，一个提供上下文，一个评估合理性。陪审团方法是介于简单投票（廉价，易受单一文化影响）和完整 MAD（昂贵，易受从众影响）之间的中间点。

### 何时投票加辩论占主导

-   问题存在真实答案（事实、数学、代码行为）。投票收敛是有意义的。
-   智能体可以访问不同的来源或工具（异构性可用）。
-   轮次有限制（通常为 2-3 轮）并有独立的裁判或验证器。
-   预算允许 3-5 个智能体。在图型拓扑中超过 5-7 个，协调税将占主导。

### 何时投票加辩论有害

-   问题是观点型的。智能体会收敛到看起来最自信的答案，而非最正确的答案。
-   所有智能体共享一个基础模型。单一文化使得共识毫无意义。
-   轮次无限制。从众行为总会获胜。
-   任务很简单。一个使用自一致性（N=5）的单个智能体更便宜且同样准确。

## 构建它

`code/main.py` 实现了：

-   `run_star(agents, hub, question)` —— 中心轮询每个工作者，进行聚合。
-   `run_chain(agents, question)` —— 顺序优化。
-   `run_tree(root, children, question)` —— 具有深度-2 聚合的层级结构。
-   `run_graph(agents, question, rounds)` —— 全对全辩论，有限轮次。
-   一个脚本化的异构性调节：每个智能体有一个 `error_bias` 表示其系统性错误。
-   一个测量工具，在 N=3、5、7 时运行每种拓扑，并报告（准确性、总 token 数、模拟挂钟时间）。

运行：

```
python3 code/main.py
```

预期输出：一个拓扑 × N → （准确性、token 数、延迟）的表格。图型在 N=3-5 的研究型任务上获胜；星型在快速事实任务上获胜；图型在 N=7 时显示协调税（延迟膨胀快于准确性）。

## 使用它

`outputs/skill-topology-picker.md` 是一项技能，它读取任务描述并推荐一种拓扑（星型/链型/树型/图型）、一个 N（智能体数量）、一个异构性配置（要使用的基础模型）和一个轮次限制。

## 部署它

对于任何集成：

-   首先使用**一个强大的基础模型进行 N=5 的自一致性**。这是廉价的基线。
-   如果准确性很重要，则升级为**N=3 的异构投票**。衡量差异。
-   仅当任务具有结构（研究、多步骤）且可行的有限轮次时，才升级为**辩论拓扑**。
-   始终记录少数派群体。当少数派持续正确时，你获得了一个多样性信号。
-   在衡量准确性的同时，基准测试挂钟时间和 token 数。"准确性提高 10 倍但成本增加"是一个商业决策。

1.  运行 `code/main.py`。绘制图型拓扑的协调税曲线：准确性 vs N，token 数 vs N。曲线在哪个 N 处拐点？
2.  实现 A-HMAD：三个具有刻意不同偏差的智能体。在第 14 课的单一文化攻击上，全部相同偏差的基线与 A-HMAD 相比如何？
3.  在图型拓扑中添加一个"裁判"角色，该角色不投票，只对最终共识进行评分。这是否会改变涌现的从众行为？
4.  阅读 AgentVerse 论文（ICLR 2024）。识别你的实现最强烈地表现出哪种涌现行为。你能否通过改变提示引发相反的行为？
5.  阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。使用你的工具包，在论文中的一个任务上复现"图型赢得研究"的结果。

## 关键术语

| 术语 | 人们如何说 | 它的实际含义 |
|------|------------|--------------|
| 自一致性 | "采样 N 次，投票" | Wang 2022。单模型，N 次温度>0 的采样，对推理路径进行多数投票。 |
| 异构性 | "不同的模型" | 不同基础模型或提示系列的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在多轮次中交换批评的通用术语。见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | MAD 变体，强调不同模型 + 对抗性结构。 |
| 拓扑 | "谁与谁交谈" | 星型、链型、树型、图型。决定信息流。 |
| 协调税 | "收益递减" | 在图型上超过约 4 个智能体后，成本增长快于质量。 |
| 志愿者行为 | "主动帮助" | AgentVerse 涌现模式：智能体主动提出执行一个步骤。 |
| 从众行为 | "压力下的一致" | AgentVerse 涌现模式：智能体与批评者保持一致。 |
| 陪审团 | "小型专业小组" | Sibyl 风格的具有角色（检查员、上下文提供者、评分者）的集成。 |
