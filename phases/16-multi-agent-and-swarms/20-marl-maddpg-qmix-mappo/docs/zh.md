# MARL — MADDPG, QMIX, MAPPO

> 多智能体协调技术源于强化学习，这一遗产至今仍影响着2026年的LLM智能体系统。**MADDPG** (Lowe等人, NeurIPS 2017, arXiv:1706.02275) 提出了中心化训练、去中心化执行 (CTDE) 的范式：训练期间每个评论家都能观察到所有智能体的状态和动作；测试时仅本地执行者运行。适用于合作、竞争及混合场景。**QMIX** (Rashid等人, ICML 2018, arXiv:1803.11485) 是一种价值分解方法，采用单调混合网络；每个智能体的Q值组合成联合Q值，使得 `argmax` 能够清晰分配——在星际争霸多智能体挑战赛 (SMAC) 中表现卓越。**MAPPO** (Yu等人, NeurIPS 2022, arXiv:2103.01955) 是采用中心化价值函数的PPO算法；在粒子世界、SMAC、谷歌研究足球、Hanabi游戏中均展现出“惊人的有效性”，且调参需求极低。这些算法共同构成了必须去中心化执行的智能体团队策略训练的基础。MAPPO是**2026年合作型MARL的默认基准方法**。本课程将从一个小型网格世界示例入手，逐步构建这三种算法，并在接触LLM智能体训练前，将这三种核心思想内化为肌肉记忆。

## 问题

LLM智能体系统日益需要训练用于智能体间协调的策略：何时延迟行动、何时执行动作、调用哪个同伴。指导如何训练此类策略的文献是多智能体强化学习 (MARL)，它早于LLM浪潮，并拥有一套主导算法。

在没有模式词汇库的情况下阅读MARL论文会令人痛苦。中心化训练与去中心化执行 (CTDE)、价值分解和中心化评论家并非流行词——它们是针对特定问题的具体解决方案：

- 独立强化学习 (每个智能体单独学习) 从每个智能体的视角来看是非平稳的。效果不佳。
- 中心化强化学习 (一个智能体控制所有) 无法扩展且违反执行约束。
- CTDE则兼得二者之长：训练时利用全局信息，部署时使用本地策略。

## 概念

### 论文使用的三种环境

- **粒子世界 (多智能体粒子环境)。** 具有合作/竞争任务的简单2D物理环境。MADDPG的原始测试平台。
- **星际争霸多智能体挑战赛 (SMAC)。** 合作型微观管理，部分可观察。QMIX的测试平台。离散动作，连续状态。
- **谷歌研究足球、Hanabi、MPE。** MAPPO的基准环境。

不同环境具有不同的动作/观察类型。算法会据此选择。

### MADDPG (2017) — CTDE模式

每个智能体 `i` 有一个执行者 `mu_i(o_i)`，将自身观察映射为动作。每个智能体还有一个评论家 `Q_i(x, a_1, ..., a_n)`，训练期间能看到所有观察和动作。执行者通过基于评论家评估的策略梯度进行更新。

```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

为何采用CTDE：训练时，我们知道所有人的动作；利用这些信息可以减少每个评论家的方差。部署时，每个智能体只看到 `o_i` 并调用 `mu_i(o_i)`。

失效模式：评论家的输入随智能体数量N增长 (输入包含所有动作)。若不采用近似方法，当智能体数量超过约10个时无法扩展。

### QMIX (2018) — 价值分解

仅适用于合作场景。全局奖励是每个智能体Q值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了 `argmax_a Q_tot` 可以通过每个智能体独立选择 `argmax_{a_i} Q_i` 来计算。这**正是你所需要的去中心化执行特性**。训练时，一个混合网络根据每个智能体的Q值生成 `Q_tot`。

为何QMIX在SMAC上获胜：合作型星际争霸微观管理具有同质智能体、局部观察、全局奖励——完美契合价值分解。

失效模式：单调性约束具有限制性；某些任务的奖励结构并非单调可分解 (例如一个智能体为团队牺牲)。扩展算法 (QTRAN, QPLEX) 放宽了此限制。

### MAPPO (2022) — 被忽视的默认方法

多智能体PPO：采用中心化价值函数的PPO。每个智能体有自己的策略；所有智能体共享 (或拥有各自的) 能观察全局状态的价值函数。Yu等人在2022年将MAPPO与MADDPG、QMIX及其扩展算法在五个基准上进行了比较，发现：

- MAPPO在粒子世界、SMAC、谷歌研究足球、Hanabi、MPE上匹配或超越了离策略MARL方法。
- 所需的超参数调优极少。
- 训练稳定；跨随机种子可复现。

社区在此论文之前一直低估了在线策略MARL。在2026年，MAPPO是合作型MARL的默认基准；任何新方法都必须超越它。

### 为何LLM智能体工程师应关注

三个直接用途：

1. **路由器训练。** 一个元智能体选择哪个子智能体处理任务。这是一个包含N个去中心化子智能体和一个中心化路由器的MARL问题。MAPPO很适合。
2. **角色涌现。** 在生成式智能体模拟中，训练智能体随时间采纳互补角色是一个伪装的MARL问题。QMIX式的价值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过CTDE训练它们可以产生可部署的本地策略，这些策略能遵守资源约束。

实际注意事项：在2026年，大多数生产级LLM智能体系统通过提示 (prompt) 而非训练来定义策略。MARL在以下情况发挥作用：(a) 拥有大量交互数据，(b) 有明确的奖励信号，(c) 愿意投资训练基础设施。

### CTDE作为超越强化学习的设计模式

即使不进行训练，CTDE也是一个有用的架构模式：

- 在*设计*阶段，假设团队完全可见。
- 在*运行时*，强制执行去中心化执行：每个智能体只看到 `o_i`。

该模式迫使你明确每个智能体的状态，并预先考虑部分可观察性。许多生产级多智能体系统默默假设处处共享状态——CTDE规范可以防止这种情况。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境 (包括其他智能体的策略) 是非平稳的。经典的单智能体强化学习证明失效。本课程中的MARL算法都解决了这个问题：

- MADDPG：全局评论家能看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习转移到联合Q空间，在该空间中最优性定义明确。
- MAPPO：中心化价值函数抑制了其他智能体策略变化带来的方差。

在LLM智能体系统中，非平稳性表现为“我的智能体上个月还工作正常，现在上游的另一个智能体变了，我的就出问题了”。使用CTDE训练MARL是根本性的解决方案；提示级别的修复更快但不够持久。

### 本课程未涵盖的内容

实际训练网络是第09阶段的主题。本课程构建的是演示CTDE、价值分解和中心化价值模式的脚本化策略版本，不涉及梯度更新。目标是在你使用完整的MARL库 (PyMARL, MARLlib, RLlib多智能体) 之前，先内化这些模式。

## 动手构建

`code/main.py` 实现了三种模式的演示，全部基于一个微小的2智能体合作网格世界：

- 环境：2个智能体在4x4网格上，一个奖励点。任何智能体到达奖励点则奖励=1；任务完成。
- `IndependentAgents` — 每个智能体将其他智能体视为环境。基线。
- `MADDPGStyle` — 中心化评论家计算联合价值；执行者策略据此更新。脚本化策略改进。
- `QMIXStyle` — 采用单调混合器的价值分解。
- `MAPPOStyle` — 中心化价值函数；策略针对共享基线进行更新。

所有四种运行相同的回合，并报告平均到达目标步数。CTDE变体相比独立基线收敛到更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约需6步；CTDE变体收敛到约3.5步 (4x4网格的最优步数为3)。尽管是脚本化策略，模式差异依然显现。

## 部署

生产环境中使用MARL的情况很少。当你确实使用时：

- **从MAPPO开始。** 2022年的论文确立了其作为基准的地位；先复现它，可以节省数周追逐更花哨方法的时间。
- **记录每个智能体的观察和动作流。** 没有逐智能体的轨迹，调试MARL毫无希望。
- **分离训练代码与执行代码。** CTDE是一种规范；确保执行路径真的只看到 `o_i`。
- **奖励塑形警告。** 对奖励设计极其敏感。塑形中一个协调性错误，智能体就可能学会利用它。运行对抗性测试。
- **对于LLM智能体**，优先考虑提示级别的策略。仅当交互数据、奖励信号和基础设施都具备时，才投资MARL训练。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------|----------|
| MARL | “多智能体RL” | 多智能体系统的强化学习。 |
| CTDE | “中心化训练，去中心化执行” | 训练时使用全局信息；部署时使用本地策略。 |
| MADDPG | “多智能体DDPG” | 带有能看到所有观察+动作的逐智能体评论家的CTDE。 |
| QMIX | “价值分解” | 逐智能体Q值的单调混合。合作型。 |
| MAPPO | “多智能体PPO” | 采用中心化价值函数的PPO。2026年默认基准。 |
| 价值分解 | “个体Q值之和” | 联合Q表示为逐智能体Q值的单调函数。 |
| 非平稳性 | “移动靶心” | 随着其他智能体学习，每个智能体的环境发生变化。MARL的核心问题。 |
| 在线策略/离策略 | “从当前学习/从重放缓冲区学习” | PPO是在线策略 (MAPPO)；DDPG和Q-learning是离策略。 |
| SMAC | “星际争霸多智能体挑战赛” | 合作型微观管理基准；QMIX的本土战场。 |
