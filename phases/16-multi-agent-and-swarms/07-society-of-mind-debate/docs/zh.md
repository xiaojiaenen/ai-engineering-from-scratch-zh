# 心智社会与多智能体辩论

> 明斯基 1986 年的前提——智能是由专家组成的社会——每隔十年就会被重新发现。2023 年，Du 等人将其转化为一个具体算法：多个大语言模型实例提出答案，相互阅读、批评并更新。经过 N 轮迭代，它们会收敛到一个共识，其在六项推理和事实性任务上的表现超越了零样本思维链和反思。有两个发现很重要：**多个智能体**和**多轮迭代**都独立贡献。社会优于单智能体独白；多轮交流优于单次投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 16 阶段 · 04（原始模型）
**时间：** 约 60 分钟

## 问题

自洽性——对一个模型进行多次采样并取多数答案——是你能附加的最廉价的推理改进方法。它有效，但饱和得很快。你可以将样本量翻倍，却看不到有意义的提升。

辩论打破了饱和。它不是从一个模型进行 N 次独立采样，而是让 N 个智能体相互阅读推理并进行修订。样本之间的相关性降低了（它们不再是独立同分布的），并且在独立同分布投票自信地犯错的地方，收敛点通常是正确的。

## 概念

### Du 等人 2023 算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体中的每一个都对问题产生一个初始答案。
2. 对于轮次 r = 2..R：每个智能体都会看到其他智能体在 r-1 轮的答案，并被要求“考虑到这些，给出你的更新答案。”
3. 在 R 轮之后，对最终答案进行多数投票。

该论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论始终优于思维链和自反思。

### 两个独立的调节旋钮

同一篇论文的消融研究：

- **仅智能体数量**（1 轮，N 个智能体的多数投票）在大多数任务上优于单智能体，但会达到平台期。
- **仅轮次数量**（1 个智能体查看自己之前的推理）几乎没有帮助——这是反思的已知弱点。
- **两者结合**会产生显著的提升。多智能体之间的多轮交流驱动了增益。

### 为什么有效

两种机制：

1. **接触分歧。** 当一个智能体看到另一个具有不同结论的推理链时，它必须要么辩护，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **降低相关误差。** 在自洽性中，所有样本都来自同一个模型，因此误差是相关的——你会平均出一个自信但错误的答案。不同的模型或不同的种子会降低相关性。经过辩论的不同*观点*会进一步降低相关性。

### 异质辩论

A-HMAD 及相关后续研究为不同智能体使用*不同的基础模型*。Llama + Claude + GPT 的辩论减少了单一种群崩溃（第 26 课），因为一个模型家族的相关误差不会被其他家族共享。

缺点：一个较弱的模型参与辩论可能会将共识拉向其错误答案（参见“我们应该进行多智能体辩论吗？”，arXiv:2311.17371）。

### NLSOM — 129 个智能体的扩展

Zhuge 等人（“基于自然语言的心智社会中的头脑风暴”，arXiv:2305.17066）将这一想法扩展到了 129 个成员的社会。结果：随着规模扩大，专业化和自组织出现，并且该系统在视觉问答等任务上优于单智能体。

### 失败模式

- **谄媚级联。** 所有智能体都向听起来最自信的那个智能体让步。辩论退化为最响亮的声音。提示设置对抗性角色（“一个智能体必须提出反对立场”）有助于解决此问题。
- **话题漂移。** 跨多轮的辩论会偏离原始问题。缓解措施：每轮重新注入问题。
- **计算爆炸。** N 个智能体 × R 轮 = N·R 次大语言模型调用，每次调用的上下文都在增长。一个 5 智能体、5 轮的辩论是 25 次调用，上下文在不断增长。每个问题的成本可能超过单次思维链调用的 10 倍。

## 动手构建

`code/main.py` 运行一个 3 智能体 × 3 轮的辩论，针对一个数学问题，每个智能体开始都有一个不同的（可能错误的）答案。智能体是预设的——每个智能体通过根据预设置信度加权平均邻居的答案来进行“更新”。收敛情况可以在逐轮日志中观察到。

该演示展示了两个关键效果：

- 一轮交流使智能体更接近正确答案。
- 超过第 2 轮后的额外轮次表现出递减回报（与 Du 等人观察到的平台期一致）。

运行：

```
python3 code/main.py
```

## 实际使用

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮次数量、异质性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性角色）。它还会在运行前估算 token 成本。

## 产品化部署

如果你要部署辩论系统：

- **轮次上限设为 3。** Du 等人表明 3 轮能捕捉大部分收益。更多轮次是成本，不是质量。
- **智能体上限设为 5。** 超过 5 个，上下文膨胀和成本占主导。
- **默认使用异质性。** 池中至少包含两个不同的基础模型。
- **设置对抗性席位。** 一个智能体被提示无论如何都要提出反对意见。这能打破谄媚行为。
- **记录每一轮。** 隐藏中间轮次的辩论系统无法进行调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮次设置为 5，观察递减回报。从哪一轮开始额外收敛停止？
2. 添加第四个具有对抗性角色的智能体：始终不同意当前多数意见。这会破坏还是改善收敛？
3. 绘图（打印）每轮的共识分数（多数答案上的智能体比例）。它何时达到 1.0？这等同于“正确”吗？
4. 阅读 Du 等人第 4 节的消融研究。使用此代码复制“仅智能体” vs “仅轮次” vs “两者结合”的结果。
5. 阅读“我们应该进行多智能体辩论吗？”（arXiv:2311.17371），列出两种超越循环辩论的辩论变体——例如，法官主导、链式辩论、对抗式。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|------|----------|------------|
| 心智社会 | "明斯基的想法" | 智能作为交互的专家；1986 年的框架，现在通过大语言模型辩论得以操作化。 |
| 多智能体辩论 | "智能体争论" | N 个智能体提出、相互批评、在 R 轮后修订，进行多数投票。 |
| 共识 | "他们同意" | 并非认识论上的真理——只是多数答案上的比例。可能自信地犯错。 |
| 轮次 | "交流步骤" | 一轮 = 每个智能体阅读他人答案并更新一次。 |
| 异质辩论 | "混合模型家族" | 使用不同的基础模型以降低误差相关性。 |
| 谄媚级联 | "所有人都同意声音最大的那个" | 辩论失败模式，智能体不论正确与否都向最自信的智能体让步。 |
| NLSOM | "129 个智能体的社会" | 基于自然语言的心智社会；Zhuge 等人的规模化版本。 |
| 相关误差 | "同模型，同缺陷" | 为什么自洽性会饱和；跨不同观点的辩论可以降低相关性。 |
