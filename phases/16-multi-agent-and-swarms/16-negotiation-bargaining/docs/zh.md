# 谈判与议价

> 代理对资源、价格、任务分配和条款进行谈判。2026年基准测试集很明确：NegotiationArena (arXiv:2402.05863) 表明，通过人格操纵（"desperation"），大语言模型（LLM）可以将收益提高约20%；"Measuring Bargaining Abilities" (arXiv:2402.15813) 表明，买方比卖方更难谈判，且规模无法帮助——其 **OG-Narrator**（确定性报价生成器 + LLM 旁白员）将成交率从26.67%提高到88.88%；大规模自主谈判竞赛 (arXiv:2503.06416) 运行了约18万次谈判，发现通过向对手隐藏推理过程，**思维链隐藏**代理获胜；Bhattacharya等人2025年基于哈佛谈判项目指标的研究将Llama-3评为最有效，Claude-3最具侵略性，GPT-4最公平。本课将实现合同网协议（FIPA的前身，第02课），连接一个类LLM的买方/卖方，运行一种类OG-Narrator的分解方法，并衡量每次结构选择对成交率的影响。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**先修条件：** Phase 16 · 02 (FIPA-ACL 遗产), Phase 16 · 09 (并行群组网络)
**时间：** 约75分钟

## 问题

两个代理需要就价格达成一致。如果仅使用纯粹的语言提示，2024-2026年的LLM在严格参数化的议价中，成交率低得惊人（在arXiv:2402.15813中约为27%）。规模并不能解决这个问题：GPT-4在议价能力上并不比GPT-3.5结构上更优；它只是更擅长议价的*语言表达*。

根本问题在于LLM混淆了两项工作——决定报价和描述报价。OG-Narrator将它们分离：一个确定性报价生成器计算数字变化；LLM只负责描述。成交率跃升至约89%。

这反映了经典的多代理发现：将机制与通信层解耦是成功的。合同网协议（FIPA，1996；Smith，1980）是参考的任务市场机制。将LLM插入描述环节，你就得到了一个现代的、由LLM驱动的任务市场。

## 概念

### 一段话理解合同网协议

Smith 1980年的合同网协议：一个**管理者**广播一个**招标（cfp）**；**投标人**回复包含其报价的**提议**消息；管理者选出一个胜出者，并向胜出者发送**接受-提议**，向失败者发送**拒绝-提议**。胜出者执行工作。可选消息：**拒绝**（投标人拒绝提议）。FIPA将其规范化为 `fipa-contract-net` 交互协议。

### 为什么OG-Narrator获胜

"测量语言模型的议价能力" (arXiv:2402.15813) 观察到：

- LLM经常违反议价规则（以荒谬的价格报价，忽略对方的可能协议区间ZOPA）。
- 它们锚定能力差（接受糟糕的首次报价；以象征性而非策略性的金额进行还价）。
- 仅靠规模无法解决这些问题。更大的模型只是生成更可信的语言，但战略错误类似。

OG-Narrator分解如下：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ offer generator  │ price → │  LLM narrator    │ → message
           │  (deterministic) │        │  (writes the     │
           │                  │        │   human-style    │
           └──────────────────┘        │   accompaniment) │
                                       └──────────────────┘
```

报价生成器是一种经典的谈判策略：鲁宾斯坦议价模型、泽森策略，或简单的基于价格的针锋相对策略。LLM负责描述。消息包含确定性的价格和自然语言的框架。

成交率跃升的原因：
- 价格保持在议价区间内。
- 锚点具有战略性，而非情绪化。
- LLM做它擅长的事：写作。

### NegotiationArena 的发现

arXiv:2402.05863 提供了基准测试。主要发现：

- LLM可以通过采用人格（"我必须在周五前卖掉这个"）将收益提高约20%——人格操纵是一种真实的策略。
- 公平/合作的代理会被对抗性代理利用；防御需要明确的反制姿态。
- 在约40%的基准测试场景中，对称配对会导致不公平的结果。

这并不是"LLM是糟糕的谈判者"。而是"LLM谈判太像人类，包括其中可被利用的部分"。

### 思维链隐藏

大规模自主谈判竞赛 (arXiv:2503.06416) 在众多LLM策略上运行了约18万次谈判。获胜者向对手隐藏了他们的推理过程：

- 如果一个代理打印出"我最多只能出75美元；我的保留价格是70美元"到一个公开可见的草稿区，对手就会读到它。
- 获胜者私下计算策略；输出通道只包含报价和最低限度的必要描述。

这是2026年对经典博弈论（Aumann 1976 关于理性与信息）的呼应：暴露你的私人估值会付出收益代价。LLM不会直觉理解这一点，并会欣然地将它们的保留价格写入推理追踪中，而这些追踪可能对对手可见。

工程要点：将私人草稿上下文与公共消息上下文分离。这是必须的。

### Bhattacharya等人2025年——模型排名

基于哈佛谈判项目指标（原则性谈判、尊重BATNA、利益互惠）：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最具侵略性的谈判者（高锚点、后期让步）。
- **GPT-4** 最公平（不同配对的收益方差最小）。

这是2025年的快照。重点不在于2026年4月哪个模型获胜——而在于不同的基础模型具有持久的谈判风格。异构集成（第15课）将其作为一种多样性来源。

### 通过合同网协议 + LLM 进行任务分配

将合同网协议重新用于LLM多代理的现代方法：

1. 管理者代理将任务分解为单元。
2. 向工作代理广播包含任务描述的 `cfp`。
3. 每个工作代理返回一个报价：`(price, eta, confidence)`，其中价格可以是token、计算单元或美元。
4. 管理者选择胜出者（单个或多个，取决于任务）并授标。
5. 被拒绝的工人可以自由竞标其他任务。

这可以良好地扩展到超过100个工人，因为协调是广播-响应模式，而非同步聊天。已在生产中使用：微软代理框架的编排模式，一些LangGraph实现。

### LLM-利益相关者交互式谈判

NeurIPS 2024 (https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf) 引入了具有**秘密评分**和**最低接受阈值**的多方可评分博弈。每个利益相关者都有私有效用；LLM必须从消息中推断出来。这是双方议价到N方联盟形成的推广。适用于具有异构工人能力的生产任务市场。

### 描述 vs 机制 规则

纵观所有2024-2026年的谈判基准，一致的工程规则是：

> 让LLM描述。不要让LLM计算报价。

如果报价需要是一个数字（价格、预计到达时间、数量），请根据谈判状态确定性地生成它，并让LLM生成框架描述。如果报价需要是一个提案结构（任务分解、角色分配），让LLM起草，但在发送前根据模式和约束进行验证检查。

## 构建它

`code/main.py` 实现了：

- `ContractNetManager`, `ContractNetTask`, `Bid` — 管理者 + 投标者，广播cfp，收集提议，授标。
- `og_narrator_bargain(state, rng)` — OG-Narrator 买方：确定性的泽森式向中间点让步。
- `seller_response(state, rng)` — 确定性的卖方还价策略（两种风格的结构化真实情况）。
- `naive_llm_bargain(state, rng)` — 模拟一个完全LLM的议价者：选择高方差的价格，经常超出ZOPA。
- 测量：在1000次试验中，每次试验使用新的保留价格采样，计算成交率。

运行：

```
python3 code/main.py
```

预期输出：朴素LLM的成交率约65-75%；OG-Narrator的成交率约85-95%；15-25个百分点的差距就是将报价生成与描述分解开的结构优势。另外还有一个包含三个投标人和一个任务的合同网任务市场分配示例。

## 使用它

`outputs/skill-bargainer-designer.md` 设计了一个议价协议：谁生成报价（确定性或LLM），谁负责描述，私人草稿如何与公共消息分离，以及如何监控成交率。

## 交付它

生产环境议价检查清单：

- **分离草稿。** 私人状态绝不能到达对手的上下文中。这一点不可妥协。
- **确定性报价生成。** 价格、数量、预计到达时间：计算，不要提示。
- **根据模式验证所有传入的报价。** 在协议边界拒绝超出ZOPA的报价。
- **限定轮次。** 最多3-5轮；僵局时升级到调解人。
- **持续衡量成交率和收益方差。** 成交率下降是一个症状——通常是提示漂移或对手侧攻击。
- **记录所有被拒绝的提议及其确定性理由。** 对于合同网管理者，失败的投标人需要理解原因。

## 练习

1. 运行 `code/main.py`。确认OG-Narrator在成交率上优于朴素LLM。高多少？
2. 实现**基于人格的收益提升** (arXiv:2402.05863) —— 买方仅在描述中采用“急需本周购买”的人格，报价生成器不变。成交率或收益会改变吗？
3. 实现思维链**隐藏**：维护一个不会传递给对手的私人草稿字符串。如果你不小心泄露了它会发生什么（通过交换通道来模拟）？
4. 将合同网协议扩展到具有保留价格的N方拍卖。当所有出价都超过保留价时，管理者如何在最低价格和最高质量之间做出决定？你选择哪种授标规则，为什么？
5. 阅读Bhattacharya等人2025年关于哈佛谈判项目指标的研究。实现两个具有不同风格的议价者（侵略型 vs 公平型）。衡量对称和非对称配对下的收益方差。

## 关键术语

| 术语 | 人们常说 | 它的实际含义 |
|------|----------|--------------|
| Contract Net (合同网协议) | "任务市场" | Smith 1980, FIPA 1996。cfp + 提议 + 接受/拒绝。经典的任务市场。 |
| ZOPA (可能协议区间) | "可能的协议区间" | 买方最高价与卖方最低价之间的重叠。超出此区间的报价无法成交。 |
| BATNA (谈判协议最佳替代方案) | "谈判协议的最佳替代方案" | 如果本次交易失败，你的后备选项。设定了你的保留价格。 |
| OG-Narrator | "报价生成器 + 描述员" | 分解：确定性报价，LLM描述。 |
| Zeuthen strategy (泽森策略) | "风险最小化让步" | 经典的报价生成器，基于风险限制进行让步。 |
| Rubinstein bargaining (鲁宾斯坦议价) | "交替报价均衡" | 带折扣的无限期议价的博弈论模型。 |
| CoT concealment (思维链隐藏) | "隐藏你的推理" | arXiv:2503.06416中的获胜者保持私人草稿；公共通道只显示报价。 |
| Persona manipulation (人格操纵) | "情绪姿态" | arXiv:2402.05863：通过绝望/紧迫人格可获得约20%的收益提升。 |
