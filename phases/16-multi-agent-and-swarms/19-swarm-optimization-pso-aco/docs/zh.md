# LLM的群体优化 (PSO, ACO)

> 受生物启发的优化正在LLM领域卷土重来。**LMPSO** (arXiv:2504.09247) 使用PSO，其中每个粒子的速度是一个提示词，LLM生成下一个候选解；在结构化序列输出（数学表达式、程序）上效果良好。**Model Swarms** (arXiv:2410.11163) 将每个LLM专家视为位于模型权重流形上的一个PSO粒子，并在仅使用200个实例的情况下，在9个数据集上报告了相比12个基线**平均提升13.3%**。**SwarmPrompt** (ICAART 2025) 将PSO与灰狼算法混合用于提示词优化。**AMRO-S** (arXiv:2603.12933) 是受ACO启发的多智能体LLM路由信息素专家——**实现4.7倍加速**、可解释的路由证据、质量门控的异步更新，将推理与学习解耦。本课程将在提示词参数空间上实现PSO，在智能体路由上实现ACO，并衡量这些经典算法为何适用于LLM时代，以及何时不适用。

**类型:** 学习 + 构建
**语言:** Python (标准库)
**前置知识:** Phase 16 · 09 (并行群体网络)，Phase 16 · 14 (共识与拜占庭容错)
**时间:** ~75分钟

## 问题

你有一个在任务评估中得分62%的提示词。你想改进它。最朴素的方法是无梯度的手动调整，但这种方法扩展性差。强化学习需要奖励信号和足够的推演来训练。通过提示词进行反向传播几乎不可能——提示词是离散字符串，而非可微分的参数。

经典的受生物启发优化——用于连续搜索空间的PSO和用于路径选择的ACO——正是为此场景设计的：无梯度、基于群体、每次评估成本低。将它们与LLM结合用于无梯度搜索步骤，你就能得到一个出奇实用的优化器。

同样的模式也适用于多智能体系统中的智能体*路由*。ACO式的信息素轨迹记录了哪个智能体在哪个任务类型上表现最好，让路由器利用该轨迹，并通过信息素衰减，使路由能够被重新发现。

## 概念

### PSO复习 (Kennedy & Eberhart 1995)

粒子群优化：在连续搜索空间中的粒子群体。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子自身的历史最佳，`g_best` 是群体的历史最佳，`w, c1, c2` 是惯性、认知和社会权重，`r1, r2` 是随机因子。

### 应用于LLM输出的PSO — LMPSO

arXiv:2504.09247 将PSO适配于LLM生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示词*，描述如何将当前输出修改为向个体/全局最佳靠拢。LLM根据速度提示词生成新的输出。速度的“惯性”是一个类似于“进行小的增量改变”的提示词。

这在以下情况下效果良好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 群体规模较小（~10-30个粒子），因此总的LLM调用次数可控。

当适应度需要人工审查时，此方法效果不佳——每次迭代的成本变得过高。

### Model Swarms

arXiv:2410.11163 将PSO从输出层提升到*模型*层。每个“粒子”是一个专家LLM（参数）。群体通过无梯度更新将参数移向集体最佳。报告称：在仅使用每个迭代200个实例的情况下，在9个数据集上相比12个基线平均提升13.3%。

关键洞察在于，LLM专家模型在共享的参数流形（适配器权重、LoRA差值）中本就相距不远。在此低维子空间上进行PSO既廉价又有效。

### ACO复习 (Dorigo 1992)

蚁群优化：蚂蚁遍历图；每条路径有一个信息素轨迹。蚂蚁的移动概率根据信息素强度加权。完成任务的蚂蚁会根据解的质量沉积信息素。信息素随时间衰减。

### AMRO-S — 用于智能体路由的ACO

arXiv:2603.12933 使用ACO进行多智能体路由。每个任务类型是一个“目的地”；每个智能体是一条可能的路径。信息素强化了产生良好输出的路径。主要贡献包括：

- **可解释的路由证据。** 信息素强度是一个人类可读的信号。
- **质量门控的异步更新。** 仅在质量检查通过后更新信息素，将推理与学习解耦。
- 在多智能体路由基准上实现**4.7倍加速**。

质量门控至关重要：没有它，快速但错误的智能体会积累信息素，系统会锁定在错误的路由上。

### 何时对LLM使用PSO/ACO

**在以下情况使用PSO：**
- 搜索空间是连续的，或可映射到连续参数（提示词嵌入、LoRA权重、数值生成参数）。
- 适应度评估成本低且自动化。
- 群体规模可以较小（10-30）。

**在以下情况使用ACO：**
- 你有路由或路径选择问题。
- 决策会随时间强化（相同的任务类型会重复出现）。
- 你需要路由决策的可解释证据。

**在以下情况不使用这两种方法：**
- 适应度需要人工审查（每次迭代成本过高）。
- 搜索空间是离散且组合的，PSO无法覆盖（应改用遗传算法）。
- 实时决策需要严格的延迟（PSO/ACO相对于单遍启发式收敛较慢）。

### 为何生物启发方法仍然有效

基于梯度的方法需要可微分的信号。LLM的输出和路由决策并非 trivially 可微分。伪梯度方法（强化学习路由器、DPO式提示词调优器）可行，但需要昂贵的训练。

PSO和ACO仅需要一个*评估*函数。如果你能对一个候选输出或一个路由决策打分，你就能在空间上进行优化。这使得适用性的门槛大大降低。

### 实际限制

- **群体预算。** N个粒子 × T次迭代 × 每次评估成本。对于每次调用约0.02美元的LLM评估，一个20个粒子的PSO运行50次迭代成本约20美元。请做好计划。
- **探索与利用。** 信息素衰减率和PSO惯性需要权衡；衰减过快会遗忘解；衰减过慢会陷入早期局部最优。
- **灾难性漂移。** 两种算法都可能收敛，但如果适应度地形发生变化（新数据分布），则会发散。需要监控最佳适应度的稳定性。

## 构建它

`code/main.py` 实现了：

- `LMPSO` — 对数值提示词参数（temperature, top_k权重）进行PSO优化。每个粒子的“LLM生成”通过脚本化的适应度函数模拟。运行算法30次迭代并展示g_best收敛过程。


预期输出：

`outputs/skill-swarm-optimizer.md` 帮助在PSO、ACO、遗传算法和基于梯度的优化器之间进行选择，用于LLM / 智能体优化问题。

## 部署它

- **从小规模开始。** 10-20个粒子，20-50次迭代。仅当收敛曲线显示明确收益时才扩大规模。
- **记录每迭代的信息素或g_best。** 没有轨迹调试群体优化器会很痛苦。
- **质量门控更新。** 特别是对于ACO路由：快速且错误的智能体不应积累信息素。
- **在分布漂移时重置衰减。** 当你的评估分布改变时，旧的信息素已过时；重置或临时加倍衰减率。
- **限制每次迭代的成本。** 输出每次迭代的成本指标。每次迭代花费500美元且仅提升0.5%的PSO是不可部署的。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| PSO | "粒子群优化" | Kennedy-Eberhart 1995。基于群体的无梯度优化器。 |
| ACO | "蚁群优化" | Dorigo 1992。通过信息素轨迹进行路径/路由优化。 |
| LMPSO | "结合LLM生成的PSO" | arXiv:2504.09247。速度是提示词；LLM产生候选解。 |
| Model Swarms | "应用于专家权重的PSO" | arXiv:2410.11163。在模型参数子空间上进行无梯度更新。 |
| AMRO-S | "用于智能体路由的ACO" | arXiv:2603.12933。任务类型 × 智能体的信息素矩阵。 |
| p_best / g_best | "个体最佳 / 全局最佳" | 每个粒子和群体迄今为止找到的最佳解。 |
| Pheromone | "路由记忆" | 边上的强度；随时间衰减；在质量良好时沉积。 |
| 质量门控更新 | "仅从好的运行中学习" | 信息素沉积取决于质量检查。 |
| 灾难性漂移 | "分布漂移" | 适应度地形改变；旧的p_best和信息素变得过时。 |
