# 生成式智能体与涌现模拟

> Park 等人 2023 (UIST '23, arXiv:2304.03442) 用一个三部分架构填充了 **Smallville**——一个包含25个智能体的沙盒：**内存流**（自然语言日志）、**反思**（智能体对其自身流生成的更高层次综合）和**计划**（日级行为，然后是子计划）。里程碑式的成果是情人节派对的涌现：一个被设定“想在2月14日举办情人节派对”的智能体，在无需进一步脚本的情况下，产生了邀请函并在人群中传播，协调了日期，派对最终举行——参与者来自最初对此毫不知情的24个智能体。消融实验表明，所有三个组成部分对于可信度都是必需的。记录的失败包括空间规范错误（进入已关闭的商店、共享单人洗手间）。这是2026年智能体模拟和多智能体社会评估的参考架构。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**前置知识：** 第16阶段 · 04（原始模型），第16阶段 · 13（共享内存）
**时间：** ~75 分钟

## 问题

大多数多智能体系统都是严格按脚本编写的团队：规划者规划，编码者编码，审阅者审阅。这对定义明确的任务有效。它无法捕捉当智能体拥有记忆、优先级和一个开放世界时所产生的涌现的、非脚本化的行为。研究、社会模拟以及越来越多的游戏AI需要这第二种类型。

Smallville 架构是其基准。在 Park 2023 之前，最好的智能体模拟是浅显的脚本遵循者；之后，该模式成为开放世界中生成式智能体的默认标准。如果你在2026年构建智能体模拟，你要么使用 Smallville 的三组件，要么明确解释为何不用。

## 概念

### 三个组成部分

**内存流。** 一个只追加的日志，记录观察、行动、反思和计划。每条记录都有一个时间戳、类型、描述（自然语言）以及衍生元数据：**新近性**、**重要性**（智能体自评1-10分）和**相关性**（与当前查询的余弦相似度）。

```
[2026-02-14 09:12:03] observation: Isabella Rodriguez asked me if I like jazz
[2026-02-14 09:14:22] reflection:   I enjoy long conversations about music
[2026-02-14 10:05:00] plan:         Attend Isabella's Valentine's Day party tonight
```

内存检索综合这三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。排名前k的条目进入当前提示。

**反思。** 定期地（每N条记忆或在重要事件发生时），智能体根据近期记忆生成更高层次的综合。反思条目会放回内存流，并可以像任何其他记忆一样被检索。这是智能体构建“理解”的方式——该架构中长期信念的等价物。

**计划。** 自上而下的分解。首先，制定一个日级计划的大致轮廓（“去上班，和克劳斯共进晚餐”）。然后是小时级计划。接着是行动级计划。计划是可修订的：当观察结果与计划相矛盾时，智能体只重新规划受影响的部分。

### 为何三者都至关重要（消融实验）

Park 等人进行了消融实验，分别移除了观察、反思和计划。每一次消融都会损害可信度：

- 没有 **观察**，智能体会错过上下文，并基于过时的信念行动。
- 没有 **反思**，智能体无法形成更高层次的信念；互动停留在浅层。
- 没有 **计划**，行为会变成被动的噪声；目标消散。

来自人类评分者的可信度得分在三者齐全时最高；移除任何一个都会产生可衡量的下降。

### 情人节涌现

一个智能体，伊莎贝拉·罗德里格斯，被设定了目标“想在2月14日下午5点于霍布斯咖啡馆举办情人节派对”。其他24个智能体没有收到这样的设定。在模拟的几天内：

1. 伊莎贝拉的计划包括邀请别人。
2. 每份邀请都成为邻居内存流中的一条观察记录。
3. 该邻居的反思产生了信念：“伊莎贝拉要举办派对。”
4. 该邻居的计划包含了“在2月14日参加派对”。
5. 邻居们告诉其他邻居。邀请在没有中央协调的情况下传播。
6. 2月14日下午5点，几个智能体汇集到霍布斯咖啡馆。

这在技术意义上是一种涌现：系统级行为（派对）源于局部互动（双边邀请 + 个体规划），而没有中央协调者。

### 记录的失败模式

Park 等人明确记录了：

- **空间规范错误。** 智能体走进已关闭的商店。智能体试图使用同一个单人洗手间。智能体在非用餐房间进食。该模型无法仅从环境推断社会物理规范。
- **内存溢出。** 深度模拟运行导致内存检索成本增长。实际补救措施：定期进行内存压缩（总结和剪枝）以及对低重要性条目进行衰减。
- **反思幻觉。** 反思可能会虚构内存流中不存在的关系。缓解措施：在反思提示中包含源内存ID，并在检索时进行验证。

这些是与生产相关的失败模式：任何2026年的智能体模拟都会继承它们。

### 三组件实现规则

1. **内存是只追加的。** 切勿修改内存条目。纠正是通过新条目实现。
2. **重要性评分是廉价的。** 在写入时调用LLM对重要性进行1-10分的评分。缓存该分数。
3. **检索是基于排序的，而非过滤的。** 根据综合分数取前k个；不要使用硬过滤器（这会丢失上下文）。
4. **反思定期运行。** 当未处理的记忆的重要性总和超过阈值（例如150）时触发。
5. **计划是可修订的。** 当新的观察结果与计划相矛盾时，仅重新生成受影响的部分，而不是整个计划。

### 超越 Smallville 的生成式智能体

2024-2026年的后续文献扩展了该架构：

- **用于政策/市场研究的多智能体社会模拟。** 类似 Smallville 的种群模拟用户行为对功能的响应。比A/B测试更快；准确性存在争议。
- **用于游戏的NPC AI。** 拥有 Smallville 智能体的RPG产生涌现的故事情节，而非脚本化的任务。
- **生成式智能体评估基准。** 指标不再是任务准确率，而是长期运行中的行为可信度和连贯性。

该架构是参考标准。扩展组件替换了组件（向量存储用于内存、检索增强反思、神经符号计划），但保留了三部分结构。

### 为何这对多智能体工程很重要

Smallville 是一个概念证明，表明当组件正确时，多智能体涌现是廉价的。该架构现在已在开源模型上得到复制（较小的LLM会平滑地失去可信度，而非急剧下降）。任何需要**涌现社会行为**的生产系统都使用这种形式。任何需要**严格任务执行**的系统则使用本阶段早期介绍的监督者/角色/原始模式。

## 构建它

`code/main.py` 使用标准库Python实现了三个组件，并采用脚本化的智能体策略（无真实LLM）。该演示以微缩形式复现了情人节派对的涌现：

- `MemoryStream` — 带有新近性/重要性/相关性检索的只追加日志。
- `reflect(stream)` — 对近期高重要性记忆的脚本化反思。
- `plan(agent_state)` — 基于当前信念的日级和小时级计划。
- 场景：5个智能体。智能体1初始设定为“在下午5点举办派对”。在模拟的节拍中，邀请传播开来，智能体们汇集。

运行：

```
python3 code/main.py
```

预期输出：逐节拍的追踪记录。到最后一个节拍时，5个智能体中至少有3个在其计划中显示派对，并且他们汇集在派对地点。单一种子在没有任何协调者的情况下产生了协调的到达。

## 使用它

`outputs/skill-simulation-designer.md` 设计一个生成式智能体模拟：智能体数量、内存模式、反思节奏、计划视野和评估指标。

## 部署它

生产环境模拟的规则：

- **内存就是数据库。** 在规模上选择一个真正的存储（向量数据库，Postgres）。内存标准库用于原型。
- **记录检索轨迹。** 对于每个行动，记录驱动它的前k条记忆。这是你的调试能力。
- **规划每个智能体的token预算。** 每个智能体每节拍的检索+反思+计划是O(k)次LLM调用。N个智能体 × T个节拍 × 每节拍调用次数可能会大大超出你的预算。
- **定期压缩内存。** 总结并剪枝低重要性条目。保留策略是一个设计决策，而非细节。
- **显式检测空间/社会规范违规。** 该架构并不学习它们。

## 练习

1. 运行 `code/main.py`。确认3个以上智能体在派对处汇集。将智能体增加到10个——涌现仍然会发生吗？
2. 移除反思步骤。行为会是什么样子？将其映射到 Park 2023 中的消融发现。
3. 引入一个竞争性的种子目标（“克劳斯想在下午5点做一个研究报告”）。智能体会分裂，还是一个目标会占主导？是什么决定了这一点？
4. 增加空间限制：霍布斯咖啡馆最多容纳4个智能体。模拟是否能优雅地处理溢出，还是会遇到“单人洗手间”失败模式？
5. 阅读 Park 等人 (arXiv:2304.03442) 第6节（涌现行为实验）。找出一个在你的微缩版中无法复现的行为。你需要增强架构的哪个组件？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 内存流 | “智能体的日记” | 观察、行动、反思、计划的只追加日志。 |
| 新近性 | “记忆有多新” | 基于年龄的指数衰减分数。 |
| 重要性 | “智能体有多在意” | 写入时自评1-10分。已缓存。 |
| 相关性 | “与当前查询有多相关” | 余弦相似度（基于嵌入）。 |
| 反思 | “更高层次的信念” | 从近期记忆中生成的综合，作为新记忆重新注入。 |
| 计划 | “日/时/行动分解” | 自上而下的计划树。当观察结果矛盾时可修订。 |
| Smallville | “Park 2023的沙盒” | 产生了情人节涌现的25智能体模拟。 |
| 可信度 | “质量指标” | 人类评分者对行为是否像一个合理智能体的评分。 |
