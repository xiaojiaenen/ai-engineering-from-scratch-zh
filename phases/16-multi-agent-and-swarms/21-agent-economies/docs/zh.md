# 代理经济、Token激励与声誉

> 长期自主代理（METR 的 1 小时至 8 小时工作曲线）需要经济自主权。新兴的**五层栈**是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（代理型 DAO）。生产级的代理激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai / ASI 联盟**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 Transformer 的工作量证明，将计算重新分配到生产性 AI 任务）。学术研究：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用分配** 公平奖励贡献代理；谷歌研究《大语言模型的机制设计》提出了在单调聚合下采用第二价格支付的 **Token 拍卖**。本课构建一个最小代理市场，将 Shapley 值信用分配应用于多代理管道，并运行一个第二价格 Token 拍卖，使博弈论机制具体落地。

## 问题

当代理共同产生价值但需要单独奖励时，多代理系统会变得复杂。经典机制——平均分配、最后贡献者独占——不公平或易被操纵。通过 Shapley 值进行基于联盟的奖励在构造上是公平的，但计算成本高。2025-2026 年的文献推动了实用的近似方法：Shapley 采样、单调聚合拍卖以及从已确认贡献中积累的链上声誉。

除了信用分配，该领域已转向实际的经济代理：Bittensor TAO 奖励挖矿计算以微调子网特定模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 的使用，Gonka 将 Transformer 工作量证明重新分配给生产性 AI 任务。能自主交易的代理今天已经存在；问题是如何调整激励。

本课将代理经济视为一个特定的问题族——信用分配、机制设计与声誉——并用最小化的数学来构建每个部分，使概念得以牢固掌握。

## 概念

### 五层代理经济栈

1. **DePIN（物理计算）。** 去中心化基础设施，用于出租 GPU、存储、带宽。Bittensor 子网、Render Network、Akash。并非代理专用；代理使用它。
2. **身份。** W3C 去中心化标识符 (DID) 为每个代理提供独立于任何平台的持久 ID。声誉累积于 DID。代理网络协议 (ANP) 将 DID 用作发现层。
3. **认知。** 代理的推理循环：LLM + RAG + MCP。这是其他阶段所构建的内容。
4. **结算。** 账户抽象 (ERC-4337) 允许代理使用自己的余额支付 Gas，而无需持有 ETH。代理可以为服务、彼此或计算付费。
5. **治理。** 代理型 DAO：人类和代理对协议变更进行投票的治理结构，投票权与声誉挂钩。

并非所有生产系统都使用全部五层。Bittensor 使用 1、2，部分使用 3、4，不使用 5。OpenAI 代理仅使用 3。该栈是参考地图，而非要求。

### Bittensor、Fetch.ai、Gonka——运行中的实例

**Bittensor (TAO)。** 子网是专门的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其进行排名；基于质押权重的评分分配 TAO 奖励。每个子网有自己的评估。经济学教训：为特定任务的输出质量付费，而不是使用的计算量。

**Fetch.ai / ASI 联盟。** ASI-1 Mini LLM 在 Fetch.ai 网络上运行；用户支付 FET 代币进行推理。这里的“代理即对等体”叙事更强：Fetch 上的代理可以调用另一个代理执行任务并用 FET 支付。

**Gonka。** Transformer 工作量证明：“工作”是 Transformer 的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚钱。资源生产性的工作量证明，而非基于哈希的工作量证明。

截至 2026 年 4 月，三者均达到生产级。收益分配不同。Bittensor 奖励相对于子网验证者的质量；Fetch 奖励由付费用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 值信用分配

三个代理合作完成一项任务。输出得分为 0.8。谁贡献了什么？

Shapley 值：满足四个公理（效率、对称性、线性、无效性）的唯一信用分配。对于代理 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中位于 `i` 之前的代理集。实际操作：枚举所有排列，记录每个排列中每个代理的边际贡献，然后取平均值。

对于 N=3 个代理，有 6 种排列。对于 N=10，有 360 万种——因此实际中采用采样排序而非枚举。

### 用于聚合的第二价格拍卖

谷歌研究（《大语言模型的机制设计》）提出了用于聚合 LLM 输出的第二价格 Token 拍卖。设置：N 个代理各提出一个补全；每个代理对被选中都有一个私有估值。拍卖师选择估值最高的提案，并支付*第二高的*估值。在单调聚合下（价值取决于选择了哪个提案，而不是出价了多少），这是真实的——代理会如实出价。

这对 LLM 系统的意义：你可以将补全任务外包给具有不同定价的多个代理；拍卖选出最佳方案并公平支付，且代理没有动机虚报。

### 声誉资本

一个与 DID 绑定的声誉分数从已确认的贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 在路由决策中读取成本低（“将困难任务发送给高声誉代理”）。
- 伪造成本高（随时间累积，绑定到 DID）。
- 可被削减：未通过验证的贡献会减分。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用分配和简单的拍卖机制。关键主张：去中心化信用分配步骤使系统可审计且免受单点操纵。

### 经济学失效之处

- **价格预言机操纵。** 如果信用函数可被操纵，代理就会操纵它。每个机制都需要对抗性测试。
- **女巫攻击。** 一个操作者启动 N 个虚假代理来夸大自己的贡献。DID 能减缓但不能阻止；声誉的伪造成本是缓解措施。
- **验证成本。** 信用分配的公平性取决于验证者。如果验证成本低（小型 LLM），它可能被操纵；如果成本高（人工小组），系统无法扩展。
- **监管隐患。** 代理经济与金融监管有交集。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都处于法律灰色地带。

### 何时代理经济有意义

- **具有异构操作者的开放网络。** 没有单一团队控制所有代理。
- **可验证的输出。** 没有验证，信用分配只是猜测。
- **长期工作流。** 一次性任务不会从声誉累积中受益。
- **代币化支付在你的司法管辖区合法可行。**

在封闭的企业系统中，经济学让位于更简单的分配（经理分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 部署它

在 2026 年运行一个代理经济：

- **从声誉开始，而非代币。** 声誉实现成本低且本身就有价值；代币会增加法律和经济复杂性。
- **先验证，再奖励。** 没有独立验证步骤，绝不分配信用。自我报告的质量会引发女巫游戏。
- **Shapley 采样，而非精确 Shapley。** 采样 100-1000 种排序；精确枚举无法扩展。
- **设定衰减因子上限和声誉下限。** 无上限的衰减会抹去合法贡献者；衰减过慢会奖励过时的高声誉代理。
- **对抗性审计机制。** 在开放网络前运行红队演练。每个机制都有博弈论；你想找出漏洞，而不是攻击者。

## 关键术语

| 术语 | 人们说什么 | 实际含义 |
|------|------------|----------|
| DePIN | “去中心化物理基础设施” | 代币激励的计算/存储/带宽。Bittensor, Akash, Render。 |
| DID | “去中心化标识符” | 用于可移植 ID 的 W3C 规范。代理声誉绑定到 DID，而非平台。 |
| ERC-4337 | “账户抽象” | 可赞助 Gas 的合约账户，实现代理支付。 |
| Shapley 值 | “公平信用分配” | 满足效率、对称性、线性、无效性的唯一分配。 |
| 第二价格拍卖 | “维克瑞拍卖” | 真实机制：获胜者支付第二高出价。与单调聚合兼容。 |
| 声誉资本 | “累积质量分数” | 来自已确认贡献的、绑定 DID 的分数；随时间衰减。 |
| 代理型 DAO | “代理 + 人类治理” | 将代理选民作为一等公民的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 积分 | “代币面额” | Bittensor TAO, Fetch.ai FET, 各种 DePIN 代币。 |
