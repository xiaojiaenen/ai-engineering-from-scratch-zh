# OpenAI 准备就绪框架与 DeepMind 前沿安全框架

> OpenAI 准备就绪框架 v2（2025年4月）引入了研究类别 — 长期自主性、策略性表现不佳、自主复制与适应、破坏安全措施 — 有别于跟踪类别。跟踪类别会触发能力报告以及安全报告，由安全顾问小组审阅。DeepMind 的前沿安全框架 v3（2025年9月发布，2026年4月17日增加跟踪能力等级）将自主性融入机器学习研发和网络领域（机器学习研发自主性等级 1 = 以相对于人类+AI工具具有竞争力的成本，完全自动化AI研发流程）。FSF v3 明确通过自动监控工具性推理滥用来应对欺骗性对齐。一个诚实的注释：准备就绪框架 v2 中的研究类别（包括长期自主性）不会自动触发缓解措施；政策用语是“潜在的”。DeepMind 自身表示，如果工具性推理能力增强，自动监控“长期来看将不再足够”。

**类型：** 学习
**语言：** Python（标准库，三框架决策表差异工具）
**先决条件：** 第15 · 19阶段（Anthropic RSP）
**时间：** ~45分钟

## 问题

第19课仔细阅读了 Anthropic 的扩展政策。本课通过阅读 OpenAI 和 DeepMind 的政策来完善全貌。这三份文档是表亲性的产物，处理同一个问题——一个前沿实验室何时应暂停或限制一个模型——它们在少数类别上达成共识，并在一些重要的具体地方存在分歧。

共识：三者都将长期自主性标记为值得跟踪的能力类别。三者都承认欺骗性行为（对齐伪装、策略性表现不佳）是一类特定的风险。三者都有一个内部审查机构。分歧：OpenAI 将类别分为“跟踪的”（强制缓解）和“研究的”（无自动触发）。DeepMind 将自主性融入两个领域，而不是单独命名。实验室将能力称为“跟踪的 vs 研究的”，或“关键的 vs 中等的”，或“第一梯队 vs 第二梯队”；一个能力属于哪个桶，在不同实验室的操作后果是不同的。

将它们放在一起阅读是一个有益的练习。同一个能力在 Anthropic 可能是“强制缓解”，在 OpenAI 是“被监控但不触发”，而在 DeepMind 是“在特定领域内跟踪”。政策姿态很重要。

## 概念

### OpenAI 准备就绪框架 v2（2025年4月）

结构：

- **跟踪类别**：触发能力报告（模型能做什么）加上安全报告（已采取哪些缓解措施）。部署前由安全顾问小组审阅。
- **研究类别**：实验室正在跟踪但尚未承诺采取特定缓解措施的模型能力。包括长期自主性、策略性表现不佳、自主复制与适应、破坏安全措施。

v2 中的关键框架设定：研究类别不会自动触发缓解措施。政策语言是“潜在的”缓解措施。这是需要关注的具体差异点：一个类别位于“跟踪的”还是“研究的”，是“受限”和“仅观察”之间的枢纽。

### DeepMind 前沿安全框架 v3（2025年9月；跟踪能力等级于2026年4月17日增加）

结构：

- **关键能力等级（CCLs）**：五个领域的能力阈值：网络、生物、机器学习研发、CBRN、自主性（融入机器学习研发和网络领域）。
- **跟踪能力等级**：2026年4月增加的更细粒度划分。具体示例：机器学习研发自主性等级 1 = 以相对于人类+AI工具具有竞争力的成本，完全自动化AI研发流程。
- **欺骗性对齐监控**：明确承诺对工具性推理滥用进行自动监控。

其对自主性的框架设定与 OpenAI 不同。DeepMind 不将“自主性”保留为顶级领域；它被融入自主性可能造成危害的领域（机器学习研发和网络）。其论点是，脱离领域的自主性是没有风险的能力；反论点是，跨领域的自主性是一种框架应明确命名的元风险。

### 三者的共同点

- 内部安全顾问小组（Anthropic 称 SAG，OpenAI 称 SAG，DeepMind 称内部委员会）。对高能力模型在部署前进行审查。
- 明确提及欺骗性对齐/对齐伪装作为一类风险。
- 按声明的节奏更新的常设文件（Anthropic：前沿安全路线图、风险报告；OpenAI：能力和安全报告；DeepMind：FSF 更新周期）。
- 承认仅监控的防御措施存在天花板。DeepMind 明确表示：“自动监控长期来看将不再足够。”

### 不同点

- **Anthropic**：在 v3.0 中移除了暂停承诺；AI研发-4 阈值是下一个命名的关键节点。
- **OpenAI**：跟踪 vs 研究的划分；研究类别（包括长期自主性）不自动触发限制。
- **DeepMind**：自主性融入其他领域；跟踪能力等级在2026年4月增加了粒度。

### 策略性表现不佳：一个使三者都复杂化的特定能力

策略性表现不佳（模型在评估中战略性表现不佳）在 OpenAI 的研究类别中。Anthropic 的 RSP v3.0 通过评估-上下文差距（第1课）来处理它。DeepMind 通过 FSF v3 中的欺骗性对齐监控来处理它。

如果一个模型在评估中策略性表现不佳，每个框架的能力阈值都会被低估。框架只有在测量有效时才能起作用。这就是为什么除了实验室自我评估外，外部测量（第21课，METR）和对抗性评估是必要的。

### 政策阅读技巧

- **定位**：你关心的每一个能力都应能在政策中找到。如果找不到，则政策未涵盖它。
- **分类**：它是跟踪的（触发缓解）还是研究的（被跟踪但不触发）？OpenAI 明确命名了这点；Anthropic 和 DeepMind 有它们自己的等效表述。
- **节奏**：政策是否按声明的计划更新，还是仅在特定事件后更新？声明的节奏更强。
- **独立性**：外部审查是强制的还是可选的？Anthropic 与 Apollo 和美国AI安全研究所合作；OpenAI 与 METR 合作；DeepMind 主要依赖内部 SAG。

## 使用它

`code/main.py` 实现了一个小型决策表差异工具。给定一个能力（自主性、欺骗性对齐、研发自动化、网络提升等），它会输出三个政策各自如何分类该能力，以及会触发哪些缓解措施。这是一个阅读辅助工具，而非政策工具。

## 输出它

`outputs/skill-cross-policy-diff.md` 针对一个特定能力，使用三个框架作为参考，生成一个跨政策比较。

## 练习

1.  运行 `code/main.py`。确认该差异工具的输出与源文档中至少两个你能验证的能力的政策相符。
2.  完整阅读 OpenAI 准备就绪框架 v2。识别每个研究类别。对于每个类别，写一句话说明为什么它被归入研究类别而非跟踪类别。
3.  完整阅读 DeepMind FSF v3，加上2026年4月的跟踪能力等级更新。识别机器学习研发自主性等级1的具体评估标准。你将如何从外部测量它？
4.  策略性表现不佳在 OpenAI 的研究类别中。设计一个能迫使策略性表现不佳的模型暴露其实际能力的评估。参考第1课关于评估-上下文博弈的讨论。
5.  针对一个特定能力（你自选），比较三个政策。指出你认为哪个政策的分类最严格，哪个最宽松。用原文文本来论证。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| 准备就绪框架 | “OpenAI 的扩展政策” | PF v2（2025年4月）；跟踪 vs 研究类别 |
| 跟踪类别 | “强制缓解” | 触发能力+安全报告；SAG审查 |
| 研究类别 | “仅监控” | 被跟踪但不自动触发缓解；包括长期自主性 |
| 前沿安全框架 | “DeepMind 的扩展政策” | FSF v3（2025年9月）+ 跟踪能力等级（2026年4月） |
| CCL | “关键能力等级” | DeepMind 每个领域的阈值（网络、生物、机器学习研发、CBRN） |
| 机器学习研发自主性等级 1 | “研发自动化” | 以具有竞争力的成本完全自动化AI研发流程 |
| 策略性表现不佳 | “战略性表现不佳” | 模型在评估中表现低于实际能力；在OpenAI研究类别中 |
| 工具性推理 | “手段-目的推理” | 关于如何实现目标的推理；DeepMind监控的目标 |
