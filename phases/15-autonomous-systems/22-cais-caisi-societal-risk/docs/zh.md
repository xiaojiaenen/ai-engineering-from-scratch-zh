# CAIS、CAISI 与社会规模风险

> 人工智能安全中心（CAIS，位于旧金山，由Hendrycks和Zhang于2022年创立）发布了四风险框架——恶意使用、AI竞赛、组织风险、失控AI——以及2023年5月由数百名教授和公司负责人签署的关于灭绝风险的声明。CAIS的2026年发布内容包括：用于前沿模型评估的AI仪表板、远程劳动力指数（与Scale AI合作）、超级智能战略论文、AI前沿通讯。另一个独立实体：美国国家标准与技术研究院（NIST）下属的AI标准与创新中心（CAISI）——面向美国政府，专注于网络、生物和化学武器风险的自愿协议和非机密能力评估。CAIS将组织风险列为四大顶级风险之一：安全文化、严格审计、多层防御和信息安全是基础，但通常会被牺牲以换取部署速度。加州SB-53法案若签署，将成为美国首个州级灾难性风险法规。

**类型：** 学习
**语言：** Python（标准库，四风险清单与缓解措施匹配器）
**前置知识：** 阶段15 · 19（RSP），阶段15 · 20（PF + FSF）

## 问题

课程19和20介绍了实验室内部的扩展策略。课程21介绍了独立的能力评估。本课程介绍第三个视角：塑造公共讨论和灾难性AI风险监管基线的民间社会和政府组织。

两个不同的实体很重要。CAIS是一个非营利研究机构，发布关于AI风险的思考框架并协调公共声明。CAISI是NIST内部的一个美国政府中心，与实验室运行自愿协议并进行非机密能力评估。名称相似，但使命不同。从业者应了解两者。

实际内容：CAIS的四风险框架是文献中最广泛引用的社会规模风险分类法。安全文化和组织风险是其中之一，也是从业者最能直接控制的。SB-53（加州）若签署，将成为美国首个州级灾难性风险法规；该法案的框架很重要，因为在美国科技政策中，州级监管历来领先于联邦行动。

## 概念

### CAIS — 人工智能安全中心

- 成立：2022年于旧金山，由Dan Hendrycks及同事创立（“Zhang”指早期合作者，非当前联合创始人；请参阅CAIS网站了解当前领导层）。
- 状态：501(c)(3)非营利组织。
- 2023年重要产出：关于灭绝风险的声明，由数百名研究人员和CEO联署。声明称：“减轻AI导致的灭绝风险应与大流行病和核战争等其他社会规模风险一起，成为全球优先事项。”
- 2026年产出：用于前沿模型评估的AI仪表板、远程劳动力指数（与Scale AI合作）、超级智能战略论文、AI前沿通讯。

### 四风险框架

CAIS的框架将灾难性AI风险归入四个顶级类别：

1.  **恶意使用**：恶意行为者使用AI造成伤害（生物武器合成、虚假信息、网络攻击）。
2.  **AI竞赛**：实验室、公司或国家之间的竞争压力将部署推进到不安全的程度。
3.  **组织风险**：实验室内部动态（安全文化失败、审计不足、安全资源不足）导致不良部署。
4.  **失控AI**：一个足够强大的AI追求与人类福祉相冲突的目标。

这不是唯一的分类法，但它是被引用最多的。这些类别并非互斥——一个因在竞赛中牺牲审计以换取速度而产生的失控AI，涵盖了全部四个类别。

### 组织风险所在

在这四个类别中，组织风险对从业者最具可操作性。一个实验室的安全文化、审计严格程度、防御分层和信息安全，决定了他们的模型是否真正实施了第10-18课的控制措施，还是这些控制措施只是无人核实的清单项目。

具体的组织风险杠杆：

- **安全文化**：团队成员能否在不承担职业风险的情况下提出担忧？CAIS调查发现这是其他杠杆的有力预测指标。
- **严格审计**：外部和内部。仅内部审计会产生乐观的报告。
- **多层防御**：没有单一层次是足够的（这是第15阶段的主题）。
- **信息安全**：模型权重泄露、评估数据泄露、监控绕过技术泄露。第19课中的RAND SL-4是一个具体标准。

### CAISI — AI标准与创新中心

- 在NIST内运作。
- 与前沿实验室运行自愿协议。
- 发布专注于网络、生物和化学武器风险的非机密能力评估。
- 与CAIS不同；缩写容易混淆；请检查URL（nist.gov）以确认您正在阅读的是哪一个。

CAISI的角色是METR私营实验室参与（第21课）的公共、面向政府的对应机构。CAISI的报告是非机密的；METR的报告通常受保密协议限制。两者都读的从业者能获得更全面的图景。

### 加州SB-53

加州参议院法案（2025-2026会期）涉及前沿模型带来的灾难性风险。草案中的关键条款：

- 触发州级义务的特定能力阈值。
- 对AI实验室员工的举报人保护。
- 灾难性故障的事件报告要求。

若签署，它将成为美国首个州级灾难性风险法规。无论签署状态如何，该法案的框架都影响了其他州立法机构处理该问题的方式。加州的从业者应跟踪该法案的状态；其他地方的从业者应阅读它，以了解美国州级监管可能的样子。

### 社会规模风险不是单层问题

第15阶段的主题——纵深防御——也适用于社会层面。没有单一的组织、法规或框架能够消除灾难性风险。生态系统只有在以下情况下才能发挥作用：

- 实验室发布扩展策略（第19、20课）。
- 外部评估者产生测量结果（第21课）。
- 民间社会跟踪和宣传（CAIS）。
- 政府运行自愿计划和基线监管（CAISI、SB-53）。
- 从业者构建多层控制措施（第10-18课）。

这是本阶段的最终综合：前面的每一课都是技术栈中的一层，其完整性比任何单一层的强度更重要。

## 应用它

`code/main.py` 实现了一个小型风险清单工具。给定一个提议的部署，它会根据四个风险类别对该部署进行标记，并返回一个缓解措施清单。这是框架的阅读辅助工具，不能替代人类判断。

## 交付它

`outputs/skill-societal-risk-review.md` 审查部署的社会规模风险态势：它涉及四个类别中的哪一个、有哪些缓解措施、组织风险暴露程度如何。

## 练习

1.  运行 `code/main.py`。输入三个不同规模的合成部署。确认四风险标签是否符合您的预期；找出一个工具标记不足或标记过度的案例。
2.  完整阅读CAIS四风险论文。选择一个风险类别，就您认为2026年该类别最重要的发展写两段文字。
3.  阅读加州SB-53的当前草案。找出一个您认为加强了灾难性风险态势的条款和一个您认为削弱了它的条款。并论证两者。
4.  选择一个您了解的生产级AI部署（您自己的或已发布的）。根据组织风险子杠杆对其进行评分：安全文化、审计严格程度、多层防御、信息安全。哪一个最弱？将其提升到标准水平需要多少成本？
5.  勾勒一个2028年版本的四风险框架，反映一年来额外的能力提升和一年来的额外部署经验。您会添加、删除或重新分组什么？
