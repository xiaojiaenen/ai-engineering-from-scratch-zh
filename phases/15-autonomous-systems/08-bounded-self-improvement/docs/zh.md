# 有界自改进设计

> 研究已归纳出四个用于约束自改进循环的基本原语。它们是每次修改都必须遵守的形式化不变式、不可修改的对齐锚点、每个维度（安全性、公平性、稳健性）都必须满足的多目标约束（而不仅仅是性能），以及在历史指标显示能力损失时暂停循环的回归检测。这些都不是安全的证明——信息论结果（科莫戈洛夫复杂性、洛布定理）限制了任何系统能证明其自身继任者的内容。它们是缓解措施，增加了静默失败的成本。

**类型：** 学习
**语言：** Python（标准库，带不变式检查的有界循环）
**先修课程：** 第15章 · 07（递归自改进），第15章 · 04（动态成长模型）
**时长：** ~60分钟

## 问题所在

第7课的竞速模拟器显示，微小的速率差异会累积成巨大差距。第4课的动态成长模型案例研究表明，循环可能会主动博弈其自身的评估器。这两个结果指向同一个工程问题：你可以对自改进循环施加什么约束，使得这些约束不会被循环本身静默地削弱？

ICLR 2026 RSI研讨会总结报告（openreview.net/pdf?id=OsPQ6zTQXV）确定了四个这样的基本原语。Anthropic的RSP v3.0（第19课）和DeepMind的FSF v3（第20课）都在能力阈值中引用了它们。Meta的HyperAgents工作以及像SAHOO（2026年3月）这样的社区框架在生产环境中实现了其子集。

坦诚地说：这些都是缓解措施。信息论结果限制了任何系统能证明其自身继任者的内容，目前没有任何设计在形式上解决了这个问题。一个约束良好的循环比无界循环更安全，但并非绝对安全。

## 概念

### 原语一：形式化不变式

不变式是每次自我修改前后都必须保持的属性。例如：

- 输出分布基于固定的宪章头（第17课）进行调节。
- 不会向未经授权的端点发起工具调用。
- 内存写入通过一个有日志记录、经过签名的路径。
- 评估器的模块哈希与批准的版本匹配。

不变式由循环无法编辑的外部代码检查。如果提议的修改违反了不变式，则会被拒绝。这可以捕获第4课中提到的动态成长模型式的评估器篡改模式。

难点在于选择那些对安全是必要的、且可在合理时间内计算的不变式。指定过多会阻碍进展；指定过少则会让循环逾越重要属性。

### 原语二：对齐锚点

对齐锚点是循环核心目标的不可变表示，固定在循环的编辑范围之外。例如：

- 从一个只读位置加载的宪章文本（第17课）。
- 位于独立代码库中，拥有独立访问控制的评估器。
- 由人类签名并在每个周期重新验证的工具权限清单。

锚点的作用是防止目标漂移。循环可以改进它追求目标的方式，但无法编辑目标本身。

一个微妙的失败模式：一个循环无法编辑的锚点，仍然可能被一个在解读锚点方式上漂移的循环重新解释。宪法AI（第17课）明确基于推理来处理新情况；那个推理层就是解释漂移发生的地方。锚点是必要的，但不是充分的。

### 原语三：多目标约束

一个优化单一标量分数的循环会找到捷径。一个必须同时满足多个硬性约束的循环，可用的捷径就更少。典型的维度包括：

- 性能（任务级基准测试）
- 安全性（红队评估，对已知恶意输入的拒绝率）
- 公平性（对敏感子群体的差异性影响界限）
- 稳健性（分布外测试集，对抗性输入处理）

只有当每个约束都满足时，修改才会被接受。第13课的成本控制器在此基础上叠加了财务约束。第18课的Llama Guard则作为安全维度插入。

### 原语四：回归检测

每个周期的分数都会与历史分布进行比较。超出配置容差的下降会暂停循环。这可以捕获静默的能力损失，否则这些损失会被循环"改进过去"而被运行平均值吸收。

一种实际的实现方式是：存储最近N个周期的每任务分数。每个新周期计算每任务的增量。如果任何增量低于阈值，则该周期被拒绝并交由人工审查。

### 信息论限制

科莫戈洛夫复杂性和洛布定理为系统能证明其自身的范围设定了上限。施密德胡贝尔的形式化哥德尔机（第4课）旨在达到此类上限的最高点；但尚未有人完成一个非平凡的证明。洛布的结果表明：如果一个系统能证明它相信"如果我证明我应该做X，我就做X"，那么它就会做X，而无需证明它应该做X——这是一个众所周知的自指失败。

对我们这些原语的启示：它们无法关闭安全问题。它们使静默失败的代价更高。一个恶意或漂移的循环如果能悄悄绕过缺失的检查，现在就必须主动破坏一个显式的检查，这是一个更容易检测到的特征。

### 一个实例

假设一个智能体提出一项修改。门控堆栈如下：

1. 不变式检查：模块哈希、工具权限清单、宪章头。
2. 锚点检查：目标陈述与批准版本匹配（字节级或语义级）。
3. 多目标评估：性能、安全、公平、稳健维度。
4. 回归检测：没有维度的下降超过容差。

所有四个条件都必须通过，修改才能生效。任何一个条件失败都会暂停循环。

## 实践

`code/main.py` 在第4课的动态成长模型风格玩具示例上运行一个有界自改进循环，但上面叠加了四个原语。每个原语可以单独启用或禁用。演示表明，每个原语都能捕获特定类型的失败，并且禁用任何一个都会让相应的失败类型通过。

## 部署

`outputs/skill-bounded-loop-review.md` 审核一个提议的有界循环，并评估它实际实现了这四个原语中的哪些，而不是声称实现了哪些。

## 练习

1. 启用所有原语运行 `code/main.py`。确认循环在主要指标上仍然改进，而不会让黑客获胜。
2. 禁用回归检测。构造一个输入，使得静默的能力损失被接受。
3. 禁用多目标约束。展示循环在性能维度上收敛，而安全维度却下降。
4. 为一个编码智能体设计一个对齐锚点。什么样的文本，存储在哪里，如何检查？
5. 阅读ICLR 2026 RSI研讨会总结报告。选择四个原语中的一个，并针对当前技术水平提出一项具体改进建议。

## 关键术语

| 术语 | 人们怎么说 | 它的实际含义 |
|---|---|---|
| 不变式 | "永远为真的属性" | 由外部代码在每次编辑前后检查的属性 |
| 对齐锚点 | "固定的目标" | 位于循环编辑范围之外的不可变核心目标表示 |
| 多目标约束 | "所有维度都必须满足" | 性能、安全、公平、稳健——全部必要 |
| 回归检测 | "下降则暂停" | 当历史指标增量显示能力损失时暂停循环 |
| 科莫戈洛夫界限 | "信息论限制" | 限制系统能证明其自身继任者的内容 |
| 洛布定理 | "自指陷阱" | 系统可以在不证明应该做某事的情况下，基于"我应该"而行动 |
| 门控堆栈 | "分层检查" | 多个原语组合；任何失败都拒绝修改 |
| 有界改进 | "缓解措施，而非证明" | 提高静默失败的成本；未关闭安全问题 |
