# 宪法式AI与规则覆盖

> Anthropic 于2026年1月22日发布的《Claude宪法》共79页，采用CC0许可。它从基于规则的对齐转向基于推理的对齐，并建立了四级优先层次：（1）安全性与支持人类监督，（2）伦理，（3）Anthropic准则，（4）有用性。行为分为硬编码禁止项（生物武器提升、儿童性虐待材料），操作者和用户无法覆盖；以及软编码默认项，操作者可在界定范围内调整。2022年的原始版本（Bai等人）通过自我批评和基于宪法的RLAIF训练无害性。诚实地说，基于推理的对齐依赖于模型将原则泛化至未预见情况的能力。Anthropic 2023年的参与式实验显示，公众来源原则与企业原则之间存在约50%的分歧；2026年版本并未纳入这些发现。

**类型：** 学习  
**语言：** Python（标准库、四级优先级解析器）  
**前置要求：** 第15阶段 · 06（自动化对齐研究），第15阶段 · 10（权限模式）  
**时间：** 约60分钟

## 问题所在

一个已部署的智能体会遇到其设计者从未预见的输入。没有足够长的规则列表能覆盖所有情况。也没有足够短的规则列表能在计算压力下快速应用。实际问题是：如何对齐一个智能体，使其遵循的原则既能处理长尾案例，又能满足快速推理的需求？

基于规则的对齐（RBA）：列出所有禁止事项。检查速度快、易于审计，但无法保持更新，且经常在未预料的相似案例上过度拒绝。基于推理的对齐（2026年Claude宪法）：编码原则，让模型进行推理。能扩展至未见案例，更难审计，失败模式是原则误用而非遗漏规则。

2026年宪法采取了明确的中间立场。硬编码禁止项——那些错误性不依赖于情境的事物（生物武器提升、儿童性虐待材料）——属于RBA：无论操作者或用户指令如何，都绝不允许。其他所有事项都在四级层次内基于推理处理：安全性与支持人类监督优先；伦理其次；Anthropic宣布的准则第三；有用性最后。操作者可在软编码区域内调整默认值，但不能触及硬编码禁止项。

## 核心概念

### 四级优先层次

1. **安全性与支持人类监督。** 最高优先级。模型优先不削弱人类和Anthropic监督及纠正AI的能力。这不是“谨慎行事”；而是专门指“不以使人类监督更困难的方式行事”。
2. **伦理。** 诚实、避免伤害个人、不欺骗、不操纵。当与Anthropic准则冲突时，伦理优先。
3. **Anthropic准则。** Anthropic认为重要的操作规范：产品范围、交互模式、何时使用何种工具。
4. **有用性。** 最低优先级。在更高优先级约束下尽可能提供帮助。

当各层次冲突时，更高层次优先。这与Unix优先级或网络服务质量（QoS）的结构相同——这种设计旨在产生可预测的解决方案，而非在任何单一维度上追求最佳行为。

### 硬编码禁止项 vs 软编码默认项

**硬编码项：**
- 生物武器/化学、生物、放射性和核（CBRN）提升
- 儿童性虐待材料（CSAM）
- 对关键基础设施的攻击
- 当被直接询问时欺骗用户关于模型的身份

操作者无法覆盖这些。用户无法覆盖这些。它们在可能的情况下通过模型权重层（强化学习人类反馈/宪法式AI训练）强制执行，在不可能的情况下通过推理层强制执行。

**软编码默认项（操作者可调）：**
- 响应长度默认值
- 话题范围（模型可以拒绝操作者部署范围外的话题）
- 风格（正式 vs 随意）
- 工具使用模式

操作者的调整发生在声明的边界内。操作者无法通过重命名来移除硬编码禁止项。

### 2022年宪法式AI训练

原始的宪法式AI（Bai等人，2022年）训练无害性：
1. 生成对一组提示的响应。
2. 要求模型根据宪法（明确原则）对每个响应进行批评。
3. 基于批评修订响应。
4. 对修订后的响应对进行基于AI反馈的强化学习（RLAIF）。

结果：一个能以原则性解释拒绝有害请求的模型，而非笼统拒绝。2026年宪法使用了该训练的后继方法，并额外增加了关于明确层次结构的后续训练。

### 基于推理的对齐能捕捉和遗漏什么

**能捕捉：**
- 允许的基元组合中，原则清晰适用的未预见组合。
- 与禁止项相似的新型请求。
- 依赖“你没说X被禁止”的社会工程攻击。

**会遗漏：**
- 利用原则模糊性的攻击（“用户要求了这个，所以有用性说可以”）。
- 原则以未预见方式冲突，且层次顺序模糊的情景。
- 训练周期中原则解释的缓慢漂移（重新解释）。

### 2023年参与式实验

Anthropic在2023年进行了一项实验，比较了企业编写的宪法与通过公众输入（约1000名美国受访者）生成的宪法。两个版本在约50%的原则上达成一致。在分歧处，公众来源版本在某些问题（政治内容处理）上更严格，在其他问题（AI身份自我披露）上则较不严格。2026年宪法并未纳入公众来源的发现。这是该方法中记录在案的紧张关系。

### 为什么需要硬编码禁止项

仅靠基于推理的对齐无法关闭长尾风险。能让模型接受一个前提（例如，“我们是经许可的生物武器研究实验室”）的攻击者，通常可以绕过依赖案例推理的原则。硬编码禁止项不会屈服于前提框架。它们是第14课所述“硬性宪法限制”在对齐层的体现。

### 宪法在技术栈中的位置

宪法不是第14课的紧急关闭开关。它位于模型层：模型权重被训练为偏好什么。紧急关闭开关和金丝雀令牌位于运行时层：运行时允许什么。两者都是必需的。一个因为模型权重宽容而触发所有错误动作的运行时，是运行时问题。一个因为运行时过度限制而拒绝所有正确动作的模型，是运行时问题。不同层次覆盖不同类别。

## 使用它

`code/main.py` 实现了一个最小的四级优先级解析器。解析器接收一个拟议动作和一组原则评估（安全性、伦理、准则、有用性），并返回该动作、拒绝或修改后的动作。驱动程序运行一个小型案例集：明确允许、明确禁止、硬编码禁止项、跨层次的模糊案例。

## 部署它

`outputs/skill-constitution-review.md` 审计部署的宪法层：什么是硬编码的，什么是软编码的，操作者可以调整什么，以及四级层次是否确实是解析顺序。

## 练习

1. 运行 `code/main.py`。确认硬编码禁止项即使在高有用性时也会触发。修改解析器使有用性权重高于伦理；观察失败模式。

2. 阅读《Claude宪法》（公开，79页，CC0）。识别一条你认为规范不足的原则。写两段文字解释具体的模糊性，并提出更严谨的表述。

3. 为一个客户支持智能体设计一套软编码默认项。操作者可以调整什么？操作者不能触碰什么？请为每个边界提供理由。

4. 阅读Bai等人2022年的CAI论文。描述一个宪法式AI的批评-修订循环会产生比笼统规则更差结果的案例。识别该类别。

5. Anthropic 2023年的参与式实验发现，公众原则与企业原则之间存在约50%的分歧。选择一个对生产部署重要的类别（例如，政治中立性）。提出一种设计，让操作者能表达自己的价值观，同时硬编码禁止项保持不变。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|---|---|---|
| 宪法式AI | “Anthropic的对齐方法” | 自我批评 + 基于书面宪法的RLAIF |
| 基于推理的对齐 | “原则，而非规则” | 模型基于原则推理以处理未见案例 |
| 硬编码禁止项 | “绝不做X” | 操作者或用户无法覆盖的基于规则的禁令 |
| 软编码默认项 | “操作者可调” | 在声明边界内的行为，操作者控制 |
| 四级层次 | “优先级顺序” | 安全性 > 伦理 > 准则 > 有用性 |
| RLAIF | “AI反馈强化学习” | 奖励来自模型生成批评的强化学习 |
| 参与式宪法 | “公众来源的原则” | 2023年Anthropic实验；与企业原则约50%分歧 |
| 原则漂移 | “解释滑移” | 模型对固定原则文本解读的缓慢变化 |
