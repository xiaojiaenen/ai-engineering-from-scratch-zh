# AI Scientist v2 — 工坊级自主研究系统

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）能够执行完整的研究循环：提出假设、编写代码、进行实验、生成图表、撰写论文并提交。这是首个在 ICLR 2025 工坊中通过同行评审的生成式论文系统。独立评估（Beel 等人）发现 42% 的实验因编程错误而失败，且文献综述常将已知概念误标为创新。Sakana 的官方文档警告该代码库会执行大语言模型编写的代码，并建议使用 Docker 进行隔离。这两个方面都至关重要。

**类型：** 学习
**语言：** Python（标准库、研究循环状态机示例）
**前置知识：** 第 15 阶段 · 03（AlphaEvolve），第 15 阶段 · 04（DGM）
**时间：** 约 60 分钟

## 问题背景

研究是一项开放性任务。与 AlphaEvolve 的算法搜索或 DGM 的基准约束自我修正不同，研究成果没有机器可验证的正确性标准。论文由评审专家评判，而非单元测试。这使得研究循环更难闭合——而一旦成功闭合，其价值也更大，因为研究是复利式进步的核心驱动力。

AI Scientist v1（Sakana，2024）通过从人工撰写的模板出发来闭合循环。大语言模型在固定框架内填充实验内容。AI Scientist v2（Yamada 等人，2025）则移除了模板依赖，采用基于视觉-语言模型评审循环的智能体树搜索系统。该系统生成想法、实施实验、产生图表、撰写论文，并根据评审反馈进行迭代。

同行评审结果：一篇由 v2 生成的论文在 ICLR 2025 工坊中被接受（已披露生成方式）。独立评估结果：该系统远未达到可靠水平。两者皆为事实。

## 核心概念

### 系统架构

1.  **想法生成。** 大语言模型基于主题和已有文献提出研究想法。v1 使用模板；v2 采用智能体搜索，在假设空间中探索。
2.  **新颖性检查。** 通过文献检索步骤检查想法是否已被发表。Beel 等人的评估发现此步骤存在误标问题——成熟方法常被错误分类为创新。
3.  **实验计划。** 智能体起草实验方案并编写代码。
4.  **执行。** 代码在沙箱中运行。失败会被反馈到重试循环中。在 Beel 等人的测量中，42% 的实验因编程错误在此阶段失败。
5.  **图表生成。** 视觉-语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术新增。
6.  **论文撰写。** 大语言模型起草论文，并与内部评审者迭代修改。
7.  **可选：提交。** 论文被提交到学术会议或期刊。

### 工坊接受结果的意义

一篇由 v2 生成的论文通过了 ICLR 2025 工坊的同行评审。作者向程序委员会披露了论文的生成来源。这是一个数据点；但这并不意味着可以宣称该系统“能做研究”。

重要背景：工坊论文的标准低于主会议论文。同行评审存在不确定性；任何一天接受的论文都只是少数。一次成功是概念验证，而非可靠性声明。那篇 Nature 2026 论文记录了端到端循环，且其本身是与人类研究者合著的；这并非“系统撰写了 Nature 论文”。

### 独立评估的发现

Beel 等人（arXiv:2502.14297）进行了外部评估。主要发现包括：

- **实验失败。** 42% 的实验因编程错误失败（错误的导入、形状不匹配、未定义变量）。重试循环捕获了部分错误，但非全部。
- **新颖性误标。** 文献检索步骤频繁将成熟概念标记为新颖。这是研究领域的“幻觉”。
- **呈现质量差距。** 视觉-语言图表评生产出的图表具有出版级视觉效果，掩盖了潜在的实验缺陷。

最后一点对本阶段的学习尤为重要。一个系统若能生成令人信服的输出，却未进行令人信服的研究，比一个明显失败的系统更危险。评估必须触及底层主张，而不能停留在图表表面。

### 沙箱逃逸风险

Sakana 自身的仓库 README 警告道：

> 由于该软件会执行大语言模型生成的代码，我们无法保证其安全性。存在引入危险包、不受控的网络访问以及生成非预期进程的风险。使用风险自负，并考虑 Docker 隔离。

这是自主性在未验证领域中的实际体现。大语言模型编写代码；代码运行；代码可以执行进程允许的任何操作。如果没有严格限制文件系统、网络和进程操作的沙箱，任何自主研究智能体都可能窃取数据、消耗计算资源或重写自身。

AlphaEvolve 的沙箱策略相对简单，因为其评估器很严格。AI Scientist v2 的循环则运行着目标开放的开放式代码。因此它需要更强的隔离（至少 Docker；优选 seccomp / gVisor），并在系统外提交前对每份产出进行人工审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失效模式 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准 | 受限于评估器严格性 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励黑客 |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（较弱） | 实验失败、误标、美化掩盖缺陷 |

v2 在三者中拥有最弱的自动评估器、最广泛的输出面，以及通向公开成果的最短路径。其安全工作主要依赖操作控制（沙箱、审查、披露）。

## 动手实践

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 论文 → 评审 → 接受或迭代。每个状态都根据 Beel 等人的发现设定了可配置的失败概率。运行模拟器 N 个循环，并统计：

- 有多少想法最终被提交。
- 有多少提交论文包含被精美图表掩盖的关键实验缺陷。
- 重试预算如何权衡质量与产出率。

## 部署上线

`outputs/skill-ai-scientist-sandbox-review.md` 是一份针对研究循环智能体产物的双关卡审查清单，用于在离开沙箱前进行检查。

## 练习

1.  使用默认参数运行 `code/main.py`。循环运行中产生“干净”论文的比例是多少？产生包含实验失败缺陷但被图表评审美化掩盖的论文的比例是多少？
2.  默认参数已采用 Beel 等人的 42% / 25% 数据。分别使用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 和 `--experiment-failure 0.60 --novelty-mislabel 0.40` 重新运行。在两次运行中，“美化但有缺陷”的份额如何变化？
3.  阅读 Sakana 的 AI Scientist v2 仓库 README 中关于沙箱要求的内容。针对多天自主运行，列举除 Docker 外你还愿意应用的另外两项限制。
4.  阅读 Beel 等人第 4 节关于呈现质量差距的内容。设计一个额外的评估器，用于捕捉外观精美但实验有缺陷的论文。
5.  为研究智能体产物提出一种比“博士阅读每篇论文”更具扩展性的人工审查方案。识别瓶颈并围绕它进行设计。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|---|---|---|
| AI Scientist v1 | “Sakana 的模板化研究智能体” | 将实验填充到固定脚手架中 |
| AI Scientist v2 | “无模板研究智能体” | 采用视觉-语言模型图表评审的智能体树搜索 |
| 智能体树搜索 | “分支研究智能体” | 并行展开多个实验方案；通过内部评审器剪枝 |
| 视觉-语言评审 | “VLM 美化图表” | 多模态模型读取图表并重写以提高清晰度 |
| 文献检索 | “新颖性检查” | 搜索先前工作以确认想法新颖性——已知存在误标问题 |
| 美化掩盖 | “漂亮论文，破败研究” | 呈现质量超过实验质量；隐藏缺陷 |
| 沙箱逃逸 | “大语言模型代码突破” | 智能体执行的代码做了循环设计者未意图的操作 |
