# 思维树与LATS：审慎搜索

> 单条思维链轨迹无法回溯。思维树（ToT，Yao等，2023年）将推理转化为带自我评估的树结构。LATS（Zhou等，2024年）在蒙特卡洛树搜索框架下统一了ToT、ReAct与Reflexion。在24点游戏中，思维链（CoT）准确率仅为4%，思维树提升至74%；LATS在HumanEval测试中达到92.7%的通过率。

**类型：** 构建
**语言：** Python（标准库）
**前置知识：** 第14阶段·01（智能体循环）、第14阶段·03（Reflexion）
**时间：** 约75分钟

## 学习目标

- 将推理建模为搜索：节点是"思维"，边是"扩展"，价值是"前景评估"
- 实现基于标准库的思维树风格广度优先搜索，包含自我评估评分机制
- 扩展为玩具版LATS蒙特卡洛树搜索循环，包含选择/扩展/模拟/回溯流程
- 判断何时值得使用搜索的token倍增成本（如24点游戏、代码生成），何时单次轨迹足够（如简单问答）

## 问题本质

思维链是线性行进。若首步出错，后续步骤都将基于错误前提构建。在24点游戏（用+ − × ÷运算使四个数字结果为24）中，GPT-4思维链准确率仅4%。模型过早选错子表达式后无法纠正。

推理需要的是：提出多候选方案、评估它们、选择有潜力方案、遇到死胡同时能回溯。这正是搜索。思维树和LATS是两种标准实现方案。

## 核心概念

### 思维树（Yao等，NeurIPS 2023）

每个节点是连贯的中间步骤（"一个思维"）。每个节点可扩展出K个子思维。大语言模型通过评分提示自我评估每个节点。搜索探索树结构——可采用广度优先、深度优先或集束搜索。

```
                     (root: "find 24 from 4 6 4 1")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- Score: HIGH
              /   \              |                  |
          ...    ...          ...                finish
```

自我评估是核心支撑。论文展示三种变体：`sure / likely / impossible`分类、`1..10`数值评分、候选方案投票。在24点游戏中，这三种方案均显著优于思维链（GPT-4实现4%到74%的提升）。

### LATS（Zhou等，ICML 2024）

LATS在蒙特卡洛树搜索框架下统一了思维树、ReAct和Reflexion。大语言模型承担三种角色：

- **策略网络**：提出候选下一步动作（ReAct风格）
- **价值函数**：为部分轨迹评分（思维树风格的自我评估）
- **自省器**：失败时生成自然语言反思（Reflexion风格），并用于重新引导后续推演

环境反馈（观测结果）融入价值函数，使搜索基于真实工具结果而非单纯模型观点。论文发布时的成果：GPT-4在HumanEval测试通过率达92.7%（当时最优），GPT-3.5在WebShop任务平均得分为75.9（接近微调模型水平）。

### 蒙特卡洛树搜索精要

每次迭代包含四个阶段：

1. **选择** — 从根节点到叶节点使用UCT（树的置信上界）遍历
2. **扩展** — 通过策略网络生成K个子节点
3. **模拟** — 从子节点使用策略网络推演，用价值函数（或环境奖励）为叶节点评分
4. **回溯** — 沿路径更新访问计数和价值估计

UCT公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。首项为利用项；次项为探索项。根据任务调整`c`参数。

### 成本现实

搜索使token消耗激增。思维树在24点游戏的token消耗是思维链的100-1000倍。LATS类似。这并非免费资源，应将搜索用于：

- 单次轨迹明显不足的任务（24点游戏、复杂代码）
- 时钟时间不如正确性重要的任务
- 拥有廉价可靠价值函数的任务（代码的单元测试、数学的显式目标）

若任务只有单一正确答案且评估器噪声大，搜索往往适得其反——它会找到"高评分"的错误答案。

### 2026年技术定位

多数生产环境智能体不运行LATS。它们使用带工具验证的ReAct（CRITIC，课程05）。搜索出现在特定细分领域：

- 以测试为价值函数的编程智能体（HumanEval类任务）
- 探索多查询路径的深度研究智能体
- LangGraph子图中规划密集型工作流

AlphaEvolve（课程11）是2025年的极端案例：基于代码的进化搜索、机器可检查的适应度函数、前沿突破（56年来首次实现4×4矩阵乘法改进）。

## 关键术语

| 术语 | 常见表述 | 实际含义 |
|------|----------|----------|
| 思维树 | "分支思维链" | Yao等提出——带自我评估的思维节点树 |
| LATS | "大语言模型蒙特卡洛树搜索" | Zhou等提出——在MCTS框架下统一ToT+ReAct+Reflexion |
| UCT | "置信上界" | 平衡利用（Q值）与探索（ln N / n）的选择公式 |
| 价值函数 | "状态优劣评估" | 提示驱动的大语言模型评分或环境奖励；输入回溯过程 |
| 策略网络 | "动作提议器" | ReAct风格生成器；输出候选思维/动作 |
| 推演 | "模拟轨迹" | 使用策略网络从节点到叶节点的路径行走，通过价值函数评分 |
| 回溯 | "更新祖先节点" | 将叶节点奖励沿路径上传，更新访问计数与Q值 |
| 搜索成本 | "token爆炸" | 24点游戏中消耗量为思维链的100-1000倍；采用前需规划预算 |
