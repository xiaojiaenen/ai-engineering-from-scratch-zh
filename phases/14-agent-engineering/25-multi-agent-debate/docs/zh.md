# 多智能体辩论与协作

> Du 等人（ICML 2024，《思维社会》）运行 N 个模型实例，它们独立提出答案，然后在 R 轮中相互批评以达成收敛。该方法提升了事实准确性、规则遵循和推理能力。在 token 成本上，稀疏拓扑优于全连接网格。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**前提：** 第14章 · 12（工作流模式），第14章 · 05（自我精炼与 CRITIC）
**时间：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提议者，R 轮迭代，最终收敛于一个共享答案。
- 描述辩论如何提升事实准确性、规则遵循和推理能力。
- 解释稀疏拓扑：并非每位辩手都需要看到其他所有人。
- 实现一个基于脚本化 LLM 的标准库辩论，包含全连接网格和稀疏变体；衡量 token 成本与准确率的关系。

## 问题

自我精炼（第05课）是一个模型自我批评——存在群体思维的风险。CRITIC（第05课）将批评基于外部工具——这些工具并非总是可用。辩论引入了第三种模式：多个实例，相互批评，通过分歧实现收敛。

## 概念

### 思维社会（Du 等人，ICML 2024）

- N 个模型实例针对同一问题独立提出答案。
- 在 R 轮中，每个模型阅读其他人的提议并提出批评。
- 模型根据批评更新其答案。
- R 轮结束后，返回收敛的答案。

由于成本考虑，原始实验使用 N=3, R=2。在困难问题（MMLU, GSM8K, 国际象棋走步有效性、传记生成）上，增加智能体数量和轮次可以提高准确率。

跨模型组合优于单一模型的辩论：ChatGPT + Bard 的组合 > 任一单独模型。

### 稀疏拓扑

《通过稀疏通信拓扑改进多智能体辩论》(arXiv:2406.11776, 2024-2025) 表明，全连接网格辩论并非总是最优。稀疏拓扑（星型、环形、中心辐射）可以在更低的 token 成本下达到相当的准确率。每位辩手只看到同伴的一个子集。

影响：
- 全连接网格 N=5, R=3 = 5 × 3 = 15 个提议，每个阅读 4 个同伴 = 60 次批评操作。
- 星型 N=5, R=3（1 个中心 + 4 个分支）= 15 个提议，分支只阅读中心 = 12 次批评操作。

### 辩论何时有帮助

- **事实准确性。** N 个独立提议，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走步有效性——一个模型可能遗漏规则，其他模型可以抓住。
- **开放式推理。** 多种框架逐步逼近正确答案。

### 辩论何时有害

- **延迟敏感的用户体验。** N × R 轮串行迭代带来延迟，这可能是你无法承受的。
- **成本敏感的扩展。** 每个问题 N × R 个 token。
- **简单事实查询。** 一次查询比五次辩论更便宜。

### 2026 年的实际应用示例

- **Anthropic 编排-工作者**（第12课）——带有合成步骤的辩论变体。
- **LangGraph 监督者**（第13课）——中央路由器 + 专家智能体可以将辩论实现为一个节点。
- **OpenAI Agents SDK**（第16课）——智能体来回交接进行迭代批评。
- **多智能体评估**——将辩论与评估-优化器结合以获取评估信号。

### 这种模式出错的地方

- **收敛崩溃。** 所有智能体都收敛于第一个错误的答案。通过强制要求分歧轮次来缓解。
- **中心故障。** 在星型拓扑中，一个糟糕的中心会带坏所有人。轮换中心或使用多个中心。
- **提示同质化。** 所有智能体使用相同的提示；它们会产生相同的答案。使用多样化的提示和/或模型。

## 动手构建

`code/main.py` 实现了标准库辩论：

- `Debater` 类（脚本化 LLM，每个辩手的意见会漂移）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实型，一个规则型，一个推理型。
- 度量指标：收敛答案、收敛轮次、总批评操作数。

运行它：

```
python3 code/main.py
```

输出：每种协议的准确率和成本；在 2/3 个问题上，稀疏拓扑以更低成本匹配了全连接网格的效果。

## 使用它

- **Anthropic 编排-工作者** 用于简单的 2-3 个工作者的辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **定制方案** 用于研究或专门的正确性保证。

## 练习

1. 实现一个“强制分歧”规则：在第 1 轮，每个辩手必须提出一个独特的提议。衡量其对收敛速度的影响。
2. 添加一个基于置信度的加权聚合：辩手返回（答案，置信度）；聚合器按置信度加权。这有帮助吗？
3. 将一个“智能体”替换为一个具有不同观点的、不同的脚本化 LLM。异质性是否提高了准确率？
4. 在你的 3 个问题上，衡量全连接网格与稀疏拓扑的 token 成本。绘制成本 vs 准确率图。
5. 阅读《思维社会》论文。将你的玩具移植到 N=5, R=3。什么出问题了？什么变得更好了？

## 关键术语

| 术语 | 人们怎么说 | 其实际含义 |
|------|----------------|------------------------|
| 辩论 | “多智能体批评” | N 个提议者，R 轮交叉批评，最终收敛 |
| 全连接网格 | “每个人都阅读每个人” | 每位辩手每轮都阅读每个同伴 |
| 稀疏拓扑 | “受限的同伴视图” | 辩手只阅读同伴的一个子集 |
| 中心辐射型 | “星型拓扑” | 一个中心辩手，N-1 个分支只阅读该中心 |
| 收敛 | “达成一致” | 辩手收敛于一个共享答案 |
| 思维社会 | “Du 等人的辩论论文” | ICML 2024 多智能体辩论方法 |
