# LangGraph：状态图与持久化执行

> LangGraph 是 2026 年底层状态化编排的参考实现。智能体是一个状态机；节点是函数；边是转换；状态不可变并在每个步骤后检查点。可从任何故障点精确恢复执行。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 14 · 01（智能体循环），阶段 14 · 12（工作流模式）
**时长：** 约 75 分钟

## 学习目标

- 描述 LangGraph 的核心模型：具有不可变状态、函数节点、条件边和步骤后检查点的状态机。
- 列出文档强调的四大能力：持久化执行、流式传输、人机协同、全面记忆。
- 解释 LangGraph 支持的三种编排拓扑：监督者模式、点对点（群蜂）模式、分层（嵌套子图）模式。
- 使用标准库实现一个具有不可变状态、条件边以及检查点/恢复周期的状态图。

## 问题所在

智能体和工作流共享一个常见问题：当一个 40 步的运行在第 38 步失败时，你希望从第 38 步恢复，而非从头开始。二等公民的状态模型迫使开发者围绕一个假定全新运行的库来拼凑重试逻辑。

LangGraph 的设计解答：状态是一等类型化对象，变更是显式的，检查点在每个节点后持久化。恢复只需一个 `load_state(session_id)` 调用。

## 核心概念

### 图

一个图由以下部分定义：

- **状态类型。** 一个类型化字典（或 Pydantic 模型），每个节点读取并修改它。
- **节点。** 纯函数 `(state) -> state_update`。返回后，更新合并到状态中。
- **边。** 节点之间的条件或直接转换。
- **入口与出口。** `START` 和 `END` 哨兵节点标记边界。

示例：一个包含 `classify`、`refund`、`bug`、`sales`、`done` 节点的智能体——一个作为图的路由工作流。

### 持久化执行

每个节点返回后，运行时序列化状态并将其写入检查点存储器（SQLite、Postgres、Redis、自定义）。如果在第 N 步失败，运行时可以 `resume(session_id)` 并从第 N+1 步继续执行，状态完全一致。

LangGraph 文档明确指出了对此有需求的生产用户：Klarna、Uber、摩根大通。其主张并非图的形状本身，而是图的形状加上检查点使得恢复成本低廉。

### 流式传输

每个节点可以产出部分输出。图将每节点的增量事件流式传输给调用者，以便 UI 在图运行时实时更新。

### 人机协同

在节点之间检查并修改状态。实现方式：在关键节点前暂停，将状态呈现给人类，接受修改，然后恢复。检查点使此过程变得简单，因为状态已经序列化。

### 记忆

短期（单次运行内——对话历史保存在状态中）和长期（跨次运行——通过检查点加独立长期存储实现持久化）。LangGraph 通过工具与外部记忆系统（Mem0、自定义）集成。

### 三种拓扑

1.  **监督者模式。** 中心路由 LLM 调度专家子智能体。`create_supervisor()` 在 `langgraph-supervisor` 中（尽管 LangChain 团队在 2026 年推荐直接通过工具调用实现，以获得更多上下文控制）。
2.  **群蜂/点对点模式。** 智能体通过共享的工具表面直接交接。没有中心路由器。
3.  **分层模式。** 监督者管理子监督者，实现为嵌套子图。

### 此模式的常见误区

- **检查点过小。** 仅检查点对话轮次会导致工具状态和内存写入无法恢复。必须序列化完整状态。
- **非确定性节点。** 恢复假设节点输入产生相同的状态更新。必须捕获随机种子、系统时钟、外部 API 调用。
- **过度使用条件边。** 每条边都是条件的图是一个无法推理的状态机。优先使用带偶尔分支的线性链。

## 动手构建

`code/main.py` 实现了一个标准库的状态图：

- `State` — 一个包含 `messages`、`step`、`route`、`output`、`human_approval` 的类型化字典。
- `Node` — 接受状态并返回更新字典的可调用对象。
- `StateGraph` — 节点 + 边 + 条件边 + 运行 + 恢复。
- `SQLiteCheckpointer`（内存模拟）— 在每个节点后序列化状态；`load(session_id)` 恢复。
- 一个演示图：分类 -> 分支（退款/缺陷/销售）-> 人工门控 -> 发送。

运行它：

```
python3 code/main.py
```

跟踪记录显示第一次运行在人工门控处失败，持久化，然后恢复并产生最终输出。

1.  当分类置信度低于阈值时，从 `classify` 向 `end` 添加一条条件边。在人工手动设置 `route` 后恢复运行。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| 状态图 | “智能体作为状态机” | 类型化状态 + 节点 + 边 + reducer |
| 检查点存储器 | “持久化后端” | 在每个节点后序列化状态；实现恢复 |
| reducer | “状态合并器” | 将当前状态与节点的更新进行合并的函数 |
| 条件边 | “分支” | 由状态函数选择的边 |
| 子图 | “嵌套图” | 作为另一个图中的节点使用的图 |
| 持久化执行 | “从故障中恢复” | 使用精确状态在上一个成功节点处重新启动 |
| 监督者 | “路由 LLM” | 专家子智能体的中心调度器 |
| 群蜂 | “点对点智能体” | 智能体通过共享工具交接；无中心路由器 |
