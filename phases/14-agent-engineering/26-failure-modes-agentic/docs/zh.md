# 故障模式：智能体为何失效

> MASFT（加州大学伯克利分校，2025）将14种多智能体故障模式归类为3大类别。微软的分类体系文档记录了现有AI故障如何在智能体场景中被放大。行业现场数据归结为五种反复出现的模式：幻觉行为、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 学习 + 实践
**语言：** Python（标准库）
**前置要求：** 第14阶段 · 05（自我优化与CRITIC），第14阶段 · 24（可观测性）
**时间：** 约60分钟

## 学习目标

- 列举MASFT的三个故障类别，以及每个类别中至少四种具体模式。
- 解释为何智能体故障会放大现有的AI故障模式（如偏见、幻觉）。
- 描述五种行业反复出现的模式及其缓解措施。
- 使用标准库实现一个检测器，为智能体跟踪标记故障模式标签。

## 问题所在

团队部署的智能体在90%的跟踪中运行良好。那10%的故障并非随机噪声——它们归结为少数几个反复出现的类别。一旦你能为它们命名，你就可以监控并修复它们。

## 核心概念

### MASFT（加州大学伯克利分校，arXiv:2503.13657）

多智能体系统故障分类体系。14种故障模式聚类为3大类别。标注者间一致性科恩卡帕系数为0.88——这些类别可以可靠区分。

核心论点：故障是多智能体系统设计的根本缺陷，而非可通过改进基础模型来修复的LLM局限性。

### 微软智能体AI系统故障模式分类

- 现有AI故障（偏见、幻觉、数据泄露）在智能体场景中被放大。
- 新故障源于自主性：大规模意外行为、工具误用、任务漂移。
- 该白皮书是智能体产品的风险登记册。

### 智能体AI中的故障特征刻画（arXiv:2603.06847）

- 故障源于编排、内部状态演化及环境交互。
- 不仅仅是“糟糕的代码”或“糟糕的模型输出”。

### LLM智能体幻觉调查（arXiv:2509.18970）

两种主要表现形式：

1. **指令遵循偏差** — 智能体不遵循系统提示。
2. **长程上下文误用** — 智能体忘记或误用早期对话轮次中的上下文。

子意图错误：遗漏（遗漏步骤）、冗余（重复步骤）、顺序错乱（步骤乱序）。

### 五种行业反复出现的模式

Arize、Galileo、NimbleBrain 2024-2026年的现场分析归结为：

1. **幻觉行为。** 智能体调用不存在的工具或编造参数。
2. **范围蔓延。** 智能体将任务扩展到用户要求之外（创建额外的PR、发送额外的电子邮件）。
3. **级联错误。** 一个错误调用引发下游效应。一个虚构SKU的幻觉触发了四次API调用——导致多系统事件。
4. **上下文丢失。** 长期任务忘记早期轮次的约束条件。
5. **工具误用。** 以错误的参数调用正确的工具，或完全调用了错误的工具。

级联错误是最致命的。智能体无法区分“我失败了”和“任务不可能完成”，并且通常在400错误时仍会幻觉出成功消息以关闭循环。

### 缓解措施：每一步设置关卡

在推理链的每一步设置自动化验证关卡，根据环境状态检查事实依据。具体而言：

- 每步安全分类器（第21课）。
- 工具调用参数验证（第06课）。
- 将检索内容与已知事实交叉核对（第05课，CRITIC）。
- 通过重新探测状态检测成功幻觉（文件是否真的创建了？）。

### 故障监控常见的错误做法

- **只标记崩溃。** 大多数智能体故障会产生看似有效的输出。需要内容层面的检查。
- **没有基线。** 漂移检测需要最后已知正常状态；没有它，就无法判断“情况是否正在恶化”。
- **过度告警。** 每个故障都触发页面警报。应进行聚类和速率限制。

## 动手实践

`code/main.py` 实现了一个基于标准库的故障模式标记器：

- 一个覆盖五种模式的合成跟踪数据集。
- 每种模式的检测函数（基于工具调用、输出、重复操作的签名模式）。
- 一个标记器，为每个跟踪标记标签并报告模式分布。

运行它：

```
python3 code/main.py
```

输出：每个跟踪的标签 + 聚合分布，这是Phoenix跟踪聚类所揭示内容的一种廉价复现。

1. 为“成功幻觉”添加一个检测器：智能体返回成功，但目标状态未发生改变。
2. 标记你构建的产品的100条真实跟踪。哪种模式占主导地位？修复它的成本是多少？

## 关键术语

| 术语 | 人们通常的说法 | 实际含义 |
|------|----------------|------------------------|
| MASFT | “多智能体故障分类体系” | 伯克利的14种模式分类 |
| 级联错误 | “波浪式失败” | 一个早期错误通过N个步骤传播 |
| 上下文丢失 | “忘记约束条件” | 长期对话轮次遗漏了早期轮次的事实 |
| 工具误用 | “用错工具/参数错误” | 调用有效，但调用方式错误 |
| 成功幻觉 | “伪造完成” | 智能体在400错误时声称成功；状态未改变 |
| 范围蔓延 | “越界” | 智能体做了超出要求的事情 |
| 指令遵循偏差 | “违抗” | 忽略系统提示或用户约束 |
| 子意图错误 | “计划缺陷” | 计划执行中的遗漏、冗余、顺序错乱 |
